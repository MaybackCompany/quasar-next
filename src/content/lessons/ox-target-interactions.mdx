>
  Most FiveM servers have things players can interact with: NPCs, doors, trunks, registers, ATMs. Before `ox_target`, building those interactions meant lots of repeated distance checks and prompt code. With `ox_target`, most of that becomes one small options table. This lesson teaches that pattern slowly.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_ox_target_interactions
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_ox_target_interactions/
  fxmanifest.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

client_script 'client.lua'

dependency 'ox_target'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `client.lua` and paste this:

    ```lua
CreateThread(function()
    exports.ox_target:addBoxZone({
        coords = vec3(215.76, -810.12, 30.73),
        size = vec3(2.0, 2.0, 2.0),
        rotation = 0.0,
        debug = true,
        options = { { name = 'qu_ox_target_interactions_use', icon = 'fa-solid fa-hand', label = 'Use target', canInteract = function(entity, distance) return distance <= 2.0 end, onSelect = function() print('[qu_ox_target_interactions] selected') end } }
    })
end)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_ox_target_interactions
    ```

    Save, then run:

    ```text
    restart qu_ox_target_interactions
    ```

    Run this test:

    ```text
    Select Use target at the debug box
    ```

    <ExpectedOutput>
      F8: [qu_ox_target_interactions] selected
    </ExpectedOutput>
  </Step>
</StepGroup>
