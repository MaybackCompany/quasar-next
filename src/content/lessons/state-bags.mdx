  Half the time you reach for `TriggerClientEvent`, what you actually want is a **state bag**: a key/value blob attached to an entity, a player, or the whole server, automatically replicated across the network. Set the value once on the server; every client sees it. Hook a handler; you get notified when it changes.

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_state_bags
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_state_bags/
  fxmanifest.lua
  server.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

client_script 'client.lua'
server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
RegisterCommand('setmood', function(src, args)
    local mood = args[1] or 'calm'
    Player(src).state:set('qu_state_bags_mood', mood, true)
    print('[qu_state_bags] set mood ' .. mood)
end, false)
    ```

    Open `client.lua` and paste this:

    ```lua
AddStateBagChangeHandler('qu_state_bags_mood', nil, function(bagName, key, value)
    local player = GetPlayerFromStateBagName(bagName)
    if player == PlayerId() then print('[qu_state_bags] mood ' .. tostring(value)) end
end)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_state_bags
    ```

    Save, then run:

    ```text
    restart qu_state_bags
    ```

    Run this test:

    ```text
    /setmood focused
    ```

    <ExpectedOutput>
      Server: [qu_state_bags] set mood focused
F8: [qu_state_bags] mood focused
    </ExpectedOutput>
  </Step>
</StepGroup>
