<Callout variant="danger" title="A webhook URL is a password">
  Anyone with the webhook URL can post to that channel. Never put it in client code, never commit it, never paste it in public. Store it in a convar and read it server-side only. If it leaks, delete the webhook in Discord and make a new one.
</Callout>

## Log safely

<StepGroup>
  <Step num={1} title="Send only from the server" outcome="The webhook URL never reaches a client.">
    All webhook calls happen in server scripts, with the URL read from a convar. A client never sees the URL and never triggers the send directly without a server check.

    ```lua
    local hook = GetConvar('logs_webhook', '')
    ```
  </Step>

  <Step num={2} title="Decide what NOT to log" outcome="No private or sensitive data goes to a channel.">
    Log events, not secrets. Never send license keys, passwords, full IPs, tokens, or anything that identifies a real person beyond what you actually need. Redact or hash identifiers where you can.
  </Step>

  <Step num={3} title="Rate-limit your sends" outcome="Discord does not throttle or block your webhook.">
    Discord rate-limits webhooks. Batch or throttle high-frequency events instead of firing one request per tick. A log that spams itself stops logging when Discord cuts it off.
  </Step>
  <Step num={4} title="Separate channels by severity" outcome="Important logs are not buried in noise.">
    Send errors and security events to a private staff channel, routine activity to another. Mixing a critical ban alert into a firehose of chat logs means nobody sees it.
  </Step>
</StepGroup>

## Common mistakes

  items={[
    { symptom: "Webhook URL leaked", fix: "Delete the webhook in Discord, create a new one, and move the URL into a server convar read server-side only." },
