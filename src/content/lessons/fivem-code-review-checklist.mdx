<Callout variant="fivem" title="Review server trust first">
  The first question on every FiveM review is the same: does the client decide anything that should be the server's call? Money, items, jobs, admin, prices. If yes, that is the bug, no matter how clean the rest looks.
</Callout>

## The checklist

<Cheatsheet
  title="Security"
  rows={[
    { cmd: "Server authority", desc: "Does the server validate every money/item/job/admin action, or trust a client event?" },
    { cmd: "Event safety", desc: "Are net events validated for source and inputs? No raw client value used as truth." },
    { cmd: "SQL", desc: "Parameterized queries via oxmysql only. No string-concatenated SQL." },
    { cmd: "Secrets", desc: "No keys, passwords, or webhooks in the code. Read from convars." },
  ]}
/>

<Cheatsheet
  title="Performance"
  rows={[
    { cmd: "Threads and Wait", desc: "No tight loops without a Wait. Idle loops use long waits, not Wait(0)." },
    { cmd: "resmon budget", desc: "Does the resource stay well under budget at idle and under load?" },
    { cmd: "Cleanup", desc: "Every spawned entity has a delete path on stop, drop, and timeout." },
    { cmd: "Distance work", desc: "Heavy per-frame work is gated by distance or state, not always running." },
  ]}
/>

<Cheatsheet
  title="Correctness"
  rows={[
    { cmd: "Config", desc: "Tunables live in a config, not hardcoded magic numbers." },
    { cmd: "Errors", desc: "Failures are handled and logged, not swallowed silently." },
    { cmd: "Docs", desc: "A README explains install, dependencies, and config." },
    { cmd: "Versioning", desc: "Version bumped and changelog updated for the change." },
  ]}
/>

## Common mistakes

<FailureList
  items={[
    { symptom: "Clean code that still gets exploited", fix: "You reviewed style, not trust. Check server authority on every value-bearing action first." },
    { symptom: "Approved a resource that tanks resmon", fix: "Add the performance section to every review: threads, budget, cleanup, distance gating." },
    { symptom: "Merged code with hardcoded secrets", fix: "Block it. Secrets come from convars; the review fails if a key is in the source." },
    { symptom: "No README, buyers cannot install", fix: "Docs are part of the review. No install/dependency/config doc, no merge." },
  ]}
/>
