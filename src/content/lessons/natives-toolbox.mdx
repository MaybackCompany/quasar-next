## Find a native

A **native** is a built-in game function FiveM gives you. There are thousands. Nobody memorizes them; everyone looks them up. Keep these four tabs open while you script. The first two are the native references, the third is the docs hub, the last is the Lua manual for the language around them.

<TechLinks ids={["natives", "cfxnatives", "fivem-docs", "lua"]} />

When you open a native, read three things before you use it:

- **The side badge.** Each native is marked **CLIENT** or **SERVER**. A client native like `GetEntityCoords` will not run in a `server_script`, and the reverse is also true. Match the native's side to the file it lives in.
- **The arguments.** The signature shows what to pass in and in what order, for example `GetEntityCoords(entity)`. Pass the wrong type and you get `nil` or a crash.
- **The return value.** Some natives hand you something back (coords, health, a handle); some just do a thing and return nothing.

<Checkpoint question="A native is marked CLIENT. Can you call it from your server.lua?">
  No. Client natives only run in client scripts (loaded with `client_script`), and server natives only run in server scripts. The side badge in the docs tells you which file the call belongs in. Calling across the line is the single most common beginner native error.
</Checkpoint>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_natives_toolbox
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_natives_toolbox/
  fxmanifest.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

client_script 'client.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `client.lua` and paste this:

    ```lua
RegisterCommand('whereami', function()
    local ped = PlayerPedId()
    local coords = GetEntityCoords(ped)
    print(('[qu_natives_toolbox] %.2f %.2f %.2f health %d'):format(coords.x, coords.y, coords.z, GetEntityHealth(ped)))
end, false)

RegisterCommand('stepforward', function()
    local ped = PlayerPedId()
    local coords = GetEntityCoords(ped)
    SetEntityCoords(ped, coords.x + 2.0, coords.y, coords.z, false, false, false, false)
    print('[qu_natives_toolbox] moved')
end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_natives_toolbox
    ```

    Save, then run:

    ```text
    restart qu_natives_toolbox
    ```

    Run this test:

    ```text
    /whereami then /stepforward
    ```

    <ExpectedOutput>
      [qu_natives_toolbox] coords and health line
[qu_natives_toolbox] moved
    </ExpectedOutput>
  </Step>
</StepGroup>
