## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_variables_numbers_strings
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_variables_numbers_strings/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
local serverName = 'Quasar Practice'
local slots = 48
local version = 1.0
print('[qu_variables_numbers_strings] ' .. serverName .. ' slots=' .. slots .. ' version=' .. version)
print('[qu_variables_numbers_strings] slots type ' .. type(slots))
print('[qu_variables_numbers_strings] name type ' .. type(serverName))
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_variables_numbers_strings
    ```

    Save, then run:

    ```text
    restart qu_variables_numbers_strings
    ```

    Run this test:

    ```text
    restart qu_variables_numbers_strings
    ```

    <ExpectedOutput>
      [qu_variables_numbers_strings] Quasar Practice slots=48 version=1.0
[qu_variables_numbers_strings] slots type number
    </ExpectedOutput>
  </Step>
</StepGroup>
