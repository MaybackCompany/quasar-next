  You don't need XAMPP, a framework, or a database for your first server. You need two things on disk: the server binaries (FXServer) and the base server data (the default resources). Then a license key, a short server.cfg, and one command to start it. Everything else (QBCore, ESX, more resources, SQL) comes after you confirm the base works.

## Build it

<StepGroup>
  <Step num={1} title="Download the server binaries" outcome="FXServer.exe is on your computer.">
    Go to https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/

    Download the **latest** build (not the green "recommended" one). Open the `.7z` with 7-Zip or WinRAR and extract it to:

    ```text
C:\FXServer\server\
    ```

    Expected contents: `FXServer.exe`, `citizen/` folder, `.dll` files.

    <ExpectedOutput>
      `C:\FXServer\server\FXServer.exe` exists.
    </ExpectedOutput>
  </Step>

  <Step num={2} title="Download the base server data" outcome="The default resources are on your computer.">
    FXServer is just the engine. The default resources (spawn, chat, base gamemode) live in a separate repo. Open a command prompt (Win + R, type `cmd`, Enter) and run:

    ```bash
cd /d C:\FXServer
git clone https://github.com/citizenfx/cfx-server-data.git server-data
    ```

    This creates `C:\FXServer\server-data\`, a sibling of `server\`. Your `server.cfg` and resources live here.

    <ExpectedOutput>
      `C:\FXServer\server-data\resources\` exists and contains folders like `[gamemodes]` and `[system]`.
    </ExpectedOutput>
  </Step>

  <Step num={3} title="Get your license key" outcome="You have the one mandatory key.">
    Go to https://portal.cfx.re/ and register a free license key. Copy it.

    The key looks like: `cfxk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

    This is required even for a local server. Without it, the server won't start.
  </Step>

  <Step num={4} title="Write a minimal server.cfg" outcome="The server has endpoints, base resources, and a license.">
    Create `server.cfg` inside `C:\FXServer\server-data\`:

    ```bash server.cfg
# How players reach your server
endpoint_add_tcp "0.0.0.0:30120"
endpoint_add_udp "0.0.0.0:30120"

# Start the base resources (these came with server-data)
ensure mapmanager
ensure chat
ensure spawnmanager
ensure sessionmanager
ensure basic-gamemode
ensure hardcap

# Enable OneSync (required for modern servers)
set onesync on

# What players see
sv_hostname "My First FiveM Server"
sv_maxclients 32

# Your license key from portal.cfx.re
sv_licenseKey "cfxk_your_key_here"
    ```

    The `ensure` lines start the default resources that ship in `server-data`. Without them, a player who connects loads into nothing. You don't need tags, Discord links, or extra resources yet.
  </Step>

  <Step num={5} title="Start the server" outcome="The server console shows it's running.">
    Open a command prompt, go to your `server-data` folder, and point FXServer at your config:

    ```bash
cd /d C:\FXServer\server-data
C:\FXServer\server\FXServer.exe +exec server.cfg
    ```

    Run it from `server-data` (that is where your resources are), and the `+exec server.cfg` part is what loads your config. Skip either one and the server starts with no resources, or won't accept connections.

    <ExpectedOutput>
      The console prints resource start lines and then `Server started`. No `no license key was specified` or `Failed to start resource` errors.
    </ExpectedOutput>
  </Step>

  <Step num={6} title="Join your server" outcome="You're standing in Los Santos on your own server.">
    Open FiveM. Press F8 to open the console. Type:

    ```text
connect 127.0.0.1:30120
    ```

    Or if you want friends on your local network to join:

    ```text
connect 192.168.x.x:30120
    ```

    (Replace with your PC's local IP - find it with `ipconfig` in a command prompt.)

    <ExpectedOutput>
      You load into Los Santos. The server name appears in the bottom-right corner. You did it.
    </ExpectedOutput>
  </Step>
</StepGroup>

## Common failures

| Exact error | Cause | Fix |
|---|---|---|
| `FXServer.exe` window opens then closes instantly | Missing VC++ Redistributable | Download and install VC_redist.x64.exe from Microsoft |
| `no license key was specified` | You ran without `+exec server.cfg`, or `sv_licenseKey` is missing/wrong | Start with `+exec server.cfg`, and check the key in your config |
| `Failed to start resource` / no resources found | You started FXServer from the wrong folder | `cd /d C:\FXServer\server-data` first, then run FXServer with `+exec server.cfg` |
| `connect` times out or says connection refused | Server not started with `+exec`, or still loading | Add `+exec server.cfg`, wait for `Server started`, then try again |
