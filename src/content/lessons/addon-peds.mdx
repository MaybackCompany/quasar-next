## Build it

<StepGroup>
  <Step num={1} title="Prepare the resource" outcome="The ped files are in the right place.">
    ```text
resources/my_ped/
  fxmanifest.lua
  stream/
    ped_model.yft
    ped_model.ytd
    ```

    Unlike vehicles, peds usually only need two files: the model (`.yft`) and textures (`.ytd`). No metadata XML files required unless you're replacing a GTA ped or adding special properties.
  </Step>

  <Step num={2} title="Write fxmanifest and spawn" outcome="The ped appears in-game.">
    ```lua fxmanifest.lua
fx_version 'cerulean'
game 'gta5'

files {
    'stream/*.yft',
    'stream/*.ytd',
}
    ```

    ```lua client.lua
spawnedPed = nil  -- shared so step 3 can task the same ped

RegisterCommand('spawnmyped', function()
    local model = 'ped_model'  -- must match the .yft filename (without extension)
    RequestModel(model)
    while not HasModelLoaded(model) do Wait(0) end

    local coords = GetEntityCoords(PlayerPedId())
    spawnedPed = CreatePed(4, model, coords.x + 2, coords.y, coords.z, 0.0, true, false)

    -- Make the ped non-hostile: put it in a group that respects the player.
    -- This is what controls aggression - NOT SetPedAsNoLongerNeeded.
    AddRelationshipGroup('MY_PEDS')
    SetPedRelationshipGroupHash(spawnedPed, GetHashKey('MY_PEDS'))
    SetRelationshipBetweenGroups(1, GetHashKey('MY_PEDS'), GetHashKey('PLAYER'))  -- 1 = respect
    SetRelationshipBetweenGroups(1, GetHashKey('PLAYER'), GetHashKey('MY_PEDS'))

    -- We are keeping this ped, so we only release the MODEL, never the ped.
    SetModelAsNoLongerNeeded(model)
end, false)
    ```

    `CreatePed(pedType, model, x, y, z, heading, isNetwork, bScriptHostPed)`:
    - pedType `4` = civilian (peaceful by default)
    - pedType `6` = cop (will attack criminals)
    - isNetwork `true` = other players can see this ped

    <Callout variant="gotcha" title="SetPedAsNoLongerNeeded does not stop attacks">
      Do not call `SetPedAsNoLongerNeeded` on a ped you intend to keep. It does not control aggression - it releases the ped to the engine, so the game can stream it out or despawn it. To control hostility, set a relationship group (as above) or use a peaceful pedType. Only release the model with `SetModelAsNoLongerNeeded` once the ped exists.
    </Callout>

    <ExpectedOutput>
      A custom ped spawns in front of you. It stands idle and doesn't attack.
    </ExpectedOutput>
  </Step>

  <Step num={3} title="Make the ped do something" outcome="The ped has behavior — animation, scenario, or combat.">
    ```lua client.lua
-- spawnedPed is the ped created by the /spawnmyped command in step 2.
local ped = spawnedPed
if not ped or not DoesEntityExist(ped) then return end

-- Make the ped play a scenario (stand at a counter, lean on wall, etc.)
TaskStartScenarioInPlace(ped, 'WORLD_HUMAN_STAND_IMPATIENT', 0, true)

-- Give the ped a weapon and make them guard
GiveWeaponToPed(ped, GetHashKey('WEAPON_PISTOL'), 999, false, true)
TaskGuardCurrentPosition(ped, 10.0, 10.0, 1)

-- Make the ped a shopkeeper who stays put
FreezeEntityPosition(ped, true)
SetBlockingOfNonTemporaryEvents(ped, true)
    ```

    Full scenario list: https://github.com/DurtyFree/gta-v-data-dumps

    <ExpectedOutput>
      The ped plays an animation, guards an area, or stands in place depending on the task.
    </ExpectedOutput>
  </Step>

## Common failures

| Exact error | Cause | Fix |
|---|---|---|
| Ped invisible or doesn't spawn | Model name doesn't match .yft file | Use exact filename without extension: `ped_model.yft` → `'ped_model'` |
| `CreatePed` returns 0 (invalid ped) | Model failed to load or doesn't exist | Check `HasModelLoaded(model)` returns true before CreatePed |
| Ped attacks player immediately | pedType is a combat type (cop, army) | Use pedType `4` (civilian) or set a relationship group that respects the player |
| Ped disappears when you look away | `SetPedAsNoLongerNeeded` was called on a ped you wanted to keep | Never release a kept ped - only call `SetModelAsNoLongerNeeded` on the model |
