  In Lua, almost every collection of data is a **table**. Player lists, config files, inventories, job data, and many other things all use tables. This lesson teaches you how to read them, write them, loop through them, and avoid the classic beginner mistake: forgetting that tables are references, not copies.

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_tables
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_tables/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
local roster = { { name = 'Maya', level = 5 }, { name = 'Rami', level = 12 } }

RegisterCommand('roster', function()
    table.insert(roster, { name = 'New Player', level = 1 })
    for index, player in ipairs(roster) do
        print(('[qu_tables] %d %s level %d'):format(index, player.name, player.level))
    end
end, true)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_tables
    ```

    Save, then run:

    ```text
    restart qu_tables
    ```

    Run this test:

    ```text
    roster
    ```

    <ExpectedOutput>
      [qu_tables] 1 Maya level 5
[qu_tables] 2 Rami level 12
[qu_tables] 3 New Player level 1
    </ExpectedOutput>
  </Step>
</StepGroup>
