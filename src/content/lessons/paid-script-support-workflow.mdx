<Callout variant="fivem" title="Scope is kindness">
  A clear support boundary protects both sides. State what is covered (bugs in your script, install help) and what is not (custom edits, conflicts with other paid scripts, framework-wide debugging). Buyers respect a clear line far more than a vague promise you cannot keep.
</Callout>

## The workflow

<StepGroup>
  <Step num={1} title="Use a fixed intake template" outcome="Every report arrives with the info you need to act.">
    Require buyers to fill the same fields. No template, no triage.

    ```text
    Script version:
    Framework + version (ESX/QBCore/Qbox):
    Artifact build:
    What you did:
    What you expected:
    What happened (exact console error):
    ```
  </Step>

  <Step num={2} title="Reproduce before you promise a fix" outcome="You know whether it is your bug or their setup.">
    Reproduce on a clean test server with only your script plus its dependencies. If it only breaks with a third paid script loaded, that is a conflict (out of scope), not your bug.
  </Step>

  <Step num={3} title="Triage by cause" outcome="Each ticket gets the right response, fast.">
    Sort tickets: **your bug** (fix and release), **install/config** (point to docs), **conflict** (out of scope, suggest direction), **feature request** (backlog, not support). Most tickets are config, not bugs.
  </Step>

  <Step num={4} title="Fix, version, and release" outcome="The fix ships with a version bump and a note.">
    Fix on a branch, bump the version (semver), write a one-line changelog, and release. Reply to the ticket with the version that contains the fix so the buyer knows exactly what to update to.
  </Step>
</StepGroup>

## Common mistakes

<FailureList
  items={[
    { symptom: "Tickets are vague and unfixable", fix: "Enforce the intake template. No version, framework, and error means no triage yet." },
    { symptom: "Drowning in 'it does not work'", fix: "Triage by cause. Config and conflicts are not bugs; point to docs or mark out of scope." },
    { symptom: "Buyers expect free custom edits", fix: "Publish the support scope up front. Customization is paid work, not support." },
    { symptom: "Nobody knows which version fixed it", fix: "Bump semver and changelog every release; reply with the fixed version number." },
  ]}
/>
