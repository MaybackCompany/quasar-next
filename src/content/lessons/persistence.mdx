>
  Reading from MySQL on every action is slow. Writing on every action floods your DB and locks tables. The fix is the cache + write-back pattern: keep player state in RAM, mark it dirty when it changes, flush on a timer and on disconnect. This is how every well-built FiveM economy survives 64-player nights.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_persistence
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_persistence/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script '@oxmysql/lib/MySQL.lua'
server_script 'server.lua'

dependency 'oxmysql'
    ```

    Run this SQL before starting the resource:

    ```sql
CREATE TABLE IF NOT EXISTS qu_persistence_rows (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(80) NOT NULL,
  amount INT NOT NULL DEFAULT 0
);
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
local cache = { cash = 0, dirty = false }
RegisterCommand('cashadd', function(_, args) cache.cash = cache.cash + (tonumber(args[1]) or 1) cache.dirty = true print('[qu_persistence] cached ' .. cache.cash) end, true)
CreateThread(function() while true do Wait(30000) if cache.dirty then MySQL.insert.await('INSERT INTO qu_persistence_rows (name, amount) VALUES (?, ?) ON DUPLICATE KEY UPDATE amount = ?', { 'wallet', cache.cash, cache.cash }) cache.dirty = false print('[qu_persistence] flushed') end end end)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_persistence
    ```

    Save, then run:

    ```text
    restart qu_persistence
    ```

    Run this test:

    ```text
    cashadd 25, then wait 30 seconds
    ```

    <ExpectedOutput>
      [qu_persistence] cached 25
[qu_persistence] flushed
    </ExpectedOutput>
  </Step>
</StepGroup>
