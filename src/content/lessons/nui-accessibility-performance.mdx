<Callout variant="fivem" title="The focus trap that ruins UIs">
  `SetNuiFocus` controls whether the cursor and keyboard go to your UI or the game. Set focus on when the UI opens and OFF the instant it closes. A UI that grabs focus and never releases it leaves the player frozen, unable to move, the single most common NUI complaint.
</Callout>

## Make it good, not just visible

<StepGroup>
  <Step num={1} title="Release focus on every close path" outcome="The player is never stuck with a hidden UI holding focus.">
    Call `SetNuiFocus(false, false)` on every way the UI can close: button, escape key, and any server-driven close. Test closing it five different ways.
  </Step>

  <Step num={2} title="Scale with viewport units, not fixed pixels" outcome="The UI looks right at 1080p, 1440p, and ultrawide.">
    Size with relative units (`vw`, `vh`, `rem`, `%`) and `clamp()`, not hardcoded pixel widths. A panel fixed at 600px is tiny on 1440p and huge on 1080p.
  </Step>

  <Step num={3} title="Stop rendering when hidden" outcome="The UI costs nothing when closed.">
    Hide the page (`display: none`) when closed and pause any animation loops/timers. A NUI that keeps animating off-screen burns frames for no reason.
  </Step>

  <Step num={4} title="Make it readable" outcome="Players can actually use it under pressure.">
    Sufficient contrast, legible font sizes, clear focus states on buttons, and not relying on color alone for meaning. People read your UI mid-firefight, not in a calm museum.
  </Step>
</StepGroup>
