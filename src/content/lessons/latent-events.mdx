>
  A regular `TriggerServerEvent` blocks the network channel until the whole payload arrives. For small messages that is fine. For big tables, image buffers, or character export blobs, blocking the channel can time the player out. Latent events stream the payload at a controlled rate so the channel stays open.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_latent_events
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_latent_events/
  fxmanifest.lua
  server.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

client_script 'client.lua'
server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
RegisterNetEvent('qu_latent_events:upload', function(payload)
    print('[qu_latent_events] received ' .. #payload .. ' bytes')
    TriggerClientEvent('qu_latent_events:done', source, #payload)
end)
    ```

    Open `client.lua` and paste this:

    ```lua
RegisterCommand('latenttest', function()
    TriggerLatentServerEvent('qu_latent_events:upload', 25000, string.rep('A', 20000))
    print('[qu_latent_events] upload started')
end, false)

RegisterNetEvent('qu_latent_events:done', function(size)
    print('[qu_latent_events] server received ' .. size)
end)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_latent_events
    ```

    Save, then run:

    ```text
    restart qu_latent_events
    ```

    Run this test:

    ```text
    /latenttest
    ```

    <ExpectedOutput>
      F8: [qu_latent_events] upload started
Server: [qu_latent_events] received 20000 bytes
    </ExpectedOutput>
  </Step>
</StepGroup>
