<Callout variant="fivem" title="Git vs GitHub">
  **Git** is the version control tool on your computer. **GitHub** is the website that hosts your code online and lets others see it. You use Git commands in the terminal; GitHub is where the code lives on the internet.
</Callout>

## Build it

<StepGroup>
  <Step num={1} title="Create a repository on GitHub" outcome="An empty repo is ready to receive your code.">
    Go to https://github.com/new. Name it `my-fivem-script`. Leave it public or private. Do NOT check "Add a README" - you already have files to push. Click **Create repository**.

    GitHub shows you a page with commands. Copy the URL: `https://github.com/YOURNAME/my-fivem-script.git`
  </Step>

  <Step num={2} title="Initialize Git in your resource folder" outcome="Git is tracking your files locally.">
    Open a terminal in your resource folder:

    ```bash
cd resources/my-script
git init
git add .
git commit -m "First commit: my FiveM resource"
    ```

    - `git init` - creates a hidden `.git` folder that tracks everything
    - `git add .` - stages every file for commit
    - `git commit -m "..."` - saves a snapshot with a message

    <ExpectedOutput>
      Terminal prints something like `[main c1a2b3] First commit: my FiveM resource` with a list of files.
    </ExpectedOutput>
  </Step>

  <Step num={3} title="Push to GitHub" outcome="Your code is online, visible on github.com.">
    ```bash
git remote add origin https://github.com/YOURNAME/my-fivem-script.git
git branch -M main
git push -u origin main
    ```

    - `remote add` - tells Git where GitHub is
    - `branch -M main` - names your branch "main" (modern default)
    - `push` - sends your commits to GitHub

    Refresh your GitHub repo page. Your files are there.

    <ExpectedOutput>
      github.com/YOURNAME/my-fivem-script shows all your resource files.
    </ExpectedOutput>
  </Step>

  <Step num={4} title="The daily workflow: edit, commit, push" outcome="You know the three commands you'll use every day.">
    Every time you make changes:

    ```bash
git add .                          # stage changes
git commit -m "Fixed the spawn bug"  # save a snapshot
git push                            # send to GitHub
    ```

    That's it. This loop is 90% of what you'll ever do with Git.

    To get your code on another computer:
    ```bash
git clone https://github.com/YOURNAME/my-fivem-script.git
    ```
  </Step>
</StepGroup>

## Common failures

| Exact error | Cause | Fix |
|---|---|---|
| `fatal: not a git repository` | You're not inside the Git folder | `cd` into the correct directory first |
| `Permission denied (publickey)` | SSH key not set up | Use HTTPS URL instead: `https://github.com/...` |
| `Everything up-to-date` after push but files not on GitHub | You forgot to `git add` and `git commit` | Always `git add .` then `git commit` before `git push` |
| `failed to push - remote contains work` | Someone else pushed to the repo | `git pull` first, resolve conflicts, then push |
