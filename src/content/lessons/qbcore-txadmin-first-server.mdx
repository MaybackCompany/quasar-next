## Build it

<StepGroup>
  <Step num={1} title="Install MariaDB" outcome="MariaDB runs as a Windows service on 127.0.0.1:3306.">
    Download the Windows x86_64 MSI from https://mariadb.org/download/ and run it. Accept the defaults, but set a strong **root password** during install and save it in a password manager. Keep port **3306** and install it as a **Windows service**.

    Confirm the service is running. Open PowerShell and run:

    ```powershell
Get-Service MariaDB*
    ```

    You want a row whose Status reads **Running**.

    <ExpectedOutput>
      Get-Service MariaDB* shows a MariaDB row with Status Running. MariaDB is listening on port 3306.
    </ExpectedOutput>
  </Step>

  <Step num={2} title="Download and extract FXServer" outcome="FXServer.exe is in your server folder.">
    Go to the official Windows artifacts page:

    ```text
https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/
    ```

    Download the **latest** build (the newest one at the top), not the green "Recommended" row. Grab `server.7z`, open it with 7-Zip or WinRAR, and extract everything to:

    ```text
C:\FXServer\
    ```

    Inside you should see `FXServer.exe`, the `citizen/` folder, and other files. Only download from this Cfx.re page. Do not use Discord mirrors or random upload sites.

    <ExpectedOutput>
      C:\FXServer\FXServer.exe exists.
    </ExpectedOutput>
  </Step>

  <Step num={3} title="Start FXServer and link your account" outcome="txAdmin opens, is linked to Cfx.re, and has a backup password.">
    Double-click `FXServer.exe`. A black console window opens and prints a **4-digit PIN**. Keep this window open. Your browser opens txAdmin at `http://localhost:40120`. If it does not, go there manually.

    Do these in order:

    1. Enter the 4-digit PIN from the console.
    2. Click **Link account**, sign in to Cfx.re, and click **Continue** to authorize.
    3. Set a **backup password** for the dashboard.
    4. Agree to the license, then click **Register**.

    The PIN is generated fresh each first boot and the example digits in any guide are not yours. If you restarted before finishing, close and reopen `FXServer.exe` for a new PIN.

    <ExpectedOutput>
      [txAdmin] Webserver is now available at http://localhost:40120/
      [txAdmin] Use the PIN below to register the master account: 0427
    </ExpectedOutput>
  </Step>

  <Step num={4} title="Deploy the QBCore recipe" outcome="QBCore is downloaded, server.cfg is written, and the server starts.">
    On the setup page, name your server and click **Next**. For **Deployment Type** choose **Popular Recipes**, then pick the **QBCore** recipe. Leave the data location as-is, click **Save**, then **Go to Recipe Deployer**.

    In the deployer:

    1. **Review Recipe**: click **Next**.
    2. **Input Parameters**: click the **Portal** link, sign in to Cfx.re, click **Create a key**, **GENERATE**, then **copy** the key.
    3. Back in txAdmin, paste the key and click **Run Recipe**.

    The recipe downloads QBCore, creates the database tables, and writes `server.cfg`. When it finishes, click **Save and Run Server**. You do not edit any file by hand here.

    <ExpectedOutput>
      [txAdmin] Running recipe... Resources downloaded. Writing server.cfg...
      [txAdmin] Server started. Status: ONLINE
    </ExpectedOutput>
  </Step>

  <Step num={5} title="Connect HeidiSQL to your database" outcome="You can browse the QBCore tables.">
    Download HeidiSQL from https://www.heidisql.com. Open it, click **New** session, and fill in:

    - Host: `127.0.0.1`
    - User: `root`
    - Password: the root password you set when installing MariaDB
    - Port: `3306`

    Click **Open**. You should see the QBCore database with tables like `players`.

    <ExpectedOutput>
      HeidiSQL shows the QBCore database. You can browse the players table and other QBCore tables.
    </ExpectedOutput>
  </Step>

  <Step num={6} title="Join your own server" outcome="Your client connects and txAdmin shows one player online.">
    Open the FiveM client (the game launcher, not the black FXServer window). With FiveM in focus, press **F8** to open the in-game console, then run:

    ```bash
    connect 127.0.0.1:30120
    ```

    Once you spawn in, go back to txAdmin. The Players panel shows 1 online and the Live Console logs your connection.

    <ExpectedOutput>
      You spawn into the server. txAdmin Players panel shows 1 player online.
    </ExpectedOutput>
  </Step>
</StepGroup>

## How it works

The pieces fit together in one line: FXServer is the server software, txAdmin is the dashboard bundled inside it, the recipe is a one-click installer that downloads QBCore and writes `server.cfg`, MariaDB stores player data, and HeidiSQL is just a window into that database.

Two things are easy to confuse. The **port 40120** is the txAdmin dashboard in your browser. The **port 30120** is the game server your FiveM client connects to. They are different ports for different jobs.

Your **license key** is created during the recipe step, not before, and txAdmin pastes it into `server.cfg` for you as `sv_licenseKey`. You never hand-edit that file to get started.

## If something went wrong

  items={[
    { symptom: "The browser never opens or localhost:40120 is refused", fix: "FXServer.exe is not running or its console was closed. Reopen FXServer.exe, keep the black window open, then visit http://localhost:40120 manually." },

## Common failures

| What you see | What it means | How to fix it |
|---|---|---|
| MariaDB service won't start | Port 3306 already in use | Stop the other MySQL instance, or change MariaDB's port in `my.ini` and update HeidiSQL to match |
| `FXServer.exe` crashes immediately | Missing Visual C++ Redistributable | Install VC_redist.x64.exe from Microsoft |
| txAdmin recipe fails to download | Firewall blocking or GitHub rate limit | Wait a few minutes and retry. If persistent, download QBCore manually |
| `ensure` not found in HeidiSQL | HeidiSQL only runs SQL, not FiveM commands | Use the txAdmin Live Console for FiveM commands, HeidiSQL for SQL only |
