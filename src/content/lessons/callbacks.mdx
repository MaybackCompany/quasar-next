>
  Events are good when you just want to send a message and move on. But some situations need a reply. For example: "do I have enough money to buy this?" In that case the client asks a question, waits, and gets an answer back. That pattern is called a callback.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_callbacks
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_callbacks/
  fxmanifest.lua
  server.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

shared_script '@ox_lib/init.lua'
client_script 'client.lua'
server_script 'server.lua'

dependency 'ox_lib'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
lib.callback.register('qu_callbacks:getBalance', function(src)
    print('[qu_callbacks] server answered ' .. GetPlayerName(src))
    return 500
end)
    ```

    Open `client.lua` and paste this:

    ```lua
RegisterCommand('mybalance', function()
    local balance = lib.callback.await('qu_callbacks:getBalance', false)
    print('[qu_callbacks] my balance is ' .. balance)
end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_callbacks
    ```

    Save, then run:

    ```text
    restart qu_callbacks
    ```

    Run this test:

    ```text
    /mybalance
    ```

    <ExpectedOutput>
      F8: [qu_callbacks] my balance is 500
    </ExpectedOutput>
  </Step>
</StepGroup>
