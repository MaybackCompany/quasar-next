  A garage is a valet with a clipboard. The clipboard is your DB. The plate is the ticket. Owning a vehicle in FiveM means: a row in `owned_vehicles`, a unique plate, and a state column that says "is the car spawned right now or sitting in the garage". This lesson walks the full take/store flow.

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_vehicles_garages
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_vehicles_garages/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script '@oxmysql/lib/MySQL.lua'
server_script 'server.lua'

dependency 'oxmysql'
    ```

    Run this SQL before starting the resource:

    ```sql
CREATE TABLE IF NOT EXISTS qu_vehicles_garages_vehicles (
  plate VARCHAR(12) PRIMARY KEY,
  model VARCHAR(40) NOT NULL,
  state VARCHAR(20) NOT NULL DEFAULT 'garage'
);
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
RegisterCommand('takecar', function()
    local vehicle = CreateVehicleServerSetter('sultan', 'automobile', 215.0, -810.0, 30.0, 90.0)
    SetVehicleNumberPlateText(vehicle, 'QUCAR1')
    MySQL.insert.await('INSERT INTO qu_vehicles_garages_vehicles (plate, model, state) VALUES (?, ?, ?) ON DUPLICATE KEY UPDATE state = ?', { 'QUCAR1', 'sultan', 'out', 'out' })
    print('[qu_vehicles_garages] spawned QUCAR1 ' .. vehicle)
end, false)

RegisterCommand('storecar', function()
    MySQL.update.await('UPDATE qu_vehicles_garages_vehicles SET state = ? WHERE plate = ?', { 'garage', 'QUCAR1' })
    print('[qu_vehicles_garages] stored QUCAR1')
end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_vehicles_garages
    ```

    Save, then run:

    ```text
    restart qu_vehicles_garages
    ```

    Run this test:

    ```text
    /takecar then /storecar
    ```

    <ExpectedOutput>
      [qu_vehicles_garages] spawned QUCAR1
[qu_vehicles_garages] stored QUCAR1
    </ExpectedOutput>
  </Step>
</StepGroup>
