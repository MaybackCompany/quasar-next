  Sometimes one handler needs to veto the rest. A spawn-protection script blocks the death-handler chain. An anti-cheat resource cancels `playerConnecting` for a banned UID before any other resource sees the join. `CancelEvent()` is the signal; `WasEventCanceled()` is how downstream handlers know.

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_cancelling_events
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_cancelling_events/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
AddEventHandler('qu_cancelling_events:check', function(name)
    print('[qu_cancelling_events] first handler checked ' .. name)
    if name == 'blocked' then
        CancelEvent()
        print('[qu_cancelling_events] event cancelled')
    end
end)

AddEventHandler('qu_cancelling_events:check', function(name)
    if WasEventCanceled() then
        print('[qu_cancelling_events] second handler saw cancel')
        return
    end
    print('[qu_cancelling_events] second handler allowed ' .. name)
end)

RegisterCommand('canceltest', function(_, args)
    TriggerEvent('qu_cancelling_events:check', args[1] or 'allowed')
end, true)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_cancelling_events
    ```

    Save, then run:

    ```text
    restart qu_cancelling_events
    ```

    Run this test:

    ```text
    canceltest blocked
    ```

    <ExpectedOutput>
      [qu_cancelling_events] event cancelled
[qu_cancelling_events] second handler saw cancel
    </ExpectedOutput>
  </Step>
</StepGroup>
