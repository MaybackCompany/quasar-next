<Callout variant="fivem" title="The #1 install mistake">
  Putting the resource folder in the wrong place. It must go in `resources/` (or `resources/[category]/`), not in `server-data/`, not in the FXServer root, not nested inside another resource's folder.
</Callout>

## The install checklist

<StepGroup>
  <Step num={1} title="Read the README first" outcome="You know what this resource needs before you touch a file.">
    Before extracting anything, open the zip and look for:
    - `README.md` or `README.txt` — installation instructions
    - `fxmanifest.lua` or `__resource.lua` — the manifest
    - A `config.lua` or `config.json` — configuration you must edit

    Read the README. Note any:
    - **Dependencies**: "requires ox_lib", "needs es_extended", etc.
    - **Database**: "import the SQL file"
    - **Configuration**: "edit config.lua and set your framework"
    - **Commands**: how to use the resource after install

    Skipping the README is the #2 install mistake.
  </Step>

  <Step num={2} title="Place the folder correctly" outcome="The resource is in the right location.">
    Extract the zip. The folder structure should look like:
    ```text
resources/
  your_script/
    fxmanifest.lua       <- must be at the root of the folder
    client.lua
    server.lua
    html/                <- if it has NUI
    config.lua           <- if it needs configuring
    ```

    Common mistake: extracting a zip that creates a folder-inside-a-folder:
    ```text
resources/your_script/your_script-v1.2/fxmanifest.lua  <- WRONG
    ```

    The `fxmanifest.lua` must be exactly one folder deep: `resources/your_script/fxmanifest.lua`.
  </Step>

  <Step num={3} title="Check dependencies and configure" outcome="Every requirement is met.">
    Open `fxmanifest.lua` and check `dependencies`:
    ```lua fxmanifest.lua
dependencies {
    'ox_lib',
    'oxmysql',
    'es_extended',   -- if ESX
    'qb-core',       -- if QBCore
}
    ```

    Make sure every dependency is installed and started BEFORE this resource in `server.cfg`. If the resource needs a framework (ESX/QBCore/Qbox), make sure you have one installed.

    Open `config.lua` or `config.json` and set:
    - Framework type (ESX, QBCore, Qbox, standalone)
    - Any API keys or webhooks
    - Any feature toggles

    <ExpectedOutput>
      All dependencies are in your resources folder and started in server.cfg. Config is edited.
    </ExpectedOutput>
  </Step>

  <Step num={4} title="Add to server.cfg, restart, and test" outcome="The resource starts without errors.">
    Add to `server.cfg` AFTER all dependencies:
    ```bash server.cfg
ensure es_extended
ensure ox_lib
ensure oxmysql
ensure your_script        # after its dependencies
    ```

    In txAdmin Live Console or server console:
    ```text
ensure your_script
    ```

    Watch the console for errors. If you see red, read the error, find the file and line, fix it, and `restart your_script`.

    <ExpectedOutput>
      Console shows `[script:your_script]` entries, no red error lines. The resource's command or feature works.
    </ExpectedOutput>
  </Step>
</StepGroup>

## Common failures

| Exact error | Cause | Fix |
|---|---|---|
| `Couldn't find resource your_script` | Folder in wrong location or misspelled | Check `resources/your_script/fxmanifest.lua` exists |
| `dependency 'ox_lib' not found` | Dependency not installed | Download and install the missing dependency first |
| Resource starts but nothing happens | Config not set for your framework | Edit config.lua to match your framework (ESX/QBCore/standalone) |
| `attempt to index a nil value (global 'ESX')` | ESX not started before this resource | Move `ensure your_script` AFTER `ensure es_extended` in server.cfg |
