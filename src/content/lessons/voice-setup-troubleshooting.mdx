<Callout variant="fivem" title="How FiveM voice works">
  FiveM has a built-in voice system; a resource like `pma-voice` configures proximity, radio, and phone channels on top of it. You do not need a separate TeamSpeak/Mumble server for proximity voice in 2026; the in-game system handles it.
</Callout>

## Get it working

<StepGroup>
  <Step num={1} title="Install the voice resource and its dependencies" outcome="The voice resource starts with no missing-dependency errors.">
    Install your chosen voice resource (commonly `pma-voice`) and any dependency it lists. Add `ensure` lines in the right order and restart. Watch the console for missing dependencies.
  </Step>

  <Step num={2} title="Set proximity and channels in config" outcome="Talk, whisper, and shout ranges behave as intended.">
    Configure the proximity ranges, radio, and phone settings in the resource config. Keep defaults first; tune ranges only after it works.
  </Step>

  <Step num={3} title="Confirm mic permission and device" outcome="Your voice is captured and transmitted.">
    In FiveM settings, set the correct input device and push-to-talk key, and allow microphone access at the OS level. A muted OS mic looks exactly like a broken server.
  </Step>

  <Step num={4} title="Test with a second player" outcome="Two players hear each other by proximity.">
    Voice is hard to test alone. Bring a second client (or a friend), stand close, and confirm proximity falloff as you walk apart.
  </Step>
</StepGroup>

## Troubleshooting order

<FailureList
  items={[
    { symptom: "Nobody can hear anyone", fix: "Check the voice resource actually started (console), then confirm OS mic permission and the FiveM input device." },
    { symptom: "I hear others but they cannot hear me", fix: "Your mic input or push-to-talk is wrong. Set the correct device and key in FiveM settings and unmute the OS mic." },
    { symptom: "Voice cuts out or is robotic", fix: "Usually network/packet loss or server thread overload. Check ping and resmon; voice rides the same UDP path as the game." },
    { symptom: "Radio or phone channels do not work", fix: "Those are config/framework integration. Confirm the framework bridge and channel settings in the voice resource config." },
  ]}
/>
