>
  A real FiveM server runs lots of resources at the same time. They do not share variables directly, so they need a way to talk. That is what events are. An event is just a named message. One script fires it, another script listens for it. This lesson starts with the simplest version: same-side events using `TriggerEvent`.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_trigger_event
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_trigger_event/
  fxmanifest.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

client_script 'client.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `client.lua` and paste this:

    ```lua
AddEventHandler('qu_trigger_event:shout', function(message) print('[qu_trigger_event] handler one ' .. message) end)
AddEventHandler('qu_trigger_event:shout', function(message) print('[qu_trigger_event] handler two ' .. message) end)
RegisterCommand('shout', function(_, args) TriggerEvent('qu_trigger_event:shout', table.concat(args, ' ')) end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_trigger_event
    ```

    Save, then run:

    ```text
    restart qu_trigger_event
    ```

    Run this test:

    ```text
    /shout hello
    ```

    <ExpectedOutput>
      F8: [qu_trigger_event] handler one hello
F8: [qu_trigger_event] handler two hello
    </ExpectedOutput>
  </Step>
</StepGroup>
