  Lua does not have classes. It has tables and a one-line trick called `setmetatable` that lets you fake them well enough to ship. This lesson teaches the canonical class pattern, single-level inheritance, and, just as important, when to skip OOP entirely and just use a plain table.

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_oop_metatables
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_oop_metatables/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
local Vehicle = {}
Vehicle.__index = Vehicle

function Vehicle.new(model, plate)
    return setmetatable({ model = model, plate = plate }, Vehicle)
end

function Vehicle:label()
    return self.model .. ' [' .. self.plate .. ']'
end

RegisterCommand('cartest', function()
    local car = Vehicle.new('sultan', 'QU123')
    print('[qu_oop_metatables] ' .. car:label())
end, true)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_oop_metatables
    ```

    Save, then run:

    ```text
    restart qu_oop_metatables
    ```

    Run this test:

    ```text
    cartest
    ```

    <ExpectedOutput>
      [qu_oop_metatables] sultan [QU123]
    </ExpectedOutput>
  </Step>
</StepGroup>
