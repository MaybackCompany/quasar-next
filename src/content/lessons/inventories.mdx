>
  An inventory is a backpack with limited pockets and a scale. There are two dominant models in FiveM: slot-based (each item lives in a numbered pocket) and weight-based (only kg cap matters). Each has trade-offs. This lesson covers both, the stacking rules, item metadata, and how to integrate with `ox_inventory` without re-implementing it.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_inventories
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_inventories/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script 'server.lua'

dependency 'ox_inventory'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
RegisterCommand('burger', function(src)
    local ok = exports.ox_inventory:AddItem(src, 'burger', 5)
    if not ok then print('[qu_inventories] add failed') return end
    local count = exports.ox_inventory:Search(src, 'count', 'burger')
    print('[qu_inventories] burger count ' .. count)
end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_inventories
    ```

    Save, then run:

    ```text
    restart qu_inventories
    ```

    Run this test:

    ```text
    /burger
    ```

    <ExpectedOutput>
      [qu_inventories] burger count 5
    </ExpectedOutput>
  </Step>
</StepGroup>
