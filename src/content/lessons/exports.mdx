>
  Events fire and forget. Exports are *function calls*: synchronous, return-a-value, same-side. They are how every framework you have ever used wires resources together, `exports.ox_inventory:GetItemCount`, `exports.ESX:getPlayerFromId`. This lesson teaches you to expose them safely and consume them without surprises.
</LessonHero>

## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_exports
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_exports/
  fxmanifest.lua
  server.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

server_script 'server.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `server.lua` and paste this:

    ```lua
local balances = {}

function GetBalance(src)
    balances[src] = balances[src] or 0
    return balances[src]
end

function AddBalance(src, amount)
    balances[src] = GetBalance(src) + amount
    return balances[src]
end

exports('GetBalance', GetBalance)
exports('AddBalance', AddBalance)

RegisterCommand('exportmoney', function(src)
    print('[qu_exports] balance ' .. AddBalance(src, 50))
end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_exports
    ```

    Save, then run:

    ```text
    restart qu_exports
    ```

    Run this test:

    ```text
    /exportmoney
    ```

    <ExpectedOutput>
      [qu_exports] balance 50
    </ExpectedOutput>
  </Step>
</StepGroup>
