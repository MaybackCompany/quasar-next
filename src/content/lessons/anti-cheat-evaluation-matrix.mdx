  New owners think anti-cheat is a product you buy that makes cheating stop. It is not. The strongest protection is a server that never trusts the client. Anti-cheat is a layer on top of that, not a replacement for it. This lesson gives you a sober way to evaluate options so you spend money on the right thing.
</LessonHero>

<Callout variant="danger" title="No anti-cheat fixes a trusting server">
  If your scripts let the client set money, items, or admin power, no anti-cheat saves you. Fix server authority first. Anti-cheat catches things authority cannot, like injected menus and resource tampering, but it is the second line, never the first.
</Callout>

## How to evaluate an option

<Cheatsheet
  title="Score each option on these"
  rows={[
    { cmd: "Server-side detection", desc: "Does it validate on the server, or just trust client reports a cheater can fake?" },
    { cmd: "Active maintenance", desc: "Updated within the last weeks/months? Cheats evolve; a stale anti-cheat is dead weight." },
    { cmd: "Performance cost", desc: "What does it add to resmon? A heavy anti-cheat that tanks the server trades one problem for another." },
    { cmd: "Transparency", desc: "Can you read what it checks, or is it an opaque blob you must blindly trust with full access?" },
    { cmd: "Support and community", desc: "Real docs, real updates, real responses, not just a hype thread." },
    { cmd: "Bypass history", desc: "How fast are public bypasses patched? Ask in neutral communities, not the seller's channel." },
  ]}
/>

## The layers that actually stop cheaters

<Recap
  items={[
    "Server authority: the client may request, only the server grants, after a check.",
    "Rate limits: cap how often an action can happen so spam and automation are blocked.",
    "Integrity checks: detect tampered resources and unexpected events.",
    "Logging and review: catch patterns after the fact and ban with evidence.",
    "Anti-cheat product: a layer for injected menus and known cheat signatures, on top of the above."
  ]}
/>
