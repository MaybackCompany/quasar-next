## Build it

<StepGroup>
  <Step num={1} title="Make the resource folder" outcome="The server has one folder for this lesson.">
    Inside your server's `resources` folder, create this folder:

    ```text
    resources/qu_debugging_like_a_developer
    ```
  </Step>

  <Step num={2} title="Create the files" outcome="Every file named in the manifest exists.">
    Create this exact file layout:

    ```text
resources/qu_debugging_like_a_developer/
  fxmanifest.lua
  client.lua
    ```
  </Step>

  <Step num={3} title="Write fxmanifest.lua" outcome="FiveM knows which files to load.">
    Open `fxmanifest.lua` and paste this:

    ```lua
fx_version 'cerulean'
game 'gta5'

client_script 'client.lua'
    ```
  </Step>

  <Step num={4} title="Write the lesson code" outcome="The topic is now represented by runnable code.">
    Open `client.lua` and paste this:

    ```lua
local function log(message)
    print('[qu_debugging_like_a_developer] ' .. message)
end

local function risky()
    error('planned failure')
end

RegisterCommand('debugfail', function()
    local ok, result = pcall(risky)
    log('ok=' .. tostring(ok) .. ' result=' .. tostring(result))
    log(debug.traceback('traceback'))
end, false)
    ```
  </Step>

  <Step num={5} title="Start and test it" outcome="The expected proof appears in the correct console.">
    Open `server.cfg` and add this line:

    ```text
    ensure qu_debugging_like_a_developer
    ```

    Save, then run:

    ```text
    restart qu_debugging_like_a_developer
    ```

    Run this test:

    ```text
    /debugfail
    ```

    <ExpectedOutput>
      F8: [qu_debugging_like_a_developer] ok=false result=
    </ExpectedOutput>
  </Step>
</StepGroup>
