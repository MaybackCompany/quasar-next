  An **artifact** is the FXServer build you download. **txAdmin** is the browser dashboard that **ships inside** that build and opens by itself the first time you run it. A **recipe** is a one-click starter that downloads a framework and writes your first `server.cfg`. You do all of this with the files Cfx.re gives you. Nothing extra to install.

## Build it

<StepGroup>
  <Step num={1} title="Download an official artifact and record the build" outcome="server.7z is downloaded and the build number is written down.">
    Open the official Cfx.re Windows artifacts page and pick a build from the **Recommended** (green) channel near the top. Recommended is the stable line; do not grab the bleeding-edge "Latest" or "Optional" rows for a first server.

    ```text
    https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/
    ```

    Download the `server.7z` for that build. Then create a tiny note so you can roll back later if an update ever breaks the server. Make the file `C:\FXServer\artifact-pin.txt` and fill in what you used:

    ```text
    Artifact OS: Windows
    Channel: Recommended
    Artifact build: <the number you clicked>
    Downloaded: <today's date>
    Source URL: https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/
    Known-good after test: pending
    Rollback build: none yet
    ```

    Only download from this Cfx.re page. Do not use Discord mirrors, YouTube links, or random upload sites.
  </Step>

  <Step num={2} title="Extract and run FXServer" outcome="txAdmin opens by itself at http://localhost:40120.">
    Make a clean folder, extract the archive into it, and run the server binary.

    ```text
    C:\FXServer\server\        <- extract server.7z here
    C:\FXServer\server-data\   <- create this empty (your config + resources live here)
    ```

    Double-click `C:\FXServer\server\FXServer.exe`. A black console window opens and starts txAdmin. Allow the Windows Firewall prompt if it appears, then wait for your browser to open the dashboard.

    ```text
    http://localhost:40120
    ```

    Keep the black FXServer console open the whole time. Closing it stops txAdmin.

    <ExpectedOutput>
    txAdmin: starting...
    [txAdmin] Webserver is now available at http://localhost:40120/
    [txAdmin] Use the PIN below to register the master account: 0427
    </ExpectedOutput>

    The PIN is a random 4-digit code generated fresh on each first boot. It stays valid until you finish setup and create the master account -- it does not time out on a clock. The number 0427 above is only an example. Type the exact digits YOUR console prints, not these. If you need a fresh PIN (for example you restarted before finishing setup), close and reopen FXServer.exe and it generates a new one.
  </Step>

  <Step num={3} title="Claim txAdmin with your Cfx.re account" outcome="txAdmin is linked to your Cfx.re identity and a local admin login exists.">
    On the txAdmin setup screen, enter the PIN printed in the console, then link your Cfx.re account (the same account you used to get the license key at the portal). Authorize txAdmin, set a **local** txAdmin password, and name your server profile.

    Two logins exist and they are not the same thing:

    - **Cfx.re account** is your global identity at `portal.cfx.re`.
    - **txAdmin password** logs you into this dashboard only.

    Security: set a long, unique txAdmin password. The txAdmin dashboard on port 40120 controls your whole server, so never expose port 40120 to the open internet. On your own PC it stays on localhost. On a VPS, leave 40120 closed in the firewall and reach it over an SSH tunnel or VPN, not a public port.
  </Step>

  <Step num={4} title="Run a framework recipe" outcome="txAdmin downloads a framework and writes your first server.cfg.">
    When txAdmin asks how to set up the server, choose the **recipe / template** path (older builds call it "Popular Recipes"). For a new 2026 server, pick one of these:

    - **ESX Legacy** for the largest script catalog and the most copy-paste tutorials.
    - **Qbox** for a QBCore-compatible framework that integrates the ox resources and is available through an official recipe.

    QBCore also has an official recipe. Choose by reading the official framework documentation and checking the compatibility list for the resources you intend to install.

    When the recipe prompts you, paste the key from your password manager into the recipe's key field and set the database name (use `fivem` if asked). You do not edit any file by hand here; txAdmin writes the key into `server.cfg` for you as the line below. Let every recipe step finish, then click **Save & Run Server**.

    ```text
    sv_licenseKey <paste from your password manager>
    ```

    <ExpectedOutput>
    [txAdmin] Running recipe...
    Resources downloaded. Writing server.cfg...
    [txAdmin] Server started. Status: ONLINE
    </ExpectedOutput>

    Exact wording varies by txAdmin build (illustrative). What matters: the recipe shows a progress list that finishes without a red error, and txAdmin then offers to start the server.
  </Step>

  <Step num={5} title="Choose the game build, then start the server" outcome="The server boots with an intentional GTA content build.">
    The **artifact build** from Step 1 is the FXServer software version. `sv_enforceGameBuild` is a different number: the GTA V content build clients must load. The artifact download page does **not** tell you which game-build number to use. Start with the value written by your official recipe. Change it only when a resource or map requires another build, using the supported game-build list in the Cfx.re server-command reference linked below.

    ```text
    sv_enforceGameBuild <supported_game_build>   # use the Cfx.re game-build list, not the artifact number
    ```

    Save the file, then use txAdmin's **Restart Server** action. `sv_enforceGameBuild` is startup-only; `restart` without a resource name is not the server-restart command.

    <ExpectedOutput>
    Server license key authentication succeeded.
    Authenticated with txAdmin.
    Started resource (your framework resources)
    Server is now listening on 0.0.0.0:30120
    </ExpectedOutput>

    Wording is illustrative and varies by build. Success signs to look for: a line confirming license-key authentication, no repeating red errors, and the server reaching an online/listening state.
  </Step>

  <Step num={6} title="Connect from your FiveM client" outcome="Your client joins and txAdmin shows one player online.">
    Open the FiveM client (the game launcher, not the black FXServer window). While FiveM is in focus, press **F8** on your keyboard to open the in-game client console, then type this command and press Enter to connect to your own machine:

    ```bash
    connect 127.0.0.1:30120
    ```

    If the server is on a VPS instead, use its public address and the same port:

    ```bash
    connect YOUR_SERVER_IP:30120
    ```

    Once you spawn in, go back to txAdmin. The Players panel shows 1 online and the Live Console logs your connection. Now update your note so a future you knows this build is safe.

    ```text
    Known-good after test: yes
    ```

    txAdmin writes its own data under `txData/`. On most installs the per-server logs sit here:

    ```text
    txData/<your-server>/logs/admin
    txData/<your-server>/logs/fxserver
    txData/<your-server>/logs/server
    ```

    The path can move between builds. If you do not see `txData`, search your FXServer folder for `txData` and look for the `logs` folder inside the server profile.
  </Step>
</StepGroup>

## Common failures

| What you see | What it means | How to fix it |
|---|---|---|
| FXServer.exe opens then closes immediately | Missing VC++ Redistributable or firewall block | Install VC_redist.x64.exe. Allow FXServer.exe in Windows Firewall |
| "Invalid license key" in console | Key is wrong or not activated | Re-copy from portal.cfx.re. No extra spaces. Key starts with "licensekey:" |
| txAdmin page shows "This site can't be reached" | Server isn't running or port blocked | Make sure FXServer.exe is still running. Check firewall isn't blocking port 40120 |
| Recipe download stuck or failed | GitHub rate limit or slow internet | Wait 5 minutes and retry. If persistent, download the framework manually |
| "MariaDB connection failed" after recipe | MySQL not running or wrong password | Check XAMPP MySQL is started (green). Verify mysql_connection_string in server.cfg |
