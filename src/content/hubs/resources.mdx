- [**Visual Studio Code**](https://code.visualstudio.com/download) for writing and organizing scripts.

<Callout variant="warn" title="One SQL rule">
  Use `?` placeholders for player input. Never build SQL by joining strings.
</Callout>

<Callout variant="info" title="Who maintains ox tools">

## Lua patterns the curriculum skims

Four short references for patterns that appear in real scripts but are not given a full lesson. Read each card once, copy the snippet into a sandbox resource, and confirm the expected output before using them in production code.

<LinkCardGrid>
  <LinkCard href="https://www.lua.org/manual/5.4/manual.html#pdf-pcall" title="Protected calls with pcall" desc="Wrap any call that can throw (oxmysql, JSON decode, third-party exports). pcall returns ok, result so a single bad row never kills the whole event." eyebrow="A" />
  <LinkCard href="https://www.lua.org/manual/5.4/manual.html#pdf-string.format" title="Readable strings with string.format" desc="Stop concatenating with .. for logs. string.format gives you typed placeholders, padding, and decimal control. The lesson logger pattern [QUIET][module][action] assumes you use it." eyebrow="B" />
  <LinkCard href="https://docs.fivem.net/docs/scripting-reference/runtimes/lua/functions/" title="JSON encode and decode" desc="FiveM ships a global json table. Use it for statebag payloads, oxmysql JSON columns, and NUI messages. Always pcall decode; never trust untrusted strings." eyebrow="C" />
  <LinkCard href="https://overextended.dev/ox_lib/Modules/Locale/Shared" title="Localization with locale tables" desc="Hard-coded English breaks the moment a server runs in Spanish or French. Keep one locales/&lt;lang>.lua per language and look up keys at call time. Falls back to English when a key is missing." eyebrow="D" />
</LinkCardGrid>
