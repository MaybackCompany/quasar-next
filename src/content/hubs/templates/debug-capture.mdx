<Callout variant="warn" title="Capture, then stop">
  After a captured error, return early. Do not continue the path that failed. A swallowed error that keeps running is worse than a crash, it corrupts state silently.
</Callout>

## Files

- **server.lua**, `log`, `capture`, `quiet_capture` command for a forced traceback.

## server.lua

```
-- PURPOSE: Structured logs and error capture.
-- DEPENDENCIES: none.
-- WARNING: Capture errors, then stop unsafe work.
-- Expand this: add Discord-free screenshot-friendly fields.
-- SECURITY:
--   - Never log raw player input as a log key, sanitize first.
--   - Bound payload sizes before printing to avoid log floods.
--   - Forward to a server-only audit channel, never echo to clients.

local MODULE = 'debug_capture'

local function log(action, message)
    print(('[QUIET][%s][%s] %s'):format(MODULE, action, message))
end

local function capture(action, fn)
    local ok, result = pcall(fn)
    if not ok then
        log(action, result)
        log('trace', debug.traceback(result))
        return nil
    end
    return result
end

RegisterCommand('quiet_capture', function(source)
    capture('command', function()
        error(('forced error source=%s'):format(source))
    end)
end, false)
```

## Install

1. Create `resources/[local]/qu_debug/`.
2. Copy `server.lua` and add a manifest with `server_script 'server.lua'`.
3. Add `ensure qu_debug` in `server.cfg`.
4. From the server console run `quiet_capture`.
5. Match `[QUIET][debug_capture][command]` followed by `[trace]` with the stack.
