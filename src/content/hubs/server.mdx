Start here

Do not install 80 resources first. Pick the path below, finish each proof artifact, then move to the next card.

- **I own the server:** start with tools, concept, stack, staff, and QA.
- **I write scripts:** finish Track 0 local setup, then go to your first Lua resource.
- **I use hosting:** follow the hosted panel proof path before resource installs.
- **I am unsure:** choose local if learning alone, hosted if staff or testers need access now.

## Which one should you choose?

| Your situation | Choose | Why | Expected proof |
| --- | --- | --- | --- |
| You are a non-coder owner planning a public city. | Owner path | You need scope, budget, staff, QA, and rollback before deep code. | `server-brief.md` and `launch-report.md`. |
| You want to learn Lua and make resources. | Track 0 local setup | You need a server you can break safely while learning. | `connect localhost` works and console is clean. |
| You already have Goodleaf or another hosting panel. | Hosted lab | Your main risk is panel paths, database credentials, and remote file placement. | txAdmin `status` and HeidiSQL DB proof. |
| You have staff waiting to test this week. | Hosted plus owner path | Staff need remote access, but you still need concept, permissions, and QA. | Direct connect from another PC and staff permission test. |
| You bought scripts but no server exists yet. | Owner path, then hosted or local | Scripts are not the foundation. Tools, DB, framework, and rollback are. | Artifact pin, stack decision, and one script tested. |

## How to use this path

1. Choose the route above: owner, developer, hosted, or local.
2. Open the first lesson for that route.
3. Complete the artifact exactly: checklist, brief, stack decision, backup, permission test, or launch report.
4. Collect proof in txAdmin, F8, HeidiSQL, file manager, or screenshots.
5. Move to the next lesson only when the pass condition is met.
