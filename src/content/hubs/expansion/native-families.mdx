## Outcome

You will group natives by entity, ped, vehicle, and HUD use.

<Callout variant="tip" title="Why it matters">
  Native names look random at first. Families make the docs easier to scan.
</Callout>

## Vocabulary

| Term | Description |
| --- | --- |
| Ped native | A native that reads or changes a character. Example: PlayerPedId() returns your ped. |
| Vehicle native | A native that reads or changes a vehicle. Example: SetVehicleEngineOn changes engine state. |
| HUD native | A native that affects screen UI. Example: BeginTextCommandThefeedPost starts a feed notice. |

## Minimal example

```lua
local MODULE = 'native_family'

RegisterCommand('quiet_family', function()
    local ped = PlayerPedId()
    local coords = GetEntityCoords(ped)
    print(('[QUIET][%s][entity] x=%.2f y=%.2f'):format(MODULE, coords.x, coords.y))
end, false)
```

<ExpectedOutput>

  [QUIET][native_family][entity] prints your current coordinates.

## Production checklist

1. Confirm the code is on the correct side.
2. Validate every payload before acting.
3. Log with `[QUIET][module][action]`.
4. Add one cleanup or rollback path.
5. Copy the expected output into your test note.

## Things to try

- Swap GetEntityCoords for a vehicle native.
- Add one HUD notification.
- Write one line explaining the native family.
