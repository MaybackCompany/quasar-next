## Outcome

You will store placed object state without trusting client labels.

<Callout variant="tip" title="Why it matters">
  Placed props need ids. Player-provided labels need validation.
</Callout>

## Vocabulary

| Term | Description |
| --- | --- |
| Stateful prop | A placed object with saved data. Example: stash id and coords. |
| Ped system | Managed NPC spawn and cleanup code. Example: shop ped registry. |
| Drop cleanup | Cleanup when a player leaves. Example: playerDropped removes temp props. |

## Minimal example

```lua
local MODULE = 'prop_ped'
local ownedProps = {}

RegisterNetEvent('quasar:props:place', function(payload)
    local src = source
    if type(payload) ~= 'table' or type(payload.kind) ~= 'string' then
        print(('[QUIET][%s][reject] source=%s bad payload'):format(MODULE, src))
        return
    end
    ownedProps[src] = ownedProps[src] or {}
    print(('[QUIET][%s][place] source=%s kind=%s'):format(MODULE, src, payload.kind))
end)
```

## Production checklist

1. Confirm the code is on the correct side.
2. Validate every payload before acting.
3. Log with `[QUIET][module][action]`.
4. Add one cleanup or rollback path.
5. Copy the expected output into your test note.
