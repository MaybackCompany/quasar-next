## Outcome

You will create one logger helper used by every script file.

<Callout variant="tip" title="Why it matters">
  One logger makes screenshots searchable. Mixed print styles hide the real failure.
</Callout>

## Vocabulary

| Term | Description |
| --- | --- |
| Context | Extra data attached to a log. Example: source, action, amount. |
| Stop condition | A rule that halts unsafe work. Example: missing Player stops payout. |
| Traceback | The call path for an error. Example: debug.traceback(err). |

## Minimal example

```lua
local MODULE = 'logger'

local function log(action, message)
    print(('[QUIET][%s][%s] %s'):format(MODULE, action, message))
end

local ok, err = pcall(function()
    error('forced logger test')
end)

if not ok then
    log('error', err)
    log('trace', debug.traceback(err))
end
```

## Production checklist

1. Confirm the code is on the correct side.
2. Validate every payload before acting.
3. Log with `[QUIET][module][action]`.
4. Add one cleanup or rollback path.
5. Copy the expected output into your test note.

## Things to try

- Add source to the log.
- Add payload size.
- Use the helper in another file.
