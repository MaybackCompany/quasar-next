## Outcome

You will run one versioned migration and verify the schema row.

<Callout variant="tip" title="Why it matters">
  Manual table edits drift. Versioned migrations explain what changed.
</Callout>

## Vocabulary

| Term | Description |
| --- | --- |
| Migration | SQL that changes schema. Example: CREATE TABLE IF NOT EXISTS. |
| Version table | A table that records applied migrations. Example: qu_schema_versions. |
| Index | A lookup helper for frequent queries. Example: INDEX citizenid speeds reads. |

## Minimal example

```lua
local MODULE = 'migrations'

MySQL.ready(function()
    MySQL.query('CREATE TABLE IF NOT EXISTS qu_schema_versions (name VARCHAR(64) PRIMARY KEY)', {})
    MySQL.insert('INSERT IGNORE INTO qu_schema_versions (name) VALUES (?)', { '001_accounts' })
    print(('[QUIET][%s][applied] 001_accounts'):format(MODULE))
end)
```

## Common failure modes

attempt to call a nil value

means a native, export, or helper is unavailable on this side.

No such export GetCoreObject in resource qb-core

means the dependency is missing or ensured late.

attempt to index a nil value

means a table, player object, or entity handle is missing.

SQL error: Table 'qu_migrations' doesn't exist

means the migration did not run.
