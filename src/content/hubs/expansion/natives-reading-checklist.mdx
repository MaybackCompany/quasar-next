## Outcome

You will read native docs without guessing the side, handle, or expected return.

<Callout variant="tip" title="Why it matters">
  Wrong-side native calls waste time. Bad handle checks create silent failures.
</Callout>

## Vocabulary

| Term | Description |
| --- | --- |
| Native | A FiveM or GTA function. Example: DoesEntityExist(entity) checks a handle. |
| Handle | A number that points at an entity. Example: vehicle is a handle. |
| Ownership | Who can safely control a networked entity. Example: server logic must not assume ownership. |

## Minimal example

```lua
local MODULE = 'natives'

RegisterCommand('quiet_native_check', function()
    local ped = PlayerPedId()
    local vehicle = GetVehiclePedIsIn(ped, false)

    if vehicle == 0 or not DoesEntityExist(vehicle) then
        print(('[QUIET][%s][missing] no vehicle'):format(MODULE))
        return
    end

    print(('[QUIET][%s][ok] vehicle=%s'):format(MODULE, vehicle))
end, false)
```

## Production checklist

1. Confirm the code is on the correct side.
2. Validate every payload before acting.
3. Log with `[QUIET][module][action]`.
4. Add one cleanup or rollback path.
5. Copy the expected output into your test note.

## Things to try

- Run it outside a vehicle.
- Run it inside a vehicle.
- Change the native and read its return type first.
