## Outcome

You will catch SQL failures and log query context.

<Callout variant="tip" title="Why it matters">
  Database errors need action names. Slow queries need timing.
</Callout>

## Vocabulary

| Term | Description |
| --- | --- |
| Connection failure | The database is unreachable. Example: oxmysql cannot connect. |
| Transaction | A group of writes that should succeed together. Example: transfer debit and credit. |
| Slow query | A query that takes too long. Example: SELECT took 320ms. |

## Minimal example

```lua
local MODULE = 'sql_failures'

local started = GetGameTimer()
MySQL.query('SELECT 1 AS ok', {}, function(result)
    local took = GetGameTimer() - started
    if not result then
        print(('[QUIET][%s][error] SELECT 1 failed'):format(MODULE))
        return
    end
    print(('[QUIET][%s][query] took_ms=%s'):format(MODULE, took))
end)
```

## Production checklist

1. Confirm the code is on the correct side.
2. Validate every payload before acting.
3. Log with `[QUIET][module][action]`.
4. Add one cleanup or rollback path.
5. Copy the expected output into your test note.

## Things to try

- Log the action name.
- Add a slow query threshold.
- Write the exact connection error.

## Common failure modes

attempt to call a nil value

means a native, export, or helper is unavailable on this side.

No such export GetCoreObject in resource qb-core

means the dependency is missing or ensured late.

attempt to index a nil value

means a table, player object, or entity handle is missing.

SQL error: Table 'qu_sql_failures' doesn't exist

means the migration did not run.
