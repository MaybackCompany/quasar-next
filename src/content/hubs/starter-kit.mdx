## 7-day checklist

Do these in order. If one day still feels shaky, repeat it before moving on. The goal is not speed. The goal is confidence.

<Callout variant="info" title="How to use this starter">
  Read the explanation first. Then do the lab. If the test fails, do not jump ahead. Read the failing line, compare it to the example, and fix one thing at a time.

  - Days 1 to 3 teach Lua and the client/server wall.
  - Day 4 adds ox_lib, which is a common FiveM helper library.
  - Days 5 and 6 compare ESX and QBCore.
  - Day 7 shows how to think across frameworks instead of blindly copying one snippet.
  - If a red error scares you, open the [debug class for beginners](/lessons/debug-class).
</Callout>
