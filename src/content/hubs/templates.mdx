<Callout variant="tip" title="10-minute setup checklist">
  1. Create one resource folder.
  2. Copy the matching template files.
  3. Add dependencies before your resource in `server.cfg`.
  4. Run once and match the expected log or QA result.
  5. Change only one rule at a time.
</Callout>

## How to use templates

Start with the closest template. Keep the file names. Run it once before adding your feature rules.

<Callout variant="warn" title="Security warning">
  A client can ask. The server decides. Never trust client money, job, item, plate, or coords.
</Callout>
