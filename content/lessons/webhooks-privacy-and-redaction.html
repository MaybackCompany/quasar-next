Discord webhooks, privacy, and PII redaction

By the end of this lesson you will send Discord webhooks from your FiveM server that are fast, reliable, and never expose player IP addresses, emails, or other personally identifiable information in public channels.

You'll ship
A webhook utility that sends embed-rich messages to Discord, with automatic PII redaction that strips IPs, license identifiers, and raw Steam IDs before they leave your server.
Time
About 30 minutes if you go slow.
You'll learn
How to POST to Discord webhooks from Lua, how to structure embeds for readability, and why redacting PII is not optional — it is a legal requirement in many jurisdictions.
Prereqs
You have a Discord server and can create a webhook URL. That is enough.
In 3 bullets

You will build a Discord webhook system that does this:

  • Send structured Discord embeds for key server events: player joins, purchases, admin actions, and errors.
  • Automatically redact IP addresses, Steam hex IDs, license identifiers, and email patterns from every payload before it hits the Discord API.
  • Rate-limit webhook calls to stay under Discord's 30-messages-per-minute cap without dropping important events.

Common failures

  • HTTP 400 Bad Request means the JSON payload is malformed. Check for trailing commas or unescaped quotes.
  • HTTP 429 Too Many Requests means you hit Discord's rate limit. Add a queue or slow down sends.
  • embed not appearing means the embeds key is missing or misspelled in the JSON payload.
  • IP address visible in embed means the redaction function is not being called on the description text before sending.

Refactor

Extract the webhook sender into webhook.send(channel, embed) so multiple channels can be configured with different URLs from a single module.

Cheat sheet

PII redaction patterns

  • IP address: %d+%.%d+%.%d+%.%d+ → [IP]
  • Steam hex: steam:[0-9A-Fa-f]+ → [STEAM]
  • License: license:[0-9a-f]+ → [LICENSE]
  • Email: [%w%.]+@[%w%.]+ → [EMAIL]

The big idea, in one minute

A Discord webhook is a public billboard. Anything you post to it can be screenshotted, shared, and archived indefinitely. If you post a player's IP address in a staff channel, that IP is now permanently stored on Discord's servers and visible to every staff member. Under GDPR, CCPA, and similar privacy laws, that is a data breach. Your server logs are private. Your Discord channel is not. Treat the webhook like a public broadcast, not a debug console.

Think of a PA system in a store.

The store manager uses the PA to announce "customer at register three needs help." The manager does not announce the customer's credit card number over the PA. Your webhook is the PA system. Announce the event. Never announce the private data.

Bad way and good way, side by side

Most developers dump raw player data into webhooks because it is convenient. That is a privacy incident waiting to happen.

Don't do this

Dump raw identifiers and IPs directly into the webhook.

Every staff member can see the player's Steam ID, license key, and IP address. One disgruntled staff member screenshots it and posts it publicly. You are now liable.

Do this

Redact PII before sending. Log raw data to server files only.

The Discord embed shows the player name and that identifiers were loaded. Raw identifiers are written to a private server log file that only you can access. Privacy preserved.

Step 1

Create the Discord webhook URL

The URL is the only secret you need. Protect it like a password.

  1. In Discord, open Server Settings → Integrations → Webhooks.
  2. Click "New Webhook", name it "FiveM Server", select a staff-only channel.
  3. Copy the webhook URL. It looks like https://discord.com/api/webhooks/....
  4. Add it to server.cfg: set qu_discord_webhook "https://discord.com/api/webhooks/...".
  5. Never commit the URL to a public repository. Use a convar or environment variable.
Step 2

Write the sendDiscord function

One function. Every event calls it. One place to add redaction and rate limiting.

  1. Create resources/qu_webhooks/server.lua.
  2. Read the webhook URL from convar with GetConvar.
  3. Write sendDiscord(embed) that encodes the embed to JSON and POSTs it.
  4. Add error logging so failed webhook calls do not disappear silently.
Step 3

Add PII redaction and test

Redact before sending. Log raw data to files. Never reverse the order.

  1. Write the redact(text) function from the cheat sheet patterns above.
  2. Call redact on every string field in the embed before encoding to JSON.
  3. Add WriteLog or file I/O to save raw identifiers to a server-side log file.
  4. Test with a real player join. Confirm the Discord embed shows no raw identifiers.
Recap

Three tests, in order

Run each test and verify the output before wiring more events.

  1. Test 1, embed arrives.
    • Action: join the server and check the Discord channel.
    • Expected: an embed appears with "Player Connecting" and your character name.
    • How to verify: the embed has a green color bar and the correct title.
  2. Test 2, PII redacted.
    • Action: inspect the embed description and fields carefully.
    • Expected: no IP address, Steam hex, license key, or email is visible.
    • How to verify: search the embed text for patterns like 192.168 or steam: — none should appear.
  3. Test 3, raw data logged.
    • Action: check the server log file after a player joins.
    • Expected: the raw identifiers are written to the log file.
    • How to verify: open the log file and confirm the identifiers are there, but they never appeared in Discord.
What if a staff member needs to see the raw identifier?

They check the server log file, which is only accessible to server administrators. Discord is public within your staff team. The log file is private.

The rules you just followed

  • Webhook URLs are stored as server convars, never in client-accessible files.
  • Every string in a Discord embed passes through the redaction function before sending.
  • Raw identifiers are written to private server log files, never broadcast to Discord.
  • Webhook errors are logged to console — a silent failure means missing alerts.
  • Rate limiting prevents Discord from blocking your webhook for exceeding 30 messages per minute.
