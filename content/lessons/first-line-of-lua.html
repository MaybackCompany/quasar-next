The mental model: what Lua is

Think of Lua as a very polite language. You tell it to do things in plain sentences, one line at a time, and it does exactly what you say. No more, no less. That's actually the gift here: if something goes wrong, it's almost always because you asked for the wrong thing, not because the computer misunderstood.

A Lua file (.lua) is just a text file with instructions. FiveM reads your file top to bottom and runs each line in order. That's it. No magic. No background threads starting unless you ask. Just lines, running one after another.

FiveM note

Everything in FiveM, every command, every menu, every job, is written in Lua files exactly like the one you're about to write. The millionaire-roleplay server with 300 players and the "hello world" script you build in 60 seconds use the same language. The only difference is reps.

Step 1

Create the resource folder

Outcome: a new folder inside resources/ called qu_hello.

Navigate to your server's resources folder. Inside it, create a new folder called qu_hello. Case matters, lowercase only. Spaces break things, underscores only.

Tip

The qu_ prefix is our convention, it groups every Quasar Academy script together and makes them easy to find. When you release scripts one day, you will want a prefix too (Quasar Store uses qs-).

Step 2

Create the manifest file

Outcome: FiveM knows your resource exists and which files to load.

Inside qu_hello/, create a new file called exactly fxmanifest.lua. This file is the "business card" of your resource, it tells FiveM: "hello, I exist, here's what I contain".

Paste this inside it and save:

Three lines actually matter here:

  • fx_version 'cerulean': the FiveM manifest version your script targets. cerulean is current. Use it, don't overthink it.
  • game 'gta5': which game. FiveM also supports RedM; we're on GTA V.
  • server_script 'server.lua': the file you'll write next, running on the server side.
Watch out

The filename is fxmanifest.lua: one word, all lowercase. Not FXManifest.lua, not fx_manifest.lua. FiveM will ignore wrong spellings silently and you'll spend 20 minutes wondering why nothing runs.

What each piece means
  • print: a function built into Lua. A function is a pre-made action you can use. print means "show this on the console".
  • ( and ): the parentheses hold whatever you want to pass into the function. Like handing someone a note.
  • 'Hello, Los Santos': a string. Strings are text wrapped in quotes. Lua accepts single ' or double " quotes, both work. We'll use single throughout this course for consistency.
Say it out loud

"Call the print function, pass it the string 'Hello, Los Santos'." When you can narrate your own code in plain English, you're reading it right.

Recap 1

Can you spot the parts?

Before you restart the server, answer this in your head:

Which part is the function, and which part is the string?

print is the function, the verb, the action. 'Your server is up' is the string, the text, wrapped in quotes. The parentheses are just the delivery envelope between them.

What would happen if you removed the quotes?

Lua would try to read Your, server, is, up as four variables, and they don't exist, so you'd get an error. Quotes turn text into a string. Without them, Lua thinks it's a name.

Store the message in a variable

Change server.lua to look like this:

Two lines. Same output. But something important happened: you created a variable.

What this code does, line by line
  1. local: says "this box only exists inside this file". Always use local. (We'll explain why in Lesson 02.)
  2. greeting: the name you chose. Any name you want, as long as it starts with a letter and has no spaces.
  3. =: "put this value into this box". Lua calls this assignment.
  4. 'Hello, Los Santos': the string you're putting inside.
  5. print(greeting): print what's in the greeting box. No quotes around greeting: you want the contents of the box, not the word "greeting".

Restart the resource: ensure qu_hello. Same output. But now, if you want to print that message in five different places later, you only have to change the text in one spot. That's what variables are for: giving values a name so you can reuse and change them without chasing copies.

Common beginner mistake

print('greeting') with quotes prints the literal word "greeting". print(greeting) without quotes prints what's inside the greeting box. Quotes = literal text. No quotes = variable lookup. This distinction trips up every beginner for about 48 hours, then it clicks forever.

Common mistakes & what the console tells you

Error: attempt to call a nil value (global 'Print')

You capitalized Print. Lua is case-sensitive. It's print: all lowercase.

Error: ')' expected near '<eof>'

You opened a parenthesis and forgot to close it. Count your ( and ): they must match.

Nothing happened at all

Three usual suspects: (1) file is named wrong, must be fxmanifest.lua, (2) folder name is wrong in your ensure command, (3) you edited the file but didn't save it. Ctrl/Cmd + S.

Error: '<name>' expected near '='

You forgot the word local before greeting, or you wrote local = 'hello' (no variable name). Format is always local name = value.

Cheat sheet: keep this open next lesson

print(x)
Writes x to the server console.
'text'
A string, literal text. Single or double quotes, stay consistent.
local name = value
Create a variable scoped to the current file. Always prefer local.
ensure resource_name
Start-or-restart a resource from the server console. Your most-used command.
fxmanifest.lua
Every resource needs one. Declares version, game, author, and which files to load.
