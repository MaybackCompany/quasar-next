## Build it

<StepGroup>
  <Step num={1} title="Install MariaDB" outcome="MariaDB runs as a Windows service on 127.0.0.1:3306.">
    A FiveM server stores player data in a database. We use MariaDB, a free standalone database that runs as a Windows service in the background. A service is a program Windows starts and keeps running on its own.

    Download the Windows x86_64 MSI installer from https://mariadb.org/download/ and run it. Accept the defaults, but set a strong **root password** during install and save it in a password manager. Keep port **3306** and install it as a **Windows service**.

    Confirm the service is running. Open PowerShell and run:

    ```powershell
Get-Service MariaDB*
    ```

    You want a row whose Status reads **Running**.

    <ExpectedOutput>
      Get-Service MariaDB* shows a MariaDB row with Status Running. MariaDB is listening on port 3306.
    </ExpectedOutput>
  </Step>

  <Step num={2} title="Download and extract FXServer" outcome="FXServer.exe is in your server folder.">
    FXServer is the actual server program. You download it as an artifact, which is just a pre-built copy of the server software. Go to the official Windows artifacts page:

    ```text
https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/
    ```

    Download the **latest** build. It sits at the very top of the list. Ignore the green "recommended" row further down. That one is older and lags behind, and you want the current build. Click the top build and download its `server.7z`.

    One rule for later: if a brand-new build ever misbehaves after an update, drop to the build directly below it. That is the only reason to ever run an older one. You do not pin or freeze anything yet. That comes once your server is live and public.

    Open `server.7z` with 7-Zip or WinRAR and extract everything to:

    ```text
C:\FXServer\
    ```

    Inside you should see `FXServer.exe`, the `citizen/` folder, and other files. Only download from this Cfx.re page. Do not use Discord mirrors, YouTube links, or random upload sites.

    <ExpectedOutput>
      C:\FXServer\FXServer.exe exists.
    </ExpectedOutput>
  </Step>

  <Step num={3} title="Start FXServer and link your account" outcome="txAdmin opens, is linked to Cfx.re, and has a backup password.">
    Double-click `C:\FXServer\FXServer.exe`. A black console window opens and prints a **4-digit PIN**. Keep this window open. Your browser opens txAdmin at `http://localhost:40120`. If it does not, go there manually.

    Do these in order:

    1. Enter the 4-digit PIN from the console.
    2. Click **Link account**, sign in to Cfx.re, and click **Continue** to authorize.
    3. Set a **backup password** for the dashboard.
    4. Agree to the license, then click **Register**.

    The PIN is generated fresh on each first boot, so the example digits in any guide are not yours. Type the exact digits your console prints. If you restarted before finishing, close and reopen `FXServer.exe` for a new PIN.

    **Keep the black FXServer console open the whole time.** Closing it stops txAdmin.

    <ExpectedOutput>
      [txAdmin] Webserver is now available at http://localhost:40120/
      [txAdmin] Use the PIN below to register the master account: 0427
    </ExpectedOutput>
  </Step>

  <Step num={4} title="Deploy the QBCore recipe" outcome="QBCore is downloaded, server.cfg is written, and the server starts.">
    A recipe is a one-click installer inside txAdmin. It downloads a framework and writes your first `server.cfg` for you. QBCore (the `qb-core` resource) is a popular roleplay framework.

    On the setup page, name your server and click **Next**. For **Deployment Type** choose **Popular Recipes**, then pick the **QBCore** recipe. Leave the data location as-is, click **Save**, then **Go to Recipe Deployer**.

    In the deployer:

    1. **Review Recipe**: click **Next**.
    2. **Input Parameters**: click the **Portal** link to open portal.cfx.re, sign in to Cfx.re, click **Create a key**, then **GENERATE**, then **copy** the key. The key is a string that starts with `cfxk_`.
    3. Back in txAdmin, paste the key and click **Run Recipe**.

    The recipe downloads QBCore, creates the database tables, and writes `server.cfg`. When it finishes, click **Save and Run Server**.

    You do not edit any file by hand here. txAdmin writes the key into `server.cfg` for you, with no prefix in front of it, as the line below:

    ```text
sv_licenseKey "cfxk_yourkeyhere"
    ```

    <ExpectedOutput>
      [txAdmin] Running recipe... Resources downloaded. Writing server.cfg...
      [txAdmin] Server started. Status: ONLINE
    </ExpectedOutput>
  </Step>

  <Step num={5} title="Connect HeidiSQL to your database" outcome="You can browse the QBCore tables.">
    HeidiSQL is a free GUI that lets you look inside the database. Download it from https://www.heidisql.com. Open it, click **New** session, and fill in:

    - Host: `127.0.0.1`
    - User: `root`
    - Password: the root password you set when installing MariaDB
    - Port: `3306`

    Click **Open**. You should see the QBCore database with tables like `players`.

    <ExpectedOutput>
      HeidiSQL shows the QBCore database. You can browse the players table and other QBCore tables.
    </ExpectedOutput>
  </Step>

  <Step num={6} title="Join your own server" outcome="Your client connects and txAdmin shows one player online.">
    Open the FiveM client (the game launcher, not the black FXServer window). With FiveM in focus, press **F8** to open the in-game client console, then run the command below. F8 is the in-game console where you type commands while playing.

    ```text
    connect 127.0.0.1:30120
    ```

    Once you spawn in, go back to txAdmin. The Players panel shows 1 online and the Live Console logs your connection. That is your proof: the server runs, and you are on it.

    <ExpectedOutput>
      You spawn into the server. The txAdmin Players panel shows 1 player online.
    </ExpectedOutput>
  </Step>
</StepGroup>

## How it works

The pieces fit together in one line. FXServer is the server software. txAdmin is the dashboard bundled inside it. The recipe is a one-click installer that downloads QBCore and writes `server.cfg`. MariaDB stores player data. HeidiSQL is just a window into that database.

Two things are easy to confuse. **Port 40120 is the txAdmin dashboard in your browser. Port 30120 is the game server your FiveM client connects to.** They are different ports for different jobs.

Your license key is created during the recipe step, not before, and txAdmin pastes it into `server.cfg` for you as `sv_licenseKey`. You never hand-edit that file to get started.

## If something went wrong

<FailureList
  items={[
    { symptom: "The browser never opens or localhost:40120 is refused", fix: "FXServer.exe is not running or its console was closed. Reopen FXServer.exe, keep the black window open, then visit http://localhost:40120 manually." },
    { symptom: "FXServer.exe opens then closes immediately", fix: "Missing Visual C++ Redistributable. Install VC_redist.x64.exe from Microsoft, then run FXServer.exe again. Also allow it through Windows Firewall." },
    { symptom: "Recipe download stuck or failed partway", fix: "GitHub rate limit or slow internet. Wait a few minutes and run the recipe again. If it keeps failing, download QBCore manually from its docs." },
    { symptom: "Server license key authentication failed", fix: "Open portal.cfx.re, confirm the key is active and belongs to your account, paste the exact cfxk_ value into sv_licenseKey with no prefix in front of it, and restart." },
    { symptom: "MariaDB connection failed after the recipe", fix: "Start the MariaDB service (services.msc, or Get-Service MariaDB* in PowerShell), and confirm the database password matches what the recipe used." },
    { symptom: "MariaDB service won't start (port 3306 in use)", fix: "Another MySQL is using port 3306. Stop the conflicting service, or change MariaDB's port in my.ini and update HeidiSQL to match." },
    { symptom: "ensure not found in the HeidiSQL console", fix: "HeidiSQL only runs SQL, not FiveM commands. ensure is the FiveM command that starts a resource. Use the txAdmin Live Console for FiveM commands, HeidiSQL for SQL only." },
    { symptom: "The txAdmin setup screen does not match these steps", fix: "Labels move between builds. Follow the order: enter PIN, link account, pick recipe, create and paste key, Run Recipe, Save and Run Server." },
  ]}
/>
