Networking, DDoS protection, and proxies

By the end of this lesson you will understand how your FiveM server talks to the internet, what a DDoS attack looks like, and how to put a proxy in front of your server so attackers never touch your real IP.

You'll ship
A plan for hiding your server IP behind a proxy, plus config changes that make DDoS attacks far less effective.
Time
About 25 minutes to understand, longer to implement if you need to buy a proxy service.
You'll learn
How UDP vs TCP works in FiveM, what amplification attacks are, and why a reverse proxy is your best defense.
Prereqs
You know your server's IP address and port. Basic networking knowledge helps but is not required.
In 3 bullets

You will understand networking and DDoS defense like this:

  • FiveM uses both UDP (for game traffic, fast and connectionless) and TCP (for reliable data like chat and events). Attackers target the UDP port because it is easier to flood.
  • A DDoS attack sends so much garbage traffic that your server cannot hear real players. It is like fifty people shouting into a phone at once.
  • A proxy sits between the internet and your server. Players connect to the proxy, not your real IP. The proxy filters out attack traffic and only forwards clean packets.

Step by step

Lesson 08: Networking, DDoS protection, and proxies

Start here

What this is
Everything between your server process and the players who connect to it.
Example: your server listens on port 30120. Players send packets to that port.
Where to find it
Networking is configured in server.cfg via endpoint_add_tcp and endpoint_add_udp.
Example: endpoint_add_udp "0.0.0.0:30120" opens the UDP game port.
How to use it
Bind the right ports, firewall everything else, and optionally add a proxy.
Example: only ports 30120 (UDP+TCP) and 40120 (TCP for txAdmin) should be open.
What to do now
Check which ports are open on your server and close everything that is not FiveM.
Example: run a port scan on your own server IP to see what is exposed.

Outcome

You will understand your server's network footprint, harden your firewall, and evaluate proxy options to hide your real IP.

Why it matters

A single DDoS attack can knock your server offline for hours. Competitors, angry players, or random attackers target FiveM servers constantly. Knowing your network setup and having a proxy plan is the difference between 30 seconds of downtime and a ruined launch day.

Vocabulary

UDP
User Datagram Protocol. Fast, connectionless. FiveM uses it for game sync, voice, and position updates. Easier to spoof and flood than TCP.
Example: player movement packets are sent over UDP because speed matters more than perfect delivery.
TCP
Transmission Control Protocol. Slower but guaranteed delivery. FiveM uses it for chat, events, and server list registration.
Example: a TriggerServerEvent call uses TCP because the server must receive it exactly once.
Reverse proxy
A server that sits in front of yours, receives all traffic, filters it, and forwards only clean packets to your real server.
Example: players connect to proxy.example.com:30120. The proxy sends clean traffic to your real IP at 10.0.0.5:30120.

Checklist

  1. Open server.cfg and confirm your endpoint_add_udp and endpoint_add_tcp are set correctly for your setup.
  2. Check your server's firewall. On Linux run sudo ufw status. On Windows check Windows Firewall.
  3. Close every port that is not 30120, 40120, and SSH (22). If using RDP, keep 3389 only if needed.
  4. If you are on a VPS, check your hosting provider's external firewall panel and apply the same rules there.
  5. Research proxy providers: TCPShield, Cosmic Guard, or a generic GRE tunnel. Pick one that fits your budget.
  6. Document your current public IP. Plan to change it when you set up the proxy.

Exercise

Find your server's public IP and run a basic test. From your personal computer, open a terminal and type ping your-server-ip. If you get a reply, your server is visible to the internet. That is what a proxy hides.

Expected output: The ping command returns a reply with a latency number. This means your server IP is publicly reachable — exactly what attackers scan for.

Common failures

  • Players cannot connect after enabling firewall means you blocked port 30120. Run sudo ufw allow 30120/udp and sudo ufw allow 30120/tcp.
  • Server shows offline in the browser list means endpoint_add_tcp is missing or blocked. The server list uses TCP on port 30120.
  • Proxy blocks legitimate players means the proxy filter is too aggressive. Check its dashboard for false positives and whitelist the affected IPs.
  • Proxy adds 100ms of latency means the proxy server is geographically far from your game server. Pick a proxy provider with a node close to your server location.

Refactor

Write a small server-side resource that logs every new connection's IP and timestamps it to a file. After a week, review the log. You will see the same IPs scanning your server repeatedly — that is your threat baseline. Use this data to tune your proxy or firewall rules.
