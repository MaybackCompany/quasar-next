Start here

What this is
A free Windows GUI for MySQL/MariaDB that lets you click tables instead of memorising commands.
Example: double-click the players table and see all rows in a spreadsheet-like grid.
Where to find it
Download from heidisql.com. It runs on Windows. Mac users can use TablePlus or DBeaver.
Example: the installer is a single .exe, under 12 MB.
How to use it
Connect with host, port, user, and password. Then everything is clickable.
Example: click your database name in the left panel, then click the Query tab.
What to do now
Connect, run SELECT 1, browse one table, run one real query.
Example: SELECT * FROM players LIMIT 10.

Outcome

You will run SELECT * FROM players LIMIT 10 in HeidiSQL and see actual rows from your live server database in a sortable grid.

Why it matters

Every FiveM server stores player data in MySQL. When a player says "my money disappeared" or "I am still banned", the answer is in the database. Without a GUI, you are SSH-ing in and typing SQL blind. With HeidiSQL, you click, sort, and find the problem in under a minute.

Vocabulary

Host / Port
The address and door where MySQL listens. Usually localhost on port 3306.
Example: a VPS database might be at 127.0.0.1:3306 or a remote IP.
SELECT
The SQL command that reads data. It never changes anything.
Example: SELECT name, money FROM players WHERE money > 1000000.
WHERE clause
The filter that limits which rows a query affects. Without it, you affect every row.
Example: UPDATE players SET banned = 0 WHERE id = 5 — only player 5.

Checklist

  1. Download and install HeidiSQL from heidisql.com.
  2. Open HeidiSQL and click "New" to create a session.
  3. Fill in your MySQL host, port (usually 3306), username, and password.
  4. Click "Open". The left panel populates with your databases and tables.
  5. Click the "Query" tab and type SELECT 1;. Press F9 to run it.
  6. Browse a real table by double-clicking it in the left panel. Data appears in a grid.

Common failures

  • Can't connect to MySQL server on 'localhost' (10061) — MySQL is not running or is on a different port. Try port 3306, then check your server.cfg connection string.
  • Access denied for user 'root'@'...' — the password is wrong or the user is not allowed from your IP. Many hosts disable remote root access. Use a dedicated database user.
  • Tables are listed but queries return empty — the database selected might be the wrong one. Check your connection string for the exact database name.
  • Unknown column 'name' in 'field list' — your framework uses a different column name. Browse the table first by double-clicking it to see the actual column names.

Refactor

Save your three most-used queries as HeidiSQL "Snippets" (right-click in the query editor → Insert Snippet) so you can paste them with two clicks instead of retyping.

The big idea, in one minute

The command-line MySQL client is powerful but unforgiving. One missing semicolon and you are stuck. HeidiSQL wraps the same database in a point-and-click interface. You see the tables on the left, the data in a grid in the middle, and the query editor below. You can sort columns by clicking the header. You can filter rows with a right-click. Most importantly, you can edit a single cell without writing an UPDATE query — but only after you learn why the WHERE clause is the most important three words in SQL.

Think of a library catalogue.

The command line is asking a librarian to find a book by reciting the Dewey Decimal number from memory. HeidiSQL is walking to the shelf, running your finger along the spines, and pulling the book yourself. Both work. One requires memorisation. The other uses recognition — and humans are better at recognition.

Before you start

If any box is unchecked, peek at the linked lesson for five minutes first.

Step 1

Download and install HeidiSQL

A lightweight installer. No bundled toolbars, no registration.

  1. Go to https://www.heidisql.com/download.php.
  2. Download the installer (or the portable .zip if you prefer no installation).
  3. Run the installer. Accept the defaults. It finishes in under 30 seconds.
  4. Open HeidiSQL. You see the Session Manager window.
Step 2

Create a new session

A session remembers your credentials. Create one for each database you manage.

  1. In the Session Manager, click "New" (bottom-left).
  2. Give it a name like "FiveM Production".
  3. Set Network type to "MySQL (TCP/IP)".
  4. Fill in: Hostname/IP (often 127.0.0.1 or your VPS IP), Port (3306), Username, Password.
  5. Leave Databases empty for now — you will pick one after connecting.
  6. Click "Open". If it connects, the left panel fills with your databases.
Step 3

Run your first queries

The Query tab is where you type SQL. The results appear in a grid below.

  1. Click the database your FiveM server uses in the left panel. It highlights.
  2. Click the "Query" tab in the main area (or press Ctrl+T).
  3. Type SELECT 1; and press F9. A grid appears with one cell containing "1".
  4. Now type SHOW TABLES; and press F9. Every table in the database is listed.
  5. Double-click any table name in the left panel to browse its data without writing SQL.
Step 4

Run a real query

Find the players table and filter it. This is the core skill.

  1. In the left panel, expand your database and find the players table (may be named differently per framework).
  2. In the Query tab, type SELECT * FROM players LIMIT 10; (replace 'players' with the actual table name).
  3. Press F9. You see the first 10 rows in a grid.
  4. Click any column header to sort. Click "Filter" in the toolbar to find a specific player by name.
  5. This is how you answer "show me player John's data" in 5 seconds.
Recap

Two tests, in order

Run each test and confirm before trusting yourself with UPDATE queries.

  1. Test 1, connection alive.
    • Action: open HeidiSQL, double-click your session, click the Query tab, run SELECT 1;.
    • Expected: a grid appears with one row and the value "1".
    • How to verify: no error popup, grid has data.
  2. Test 2, real table browse.
    • Action: double-click a table in the left panel, then click the "Data" tab.
    • Expected: rows appear in a grid. Column headers are the actual column names.
    • How to verify: the number of rows matches what you expect (e.g. 50 players = about 50 rows).
Why does HeidiSQL show a warning when I try to edit a cell?

HeidiSQL knows that editing a cell issues an UPDATE behind the scenes. If you accidentally click into a cell and type something, you are changing live data. The warning is a safety net. Always double-check which row and column you are editing before confirming.
