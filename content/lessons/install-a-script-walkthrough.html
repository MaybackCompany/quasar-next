What this is
A repeatable process for installing any FiveM script onto your server.
Example: download, extract, place, configure, restart, test.
Where to find it
Scripts come from forums, GitHub, or stores. They all end up in resources/.
Example: download a ZIP from forum.cfx.re, extract into resources/[scripts]/.
How to use it
Four universal steps. No script is special. They all follow the same pattern.
Example: every script needs a folder, a manifest, an ensure line, and a restart.
What to do now
Follow the four steps with the example script below.
Example: create the folder, write the two files, ensure it, test it.

Outcome

You will install a headlight toggle script and confirm it works by pressing a key in a vehicle.

Why it matters

Every server owner installs dozens of scripts. If you get the install process wrong, scripts break silently, crash your server, or never run at all. A reliable install habit prevents all of that.

Vocabulary

fxmanifest.lua
The identity card of a resource. It tells FiveM the resource name, version, and which files to load.
Example: fxmanifest.lua with fx_version 'cerulean' and client_script 'client.lua'.
Resource folder
A directory under resources/ that contains one fxmanifest.lua and its script files.
Example: resources/[scripts]/qu_headlights/ is a complete resource folder.
Ensure
The directive in server.cfg that starts a resource. It loads the manifest, then runs the files listed inside.
Example: ensure qu_headlights tells FiveM to start that resource.

Setup

Resource
qu_headlights
Side
client
File path
resources/[scripts]/qu_headlights/

ensure qu_headlights

Minimal example

Expected output: Pressing H in a vehicle toggles headlights on and off. F8 console prints [qu_headlights] turned on and [qu_headlights] turned off.

Checklist

  1. Create the folder resources/[scripts]/qu_headlights/.
  2. Create fxmanifest.lua inside that folder and paste the manifest above.
  3. Create client.lua inside the same folder and paste the script above.
  4. Add ensure qu_headlights to your server.cfg.
  5. Restart the server and join with a client.
  6. Spawn a vehicle, press H, and confirm the headlights toggle.

Exercise

Change the key from H (74) to G (47). Find the key code for G in the FiveM control reference, swap it in IsControlJustPressed, restart the resource, and confirm G now toggles the lights.

Expected output: Pressing G toggles headlights. H no longer does anything.

Common failures

  • Couldn't find resource qu_headlights means the folder name does not match the ensure line, or the folder is not inside resources/.
  • Resource qu_headlights has no scripts means the fxmanifest.lua is missing from the resource folder.
  • attempt to call a nil value (global 'IsControlJustPressed') means the script is running on the server side. Move it to client_script in the manifest.
  • Nothing happens when I press H means the resource started but the key check thread is not running. Confirm client_script 'client.lua' is in the manifest and you restarted the resource.

Refactor

Add a config section at the top of client.lua: local TOGGLE_KEY = 74. Use TOGGLE_KEY in the thread instead of the magic number. Now you can change the key in one place without hunting through the code.
