The mental model: code at a fork in the road

Picture your script as a person walking down a path. Up to now that path has been one long straight line: every line runs, every time, in order. An if statement is a fork. You hand Lua a question, Lua looks at the answer, and Lua walks down one branch or another.

The question has to be something Lua can answer with a yes or a no. "Is this player's level at least 50?", yes or no. "Is this name equal to 'admin'?", yes or no. The technical word for "yes" is true, and for "no" is false. Lua calls these booleans, and you'll meet them constantly from here on.

Here's the shape you're aiming for. Read it out loud before we build it piece by piece:

FiveM note

Every gameplay decision you'll ever write on a FiveM server is a chain of if statements. Is the player a cop? Is their weapon on the allowed list? Is their bank balance above the price? Is this vehicle already spawned? Get comfortable here and you've unlocked 80% of scripting logic.

What each piece means
  • if: the keyword that starts the question.
  • playerLevel >= 10: the condition. Lua evaluates this to true or false.
  • then: a required word that says "here comes the code to run if the answer is true". Forget it and Lua errors out immediately.
  • print('Regular, have fun'): the body of the branch. Indent it four spaces so you can see at a glance it's inside the block.
  • end: closes the if. Every if you open needs exactly one end to close it. Count them like parentheses.

Mental check

Exactly one branch runs. Not zero, not both. If the condition is true, the if body runs and Lua skips the else. If it's false, Lua skips the if body and runs the else. This is called mutually exclusive: make peace with the term, you'll see it everywhere.

Step 4

Chain decisions with elseif

Outcome: a three-tier greeting, VIP, regular, newbie, picked by level.

Two branches are rarely enough. Most gameplay logic has three, four, or five possible answers. The word for that is elseif: one word, no space. It's how you stack conditions in order:

This is the final shape of the resource. Save, ensure qu_gate, and confirm you see Regular, have fun. Now do the real exercise, change playerLevel and reload three times:

  • playerLevel = 80 → prints VIP, welcome back
  • playerLevel = 12 → prints Regular, have fun
  • playerLevel = 3 → prints Newbie, read the rules first

Three different paths, one block, still only one end. Lua checks the conditions top to bottom and runs the first one that's true. Everything below it is ignored, even if it would also be true. Order matters. That's why >= 50 comes before >= 10: a level 80 player matches both, and you want the VIP branch to win.

Why elseif beats nested if

You could write three separate if blocks, or nest one inside another, both technically work, both read terribly, and both let bugs hide. elseif says loud and clear: "these are the same decision with different outcomes." Flat and readable beats clever and nested, always.

Pro move

Write your conditions most-specific-first. >= 50, then >= 10, then else catches everyone under 10. If you flipped the order, every VIP in the world would fall into the >= 10 branch and you'd never know why, because technically the code still works.

Step 5

Combine conditions with and, or, not

Outcome: you can ask Lua more than one question at the same time.

Real conditions rarely come alone. "Is the player VIP and online?" "Is the car a bike or a boat?" "Is the user not banned?" Lua spells these as English words, not symbols, this throws everyone coming from C or JavaScript, where it's &&, ||, and !.

  • and: true only when both sides are true.
  • or: true when at least one side is true.
  • not: flips a value: true becomes false, false becomes true.

Sneak these into your qu_gate file and experiment. Flip hasVIP to false. Change level. Watch which branches fire. The operators read like English for a reason, lean on that when you're assembling a condition in your head.

Paste that into server.lua, reload, and read the console. The first two lines print. The third never fires. That's the rule in action. nil (Lua's word for "nothing is here") and the literal false are the only gates that block an if. Everything else walks straight through.

Practical consequence

If you want to check "is this number zero?", you must write if x == 0 then. Writing if x then will always enter the branch when x is a number, no matter the value. You'll meet this bug in your first week if you skip this step.

Common mistakes & what the console tells you

Error: syntax error near '='

You wrote if playerLevel = 10 then. Single = is for assignment, putting a value in a variable. Double == is for comparison, asking if two values match. Inside a condition, it's always ==.

Error: unexpected symbol near '!'

You wrote if name != 'admin' then. Lua's "not equal" is ~=, not !=. No other mainstream language uses this symbol, it's pure Lua. Burn it into your fingers now.

Error: 'end' expected (to close 'if' at line X)

You opened an if and forgot the end. Every if needs exactly one end to close it, the else and elseif keywords don't count. Indent your code consistently and it becomes obvious which end belongs to which if.

No error, but the wrong branch keeps firing

Almost always one of two things: (1) you wrote if count then expecting it to be false when count is 0, remember, 0 is truthy in Lua, or (2) your elseif conditions are in the wrong order, so a broad condition at the top is swallowing the specific cases you meant to reach later.

Cheat sheet: keep this open next lesson

if cond then ... end
Run the body only when cond is true (or truthy).
if ... then ... else ... end
Two-branch decision. Exactly one branch runs.
elseif
Chain more conditions. One word, no space. First true wins.
== ~= < > <= >=
Comparison operators. Return true or false.
and · or · not
Combine or flip conditions. Lua uses English words, not && || !.
Falsy in Lua
Only false and nil. 0 and '' are truthy.
= vs ==
Single equals assigns. Double equals compares. Inside an if, always double.

What you learned

  • if ... then ... end lets your code make decisions based on a condition
  • elseif chains multiple conditions and only the first true branch runs
  • The six comparison operators: ==, ~=, <, >, <=, >=
  • and, or, and not let you combine or flip conditions
  • In Lua, only false and nil are falsy -- 0 and empty strings are truthy (this is different from most other languages)
