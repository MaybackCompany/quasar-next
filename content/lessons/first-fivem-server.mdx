## Build it

<StepGroup>
  <Step num={1} title="Download FiveM server artifacts" outcome="FXServer.exe is on your computer.">
    An **artifact** is the FXServer build you download. It is the server program itself. Go to the official Cfx.re Windows artifacts page:

    ```text
https://runtime.fivem.net/artifacts/fivem/build_server_windows/master/
    ```

    Download the **newest build**. It sits at the very **top** of the list. Ignore the green "recommended" row further down. That one is older and lags behind. You want the current build. Click the top build, then download its zip and extract it to:

    ```text
C:\FXServer\
    ```

    Expected contents: `FXServer.exe`, a `citizen/` folder, and `.dll` files.

    **Use the top build on a fresh install.** The only reason to drop to the build directly below it is if a brand-new build breaks. You do not pin or freeze a build yet. That is a later production task, once your server is live and public.

    <ExpectedOutput>
      Double-clicking `FXServer.exe` opens a console window, then txAdmin starts.
    </ExpectedOutput>
  </Step>

  <Step num={2} title="Get your license key" outcome="You have the one mandatory key.">
    Go to https://portal.cfx.re/ (the Cfx.re Portal, the current home for license keys) → **Servers** → **Create Server**. Enter a name. Copy the generated key.

    The key is a single string that starts with `cfxk_`, like this:

    ```text
cfxk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    ```

    Paste that value verbatim. There is no `licensekey:` prefix. In `server.cfg` it becomes the line `sv_licenseKey "cfxk_..."`, which you write in the next step.

    The key is required even for a local server only you can reach. Without it, the server won't start.
  </Step>

  <Step num={3} title="Write a minimal server.cfg" outcome="The server has a name, endpoints, and a license.">
    `server.cfg` is the plain-text file your server reads on startup. Create it in `C:\FXServer\`:

    ```text server.cfg
# How players reach your server
endpoint_add_tcp "0.0.0.0:30120"
endpoint_add_udp "0.0.0.0:30120"

# Your license key from portal.cfx.re (paste the cfxk_ value, no prefix)
sv_licenseKey "cfxk_your_key_here"

# What players see in the server name
sv_hostname "My First FiveM Server"

# OneSync: the modern networking layer, on by default
onesync on

# Maximum players
sv_maxclients 32
    ```

    **OneSync** is the networking layer that lets a FiveM server hold more players and sync more entities. `onesync on` is the modern default. A plain server caps at 32 slots; turning OneSync on lets you go above that, up to 2048. Leave it on. Only `onesync legacy` and `onesync off` are the old modes.

    Six settings. That's it. You don't need tags, Discord links, or resource lists yet.
  </Step>

  <Step num={4} title="Start the server" outcome="The server console shows it's running.">
    Double-click `FXServer.exe`. A console window opens. You'll see:

    - "Validating license key" then "License key OK"
    - txAdmin (the browser dashboard bundled inside FXServer) starting at `localhost:40120`
    - a "Server ready" message after about 30 seconds

    Don't worry about txAdmin right now. Just watch the console until it says the server is ready.

    <ExpectedOutput>
      Console prints: `[txAdmin] Server is ready.`
    </ExpectedOutput>
  </Step>

  <Step num={5} title="Join your server" outcome="You're standing in Los Santos on your own server.">
    Open the FiveM client. Press **F8** to open the in-game console (the command box built into the FiveM client). Type:

    ```text
connect 127.0.0.1:30120
    ```

    `127.0.0.1` is your own PC. If you want friends on your local network to join, use your PC's local IP instead:

    ```text
connect 192.168.x.x:30120
    ```

    Find your local IP with `ipconfig` in Command Prompt.

    Your local server will not show up in the public FiveM server browser list. That is normal. You reach it directly with the `connect` command above. Getting listed publicly is a later step that needs port forwarding and extra config.

    <ExpectedOutput>
      You load into Los Santos. The server name appears in the bottom-right corner. You did it.
    </ExpectedOutput>
  </Step>
</StepGroup>

## If something went wrong
