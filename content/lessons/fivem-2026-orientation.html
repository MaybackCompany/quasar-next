FiveM was acquired by Rockstar in 2025. By 2026 the platform has changed. This lesson maps what is different, what stayed the same, and how to build your server on solid ground right now.

Checklist

  2. Create server.cfg without a license key for local dev.
  3. Set sv_enforceGameBuild to 3407 (current as of 2026).

Exercise

Change sv_enforceGameBuild to an older build number like 2944, restart, and observe the error. Then change it back.

Expected output: With a wrong build, clients see a version mismatch. With the correct build, they connect normally.

Common failures

  • Failed to create server license means your Rockstar account is not linked to FiveM in the launcher.
  • Game build mismatch means sv_enforceGameBuild is set to a version you do not have installed.
  • Could not start resource chat means the chat resource is missing from your artifact or not started in server.cfg.
  • HTTP 403 on server list means you are trying to list a localhost server publicly, which is not allowed.

Refactor

Move your server config into multiple .cfg files loaded with exec so each subsystem (resources, bans, endpoints) lives in its own file.

Cheat sheet

FiveM 2026 at a glance

  • Cfx.re is Rockstar. The launcher, forums, and account system changed. Lua scripts and natives did not.
  • Qbox and ox_core are active. If you start fresh in 2026, pick one of these two frameworks.
  • ESX Legacy and QBCore still work. They just do not receive new features. Existing servers are fine.

The big idea, in one minute

FiveM in 2026 is the same game modding platform it always was, but the front door moved. The old keymaster, the old forums, and the old launcher are gone. The good news: every Lua script you wrote in 2023 still runs. The natives are the same. The resource system is the same. You just authenticate through Rockstar instead of Cfx.re.

Think of a restaurant that changed ownership.

The restaurant was bought by a larger company. The menu, the kitchen, and the recipes stayed the same. You still order the same food. But now you pay at a different register, the website changed, and the phone number is new. FiveM in 2026 is that restaurant.

Your Lua resources are the recipes. The Rockstar launcher is the new register. The docs at docs.fivem.net are the new menu. Everything else works exactly as it did before.

Old way and new way, side by side

Most returning developers try the old workflow first. That is normal. The new workflow is simpler, with fewer steps.

Old way (2023 and earlier)

You visited keymaster.fivem.net, generated a license key, pasted it into server.cfg, and hoped the key was still valid.

The key had to be renewed. The keymaster site sometimes went down. If you lost the key, you had to generate a new one and update every config.

New way (2026)

Local dev: no key at all. Production: your Rockstar account handles it. No more keymaster.

The license is tied to your Rockstar account. No more separate website. No more key expiration. One less thing to break.

Before you start

If any box is unchecked, take five minutes to fix it first.

Step 1

Download the latest server artifact

You need the server binary, not the client. The client is what players use to connect.

  1. Visit https://runtime.fivem.net/artifacts/fivem/build_proot_linux/master/.
  2. Download the latest fx.tar.xz file (Linux) or the Windows server build.
  3. Extract it to a folder called fxserver on your machine.
  4. You should see run.sh (Linux) or FXServer.exe (Windows) inside.
  5. You are done with Step 1.
Step 2

Write a minimal server.cfg

The server reads this file on startup. Without it, nothing works.

  1. In your fxserver folder, create a file named server.cfg.
  2. Paste the minimal config from the "Minimal example" section above.
  3. sv_licenseKey "none" tells FiveM this is a local dev server.
  4. sv_enforceGameBuild 3407 locks the server to a specific GTA V update so all players match.
  5. Save the file. You are done with Step 2.
Step 3

Start the server and verify

If the server starts without screaming, you are in good shape.

  1. Run ./run.sh (Linux/Mac) or FXServer.exe (Windows).
  2. Watch the console. You should see lines like Started resource chat and Started resource spawnmanager.
  3. Look for Authenticated with Cfx.re or License key not set (both are fine for local dev).
  4. Open the Rockstar Games Launcher, select FiveM, and click Play.
  5. Press F8, type connect localhost, press Enter.
  6. You should spawn in Los Santos. If so, the server works. You are done with Step 3.
Recap

Three checks, in order

Run each check before you move on. If one fails, fix it now.

  1. Check 1, the server starts.
    • Action: run ./run.sh or FXServer.exe.
    • Expected: console prints resource start messages with no red error lines.
    • How to verify: look for Started resource lines in the console.
  2. Check 2, you can connect.
    • Action: open FiveM, press F8, type connect localhost.
    • Expected: you see the GTA V loading screen, then spawn in the world.
    • How to verify: you are standing on the street and can move.
  3. Check 3, chat works.
    • Action: press T to open chat, type hello, press Enter.
    • Expected: your message appears above your head and in the chat box.
    • How to verify: your message is visible. No errors in the console.
Why do we still use sv_licenseKey "none" in 2026?

Because local development servers do not need a public license. The "none" value tells FiveM to skip the license check entirely. For a public server, you link your Rockstar account in the launcher instead.

The rules you just followed

  • You downloaded the server artifact from the official source, not a random GitHub repo.
  • You used sv_licenseKey "none" for local dev, skipping the old keymaster entirely.
  • You set sv_enforceGameBuild to match your installed GTA V version.
  • You started the four built-in resources (mapmanager, chat, spawnmanager, sessionmanager) that every server needs.
  • You connected through the official launcher, not a third-party client.
