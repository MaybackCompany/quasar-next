Why it matters

Every interaction in FiveM — opening doors, picking up items, starting jobs — begins with input detection. If you cannot read and block controls, your scripts will fight the game's native actions and confuse players.

Vocabulary

Control ID
A number that maps to a keyboard key or gamepad button inside GTA V.
Example: 38 is E, 86 is the horn in a vehicle.
Just pressed
The frame when a key transitions from up to down. Fires once per press.
Example: IsControlJustPressed(0, 38) returns true for exactly one frame.
Pressed (held)
The key is currently held down. Fires every frame while held.
Example: IsControlPressed(0, 21) returns true every frame while Shift is down.

Common failures

  • attempt to call a nil value (global 'IsControlJustPressed') means the script is on the server side. Input detection is client-only.
  • Horn still works after blocking it: DisableControlAction must be called every frame while the block is active. A single call is not enough.
  • Shift+E never fires: IsControlJustPressed only returns true for one frame. If you check Shift after E is pressed, the player may have already released Shift. Use IsControlPressed for modifiers.
  • Wrong key fires: control ID 38 is E for keyboard but may be something else on a gamepad. Test with keyboard first, then add gamepad support later.

Refactor

Extract the key check into isKeyJustPressed(controlId) and the block logic into blockControlFor(controlId, milliseconds) so any key can be wired up with two lines.

Cheat sheet

Control detection rules

  • IsControlJustPressed. Use for single-press actions like opening a menu or triggering a command.
  • IsControlPressed. Use for held checks like sprint modifiers or throttle inputs.
  • DisableControlAction. Call every frame while blocking. One call = one frame of blocking.

The big idea, in one minute

GTA V processes keyboard input 60 times per second. Every frame, the game asks: "Is E down right now? Did it just go down this frame?" Your script reads those answers and decides whether the game gets the input or your script takes it. IsControlJustPressed is the "just this frame" check. IsControlPressed is the "still held" check. DisableControlAction is the "the game does not get this one" command.

Think of a security guard at a door.

Sixty times a second, the guard looks at the door handle. If someone just turned it (IsControlJustPressed), the guard checks their ID and either lets them in or blocks the door (DisableControlAction). If someone is still holding the handle (IsControlPressed), the guard keeps the door blocked until they let go.

Your input handler is the security guard. Every frame, it checks, decides, and blocks. If you stop checking for one frame, the game's default behavior sneaks through.

Bad way and good way, side by side

Most beginners block a control once and wonder why it still works. The good version calls DisableControlAction every frame.

Don't do this

Blocking a control once does nothing. It must be blocked every frame.

The horn flickers off for 1/60th of a second and then comes back. The player never notices the block.

Do this

Block the control inside a loop for a set duration.

The DisableControlAction call runs 60 times per second for five full seconds. The horn is completely suppressed. The game never sees the E press during that window.

Step 1

Create the resource and the loop

Every input system starts with a loop that runs every frame.

  1. Create the qu_input resource folder with fxmanifest.lua and client.lua.
  2. In client.lua, write a Citizen.CreateThread with while true do ... Wait(0) end.
  3. Add a placeholder print inside the loop to confirm it runs.
  4. Start the server and check F8. You should see the print line scrolling every frame.
  5. Remove the placeholder print. The loop is ready for control checks.
Step 2

Detect a single key press

IsControlJustPressed fires exactly once per key press.

  1. Inside the loop, add if IsControlJustPressed(0, 38) then (38 is E).
  2. Inside the if-block, call SetNotificationTextEntry('STRING') and DrawNotification(false, true).
  3. Join the server and press E. You should see the notification appear exactly once per press.
  4. Hold E. The notification appears only once because it is "just pressed", not "held".
  5. Save the file. Step 2 is done.
Step 3

Detect a modifier key (Shift+E)

Use IsControlPressed for modifiers because the player may hold Shift across multiple frames.

  1. Inside the E press block, add a nested if IsControlPressed(0, 21) then (21 is Shift).
  2. Show a different notification for the Shift+E combo.
  3. Use an else branch for the plain E press.
  4. Press E normally, then hold Shift and press E. Both notifications should appear correctly.
  5. If both notifications appear at once, double-check that the Shift check is inside the E-just-pressed block, not outside it.
Step 4

Block a native control for a duration

Blocking requires a timer variable and a per-frame disable call.

  1. Declare local blockHorn = false and local blockTimer = 0 at the top of the file.
  2. When E is pressed, set blockHorn = true and blockTimer = GetGameTimer() + 5000.
  3. After the E-press block, add a separate if blockHorn check that calls DisableControlAction(0, 86, true).
  4. Add a release check: if blockHorn and GetGameTimer() > blockTimer then blockHorn = false end.
  5. Get in a vehicle, press E, then immediately try to honk. Nothing happens. Wait five seconds and try again. The horn works.
Step 5

Test all three paths and verify

Run each input combination and confirm the behavior.

  1. Press E on foot. Notification: "E pressed". F8: [QUIET][input] E pressed.
  2. Hold Shift and press E. Notification: "Shift+E pressed". F8 confirms.
  3. Get in a vehicle. Press E. Immediately try to honk. No horn. Wait five seconds. Horn returns.
  4. Press E while the horn is already blocked. The timer resets to five seconds from the new press.
  5. All three paths are verified. The lesson is complete.

Why does the block timer reset on every E press instead of stacking?

Because blockTimer = GetGameTimer() + 5000 overwrites the previous timer value. Each press extends the block by five seconds from that moment. If you want a fixed total duration, check if not blockHorn then before setting the timer.

The rules you just followed

  • Input detection runs on the client, inside a per-frame loop. The server never reads keyboard state.
  • IsControlJustPressed fires on the frame the key goes down. Use it for single actions.
  • IsControlPressed returns true every frame the key is held. Use it for modifiers and continuous actions.
  • DisableControlAction must be called every frame while blocking. A single call blocks exactly one frame.
  • Control IDs are numbers. 38 = E, 21 = Shift, 86 = vehicle horn. The full list is in the FiveM controls reference.
