Outcome

Why it matters

A misconfigured server.cfg causes crashes, exposes your server to abuse, or prevents players from connecting. Every server owner must know this file cold.

Vocabulary

ConVar
A console variable that controls server behavior.
Example: sv_maxclients is a ConVar that sets the player cap.
Endpoint
The network address your server listens on.
Example: endpoint_add_tcp "0.0.0.0:30120" opens the port.
License key
A unique token that identifies your server to Cfx.re.
Example: sv_licenseKey "cfxk_xxxx" is required for public servers.

Checklist

  1. Open your server-data folder and find server.cfg.
  2. Copy the annotated template above into a new file called server-annotated.cfg.
  3. Read every comment next to each directive.
  4. Replace sv_licenseKey with your real key from keymaster.fivem.net.
  5. Replace sv_hostname with a name you like.
  6. Restart your server and confirm the new hostname appears in the console.

Exercise

Change sv_hostname to "Annotated Server Complete", then restart and verify the new name appears in your server console on the line that reads Server started.

Expected output: The console prints Server started with name 'Annotated Server Complete'.

Common failures

  • Could not parse server.cfg means a line has a typo or a missing quote. Check your last edit.
  • License key is not valid means the key is missing, expired, or pasted with extra spaces.
  • Port 30120 is already in use means another server or program is on that port. Change the port or stop the other process.
  • endpoint_add_tcp failed means the IP address does not exist on your machine. Use 0.0.0.0 unless you have a specific reason.

Refactor

Group your custom ConVars under one clearly labeled section header. Move all ensure lines into a single block near the bottom so you can scan your resource list in one glance.
