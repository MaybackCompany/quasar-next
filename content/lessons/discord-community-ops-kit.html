Common failures

  • Bot token invalid means you copied the wrong token or regenerated it without updating txAdmin.
  • Bot missing permissions means the bot invite did not include the required OAuth2 scopes (bot + applications.commands).
  • Webhook not sending means the channel ID is wrong or the webhook URL was revoked.
  • Whitelist rejects everyone means you enabled whitelist before assigning the Approved role to anyone.

Refactor

Replace raw webhook URLs with a single discord.lua utility that all your resources import, so you change the webhook URL in one place.

Cheat sheet

Discord channel template

  • #📢 announcements — Staff-only posts. Server updates, events, maintenance windows. One post per week minimum.
  • #💬 server-chat — In-game chat relay. Every in-game message appears here. Players on Discord can type here to talk in-game.
  • #📋 whitelist — Bot-only channel. Players type /apply and the bot assigns the Pending role. Staff review and approve.
  • #🛠️ support — Players ask for help. Staff or community helpers answer. Keep this channel noise-free.
  • #📊 server-status — Bot posts player count every 5 minutes. Shows online/offline status and current queue.

The big idea, in one minute

A FiveM community lives in two places: the game server and the Discord server. If they do not talk to each other, you lose players. The player who alt-tabs to Discord should see the same community they left in-game. The player who is at work should see the server is alive and full of people.

Think of a restaurant with a waiting area.

The game server is the dining room. Discord is the waiting area where people hang out before a table opens. If the waiting area is dark, empty, and has no menu — nobody waits. They leave and find another restaurant.

Your Discord is the waiting area. Keep it lit. Show the menu (server rules, features). Let people see who is eating (player list, chat relay). Make it easy to get a table (whitelist bot).

Dead Discord and alive Discord, side by side

Most server owners make a Discord, invite a bot, and never touch it again. That is a dead Discord. An alive Discord has traffic, automation, and purpose.

Dead Discord

One #general channel, no bots, no chat relay, no whitelist. Players join, see silence, and leave.

Players join the Discord expecting activity. They find a ghost town. They never launch FiveM because they assume the server is also dead.

Alive Discord

Five focused channels, chat relay, status bot, auto-whitelist. Activity is visible even when the server is empty.

New players join and immediately see activity. The status channel shows the server is alive. The chat relay proves people are playing. They apply for whitelist because it looks worth joining.

Before you start

If any box is unchecked, take five minutes to fix it first.

Step 1

Create your Discord server

Start with a clean, purpose-built server. Do not reuse your personal friend group server.

  1. In Discord, click the plus icon in the server list. Choose "Create My Own".
  2. Name it after your FiveM server. Example: "Los Santos Roleplay".
  3. Create the five channels listed in the cheat sheet above.
  4. Create three roles: @Staff, @Approved, and @Pending.
  5. Set channel permissions: only Staff can post in #announcements. Everyone can read #server-chat and #server-status. Only Approved and Staff can see voice channels.
Step 2

Create and invite your Discord bot

One bot, created by you, that txAdmin will use to bridge Discord and FiveM.

  1. Go to https://discord.com/developers/applications and click "New Application".
  2. Name it "MyServer Bot" or similar. Go to the Bot tab, click "Add Bot".
  3. Under Privileged Gateway Intents, enable SERVER MEMBERS INTENT and MESSAGE CONTENT INTENT. Save.
  4. Click "Reset Token" and copy the token. Paste it somewhere safe. You will need it in Step 3.
  5. Go to OAuth2 > URL Generator. Select scopes: "bot" and "applications.commands". Select permissions: Administrator.
  6. Open the generated URL in a browser. Invite the bot to your Discord server.
  7. You should see the bot appear offline in your member list. It connects when txAdmin starts.
Step 3

Link txAdmin to Discord

One panel, one token, and your server wakes up on Discord.

  1. Open txAdmin in your browser (usually http://your-server-ip:40120).
  2. Go to Settings > Discord Bot. Paste the bot token from Step 2.
  3. Set the guild ID to your Discord server's ID (right-click the server icon > Copy ID).
  4. Enable "Chat Relay" and select your #server-chat channel.
  5. Enable "Whitelist" and set it to role-based. Choose the @Approved role.
  6. Enable "Player Join/Leave Notifications" and point them to #server-status.
  7. Click Save. txAdmin restarts the bot. You should see it come online in Discord.
Step 4

Test the full loop

Confirm that every piece talks to every other piece.

  1. In Discord #server-chat, type "hello from Discord". It should appear in FiveM in-game chat.
  2. In FiveM, type "hello from game". It should appear in the Discord #server-chat channel.
  3. Have a friend without the @Approved role try to connect. They should be rejected with a message pointing them to Discord.
  4. Give your friend the @Approved role. They should now be able to connect.
  5. Kill your FiveM server. The bot in Discord should show the server as offline within 30 seconds.
  6. Restart the server. The bot should show it online again. Player count updates should resume.
Recap

Three tests, in order

Run each test, look at what happens, and confirm the expected behavior. If something does not match, fix it before moving on.

  1. Test 1, chat relay both ways.
    • Action: type in Discord #server-chat, then type in FiveM in-game chat.
    • Expected: messages appear on both sides within 2 seconds.
    • How to verify: open both windows side by side and watch the messages flow.
  2. Test 2, whitelist blocks and allows.
    • Action: try to connect without the @Approved role, then with it.
    • Expected: rejected without role, allowed with role.
    • How to verify: the rejection message tells the player to visit Discord. The acceptance lets them spawn.
  3. Test 3, status updates on server restart.
    • Action: stop the FiveM server, wait 30 seconds, start it again.
    • Expected: #server-status shows offline, then online again.
    • How to verify: the bot's status message updates with player count within 1 minute of restart.
Why use role-based whitelist instead of a Lua script?

Because Discord handles identity. A Lua whitelist script means you store passwords or Steam IDs in a database that you must secure, back up, and build a UI for. txAdmin's Discord whitelist uses Discord's identity system for free. If someone leaves your Discord, they lose access automatically.

The rules you just followed

  • One Discord server per FiveM server. Do not mix communities.
  • Chat relay goes both ways. Discord players and in-game players are the same community.
  • Whitelist via Discord roles, not a manual list. Roles are auditable and revocable.
  • Server status is public. Hiding player count makes your server look empty.
  • Support channel is separate from chat relay. Do not mix player help with casual conversation.
