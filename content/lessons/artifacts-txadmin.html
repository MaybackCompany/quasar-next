Quasar Academy › Track 0 › Step 03

Install pinned artifacts and choose a recipe

Download an official FXServer artifact, record the build you used, open FXServer.exe, let txAdmin guide setup, choose a Popular Recipe, and keep a rollback note if the UI or build changes later.

You'll do
Install an official artifact, record the build, open txAdmin, choose QBCore or ESX, paste your key, and start the server.
Time
~25 minutes. Recipe downloads can be slow.
You need
Step 02 done. MariaDB running. license-key.txt ready.
Result
A running FiveM server you can join with connect 127.0.0.1:30120.

Before you start

What you are installing

Artifact
The downloadable FXServer build. Use the latest one from the official list.
FXServer
The program that runs the FiveM server.
txAdmin
The browser dashboard that opens when FXServer starts.
Recipe
A starter install plan. It downloads resources and writes a first server.cfg.
Step 1

Download and pin a FiveM artifact

Outcome: server.7z is downloaded and the build number is recorded.

  1. Open the official Windows artifacts page.
  2. For a brand-new learning server, click a current official build near the top of the list.
  3. Download server.7z.
  4. Create C:\FXServer\artifact-pin.txt.
  5. Copy the artifact build number, download date, and source URL into that file.
Use official, not random

For a fresh install, use an official build from the Cfx.re artifact page. Do not use Discord mirrors, old YouTube links, or random upload sites. For an existing server, record the known-good build before updating.

Step 2

Extract and open FXServer

Outcome: txAdmin opens in your browser.

  1. Create C:\FXServer\server.
  2. Extract server.7z into that folder.
  3. Open C:\FXServer\server\FXServer.exe.
  4. Allow Windows Firewall if it asks.
  5. Wait for the browser to open http://localhost:40120.

Expected outcome

You see txAdmin with a PIN or setup screen. Keep the black FXServer console open.

Visual check

The browser shows the txAdmin setup or PIN screen, and the black FXServer console stays open behind it.

Step 3

Claim txAdmin

Outcome: txAdmin is linked to your Cfx.re account.

  1. Follow the txAdmin PIN screen.
  2. Sign in with your Cfx.re account.
  3. Authorize txAdmin.
  4. Create the local txAdmin password.
  5. Name the server profile.
Two logins exist

Cfx.re is your global identity. txAdmin password is only for this dashboard.

Recap 1

txAdmin is open?

You should be inside txAdmin setup, not reading a console-only screen.

Step 4

Choose Popular Recipes

Outcome: you choose the framework recipe intentionally.

  1. When txAdmin asks how to set up the server, choose Popular Recipes.
  2. Choose QBCore Framework if you are new.
  3. Choose ESX Legacy only if you already need ESX scripts.
  4. Do not choose Qbox unless you are advanced.
Beginner recommendation

Pick QBCore for Track 0. It has current docs, many examples, and a large resource ecosystem.

Why not Qbox yet?

Qbox expects comfort with code, ox dependencies, migrations, bridges, and stricter patterns. Learn the server first.

If the recipe UI changed

Use stable labels instead of exact button position. Look for setup type, template, or recipe selection. The beginner target is still a QBCore or ESX starter recipe. If txAdmin no longer shows Popular Recipes, open the official txAdmin setup docs from the References section and follow the current recipe path.

Step 5

Run the recipe

Outcome: txAdmin downloads resources and writes your server files.

  1. Pick the install folder txAdmin suggests.
  2. Paste your Cfx.re license key when asked.
  3. Set the database name to fivem if the recipe asks.
  4. Let txAdmin finish every recipe step.
  5. Click Save & Run Server.

Expected outcome

txAdmin shows the server online. The live console prints resources starting.

Step 6

Connect to the server

Outcome: your FiveM client joins the server.

  1. Open FiveM.
  2. Press F8 to open the client console.
  3. For local testing, type connect 127.0.0.1:30120.
  4. For a VPS, type connect YOUR_SERVER_IP:30120.
  5. Press Enter.
Custom connect comes later

Start with direct IP connect. Later you can use a domain, server listing, or branded connect instructions.

Recap 2

Server is alive?

You can join with F8 connect, txAdmin shows one player online, and the live console logs your connection. Update C:\FXServer\artifact-pin.txt so Known-good after test says yes.

Artifact rollback if a future update breaks

This is not for normal first install. This is for the day you update artifacts and the server stops booting cleanly.

  1. Stop the server from txAdmin.
  2. Rename C:\FXServer\server to C:\FXServer\server_bad_BUILDNUMBER.
  3. Extract the previous known-good server.7z into a new C:\FXServer\server.
  4. Do not delete server-data. Your config and resources live there.
  5. Open C:\FXServer\server\FXServer.exe.
  6. Run status in txAdmin and direct connect once.
  7. Write the rollback result in artifact-pin.txt.

Common mistakes

  • You extracted server.7z into Downloads instead of C:\FXServer\server.
  • You closed the black FXServer console, which stops txAdmin.
  • You picked Qbox before understanding dependencies.
  • You used an old artifact from a video link instead of an official artifact page.
  • You typed the VPS IP without :30120.
The txAdmin screen does not match this lesson

Do not click randomly. Use the stable task names: claim txAdmin, choose setup type, choose recipe/template, paste license key, set database, save and run. If the labels changed, open the current txAdmin setup docs from References and continue only when you know which screen maps to those tasks.

What you learned

  • Artifacts are the FiveM server build.
  • FXServer opens txAdmin.
  • Popular Recipes install QBCore, ESX, or Qbox starter stacks.
  • QBCore is the beginner default.
  • You connect with F8 and connect IP:30120.

Five-minute exercise

Open txAdmin Live Console and type status. Then join with FiveM and run status again.

Expected outcome

The second status output shows your player connected.
