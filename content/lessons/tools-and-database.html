Install tools, account, and database

Keep this simple. Install the tools FiveM owners use every day, create your Cfx.re key, and prepare the database before txAdmin asks for it.

You'll do
Install VS Code, WinRAR, WinSCP, MariaDB, and HeidiSQL. Enable file extensions. Create a Cfx.re account and license key.
Time
~30 minutes. Most time is installers and account email.
You need
Step 01 done, admin rights on Windows, and an email you can check.
Result
You are ready to install the latest FiveM artifacts and pick a txAdmin recipe in Step 03.

Before you start

The toolkit

Each tool has one job. Do not install random packs yet.

VS Code
Your editor for .cfg, .lua, and resource files.
WinRAR
Opens .7z, .zip, and .rar server files.
WinSCP
Moves files between your PC and a VPS later.
MariaDB
The database that stores players, money, vehicles, jobs, and inventory.
HeidiSQL
The visual app you use to see and edit MariaDB safely.
Owner mental model

MariaDB is the database engine. HeidiSQL is the window into that database. oxmysql is the FiveM resource that lets scripts talk to MariaDB.

OS variants: what changes by machine

The Track 0 practice server is easiest on Windows because the FiveM client is Windows-only. You can still use macOS or Linux for planning files, SFTP, and database inspection, but the playable client test must happen from Windows.

Step 1

Install VS Code

Outcome: right-clicking a folder shows Open with Code.

  1. Go to code.visualstudio.com/download.
  2. Download the Windows User Installer.
  3. Run it and accept the defaults.
  4. On Additional Tasks, tick both Open with Code context menu options.
  5. Finish the install.

Expected outcome

File Explorer shows Open with Code when you right-click a folder.

Visual check

Right-click any folder in File Explorer: the context menu lists Open with Code.

Step 2

Install WinRAR and show file extensions

Outcome: you can extract .7z files and see real file names.

  1. Go to win-rar.com.
  2. Install the 64-bit Windows version.
  3. Open File Explorer.
  4. Turn on File name extensions.
  5. Also turn on Hidden items.
Why extensions matter

A broken file often looks like server.cfg.txt. FiveM needs server.cfg. Hidden extensions hide that mistake.

Proof

Create Desktop\proof.cfg. File Explorer must show the full name proof.cfg, not proof.cfg.txt.

Step 3

Install WinSCP

Outcome: WinSCP is ready for VPS file transfers later.

  1. Go to winscp.net/eng/download.php.
  2. Download the installer.
  3. Run it with default settings.
  4. Do not connect to anything yet.

Expected outcome

WinSCP opens to a login screen. Close it for now.

Hosted proof later

When you get hosting, the proof changes from "WinSCP opens" to "WinSCP connects to the SFTP host and shows server.cfg plus resources/."

Recap 1

Local tools ready?

You have VS Code, WinRAR, WinSCP, visible file extensions, and hidden items enabled.

Step 4

Create your Cfx.re account and license key

Outcome: license-key.txt exists on your Desktop.

  1. Create or verify your account at forum.cfx.re.
  2. Go to portal.cfx.re.
  3. Open Server Management and create a new license.
  4. Copy the key into a new VS Code file.
  5. Save it on your Desktop as license-key.txt.
Treat the key like a password

Never paste it in Discord, GitHub, screenshots, or tutorial videos. Revoke it on the portal if it leaks.

Step 5

Install MariaDB

Outcome: MariaDB runs on 127.0.0.1:3306.

  1. Go to mariadb.org/download.
  2. Download the latest Windows x86_64 MSI package.
  3. Run the installer.
  4. Set a root password and save it in db-creds.txt on your Desktop.
  5. Enable UTF8 as the default character set.
  6. Keep port 3306 and install it as a Windows service.

Expected outcome

PowerShell can run mysql --version and print a MariaDB version.

If the service name is different, open Windows Services and look for MariaDB or MySQL. The important proof is Running.

Step 6

Install HeidiSQL and create the database

Outcome: HeidiSQL shows a database named fivem.

  1. Go to heidisql.com/download.php.
  2. Install HeidiSQL.
  3. Create a session named local FiveM.
  4. Use host 127.0.0.1, user root, your MariaDB password, and port 3306.
  5. Create a database named fivem.
  6. Use collation utf8mb4_unicode_ci.

Expected outcome

HeidiSQL opens and the left tree shows fivem.

Connection proof

In HeidiSQL, click the fivem database, then run this query in a query tab:

The result should show fivem. To confirm, HeidiSQL's result grid returns fivem for SELECT DATABASE().

Recap 2

Ready for txAdmin?

Your Desktop has license-key.txt and db-creds.txt. HeidiSQL connects to MariaDB. The fivem database exists.

Framework choice happens in Step 03

txAdmin will show Popular Recipes. For beginners, choose QBCore. Choose ESX only if you already bought ESX-only scripts. Do not choose Qbox until you are comfortable with code, dependencies, migrations, and ox resources.

Common mistakes

  • You created server.cfg.txt because file extensions were hidden.
  • You lost the MariaDB root password.
  • You leaked the Cfx.re license key in a screenshot.
  • HeidiSQL says connection refused because the MariaDB service is stopped.
MariaDB installer says the port is already in use

You probably already have MySQL, XAMPP, Laragon, or another MariaDB running on 3306. Stop the old service or choose one database stack and remove the duplicate. Do not run two database servers unless you know which one FiveM uses.

mysql is not recognized in PowerShell

The service can still be installed. Use Get-Service MariaDB* for the service proof, then use HeidiSQL for the connection proof. You can add MariaDB to PATH later.

HeidiSQL says Access denied for user 'root'

The password is wrong or root login was configured differently. Reopen your saved db-creds.txt. If it is lost on a fresh learning install, reinstall MariaDB and write the password before continuing.

The Cfx.re portal will not create a key

Confirm your Cfx.re account email is verified and that you are signed into the same account in the browser. Try a private window if old cookies are stuck.

What you learned

  • VS Code edits server files.
  • WinRAR extracts FiveM artifacts.
  • WinSCP moves files to a VPS later.
  • MariaDB stores server data.
  • HeidiSQL lets you inspect MariaDB visually.
