Set up Visual Studio Code with Lua extensions, the FiveM native autocomplete, and a workspace tuned for resource development — so you stop guessing native names and start typing them.

Common failures

  • undefined global 'RegisterCommand' — the Lua extension does not know FiveM globals. Add them to Lua.diagnostics.globals in settings.json.
  • No autocomplete for natives — the cfx-natives-ls extension is not installed, or the workspace folder is not the server root.
  • Extension "Lua" shows Lua Language Server crashed — your settings.json has a syntax error. Check for trailing commas.
  • Autocomplete shows wrong parameter count — the native definition file is outdated. Run cfx-natives-ls: Refresh from the command palette.

Refactor

Move the diagnostics globals into a shared .luarc.json file so every folder in the workspace inherits the same FiveM globals without copying settings.

Cheat sheet

VS Code shortcuts for FiveM

  • Ctrl+Shift+P then "Lua: Restart Server" — restart the language server after adding globals.
  • Ctrl+` — open the integrated terminal inside your server folder.
  • Ctrl+P then type @ — jump to a function inside the current file.

The big idea, in one minute

FiveM scripts are small files that call large native functions. There are over six thousand natives. Nobody memorises them all. A good editor remembers them for you. When you type GetEntityCoords, the editor shows you the parameters before you finish typing. That is not a luxury — it is how professional developers work. The alternative is keeping a browser tab open and copy-pasting function names one at a time, which is slow and error-prone.

Think of a mechanic's toolbox.

A mechanic does not memorise every bolt size. They open a drawer, the sockets are laid out in order, and they grab the 10 mm without thinking. VS Code extensions are your socket drawer. The Lua extension is the organisation. The FiveM native extension is the label on each socket. Without them you are digging through a bucket of loose bolts.

Before you start

Step 1

Install VS Code

If you already have VS Code, skip to Step 2.

  1. Go to code.visualstudio.com and download the installer for your operating system.
  2. Run the installer. On Windows, check the box that says "Add to PATH".
  3. Open VS Code. You see a welcome screen with a "Get Started" walkthrough. Close it for now.
Step 2

Install the Lua extension

This extension understands Lua syntax and gives you red underlines for mistakes before you run anything.

  1. Click the Extensions icon on the left sidebar (or press Ctrl+Shift+X).
  2. In the search box, type Lua.
  3. Find the one by sumneko (it is the most popular Lua extension). Click Install.
  4. Wait for the install to finish. A small blue "Installed" label appears.
Step 3

Install the FiveM native extension

This extension knows every FiveM native, its parameters, and which side it runs on.

  1. In the same Extensions panel, search for cfx-natives-ls.
  2. Install the one by overextended.
  3. After installing, open the Command Palette (Ctrl+Shift+P) and type cfx. You should see an entry like "CFX Natives: Refresh". Do not run it yet.
Step 4

Open your server folder and configure globals

VS Code needs to know which folder is your project and that FiveM globals are not mistakes.

  1. Click File → Open Folder and select your FiveM server directory (the one that contains resources/ and server.cfg).
  2. Inside that folder, create a new folder called .vscode (the dot at the start is important).
  3. Inside .vscode, create a file called settings.json.
  4. Paste the JSON from the Minimal Example above. Save the file.
  5. Press Ctrl+Shift+P, type Lua: Restart Server, and press Enter.
Step 5

Test the setup

One native, typed by hand, confirms everything works.

  1. Open any server.lua file inside your resources folder.
  2. On a new line, slowly type SetEntityHealth(. Do not copy-paste.
  3. A tooltip should appear showing the parameters: SetEntityHealth(entity: number, health: number).
  4. Type source on another line. No red squiggle should appear under it.
  5. If both work, your editor is ready for FiveM development.
Recap

Why does VS Code complain about source but not about print?

print is a standard Lua global that the language server already knows. source is a FiveM-only global injected by the server runtime. The language server has no way to know about it unless you list it in Lua.diagnostics.globals.
