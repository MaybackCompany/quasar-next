What txAdmin actually is, in one minute

txAdmin is a web app that wraps your FXServer process. The server runs as a child of txAdmin. txAdmin watches it, restarts it on crash, exposes a dashboard so you don't have to SSH into a console window.

Think of it as the nurse station outside the kitchen. The kitchen (FXServer) does the cooking. txAdmin tells you who's at the table, what's burning, when to send out plates, and lets you yell orders without walking in.

FiveM note: the panels you'll actually use

txAdmin has more than ten side-bar panels. Six matter day-to-day: Live Console, Players, Resources, Recipes, Settings, and the in-game menu (F1). Ignore the rest until you need them.

Step 1

Live Console: the heartbeat

Outcome: you can send a command to your running server and see its output.

  1. Left sidebar → Live Console.
  2. You'll see a black terminal with scrolling text. Every line FXServer prints, you read here.
  3. At the bottom there's an input box. Type status and hit Enter.
  4. The console prints connected players, slot count, uptime. That's your server replying live.
Color codes in the console

White = info. Yellow = warning. Red = error. If you see red, copy the line, search the CFX forum or Discord. 90% of red lines are typos in server.cfg or a bad fxmanifest.lua.

Step 2

Players panel: who's online

Outcome: you can find any connected player, see their identifiers, and kick or ban them in two clicks.

  1. Left sidebar → Players.
  2. Top tab: Online. Lists everyone connected right now (including you, after Step 06).
  3. Click any name → opens the Player Modal.

The Player Modal is where the day-to-day moderation happens:

TabUse it for
InfoTheir identifiers (fivem:, steam:, discord:, license:). Copy these into add_principal when promoting to admin.
HistoryPast warnings, kicks, bans. Useful before a moderation decision.
IDsHardware IDs. For ban evasion checks.
ActionsWarn / Kick / Ban / DM, with a reason field. The reason is shown to the player.
Where to find your own identifier

Connect to your own server (Step 06 covers this). Then in the Players panel, click your own name. The Info tab shows fivem:1234567. That's the value you paste into add_principal identifier.fivem:1234567 group.admin in server.cfg.

Step 3

Resources tab: start, stop, restart

Outcome: you can flip individual resources without restarting the whole server.

  1. Left sidebar → Resources.
  2. Each row = one resource folder. Green dot = running. Grey = stopped. Red = crashed.
  3. Hover a row. You get four buttons:
ButtonWhat it does
StartBoot a stopped resource.
StopShut it down. Players keep playing; just this resource goes quiet.
RestartStop + start. Use after you edit a Lua file.
Refresh ListTop of the page. Re-scans resources/ for new folders. Run after you drop a new resource in.
The dev loop

Edit a script → save → in txAdmin click Restart on that one resource → alt-tab back to FiveM. No server restart. Your changes are live in 1 second.

Step 4

Recipes: the prefab catalog

Outcome: you understand what a recipe is and when to use one (almost never, after Step 03).

A recipe is a YAML file that tells txAdmin: "here is a list of resources to download from GitHub, drop in resources/, and ensure in server.cfg". One click = a working server preset.

  1. Left sidebar → Recipes.
  2. You'll see prefabs: CFX Default FiveM, ESX/QBCore RP starters, drift servers, etc.
  3. Each one runs the same flow you saw in Step 03: download → extract → configure → ensure.
Recipes are one-time scaffolds

Use a recipe to create a new server, not to update an existing one. Running a second recipe on a working server overwrites your server.cfg. You will lose customizations.

Recap 1

You can drive the dashboard

From the Live Console, run status. From the Resources tab, restart any single resource (e.g. chat). From the Players panel, click your server's "system" entry to see what an empty player list looks like. If all three worked, you're driving txAdmin.

The Live Console input is greyed out

Your txAdmin user account doesn't have permission to send commands. Settings → Admin Manager → promote your account to Master Admin. Or log in as the master admin you created in Step 03.

Players panel shows nobody, but I'm in the server

Refresh the page. Players panel updates on a 5-second poll; sometimes the first load misses your join. If still empty, the in-game F8 console will show whether you actually joined.

Step 5

Settings → CFG Editor

Outcome: you can edit server.cfg from the browser, with syntax highlighting and a save button.

  1. Left sidebar → Settings → tab CFG Editor.
  2. The full server.cfg opens in a Monaco editor (the same engine VS Code uses). Edit, then click Save.
  3. txAdmin warns you if you changed something that requires a full Server Restart (top-right button). For most edits, click it.
Use this when you can't reach the file system

If your server runs on a remote VPS or a friend's machine, the CFG Editor is the cleanest way to edit. No SSH, no FileZilla. Just a password-protected web tab.

Step 6

Settings → Game Mode & Privacy

Outcome: you understand the difference between a public, private, and "hidden from list" server.

Two settings to know:

SettingMeaning
Server Privacy → PublicListed in the FiveM in-game server browser. Anyone can find and join.
Server Privacy → HiddenNot listed in the browser. Players join via Direct Connect only. Use during dev.
Game ModeA short label (Roleplay, Drift, Sandbox) shown next to your server in the browser. Helps the right players find you.
Don't list a dev server

While you're learning, set Privacy = Hidden. Once you start advertising in your community, flip to Public. Otherwise random players join, get a buggy alpha experience, and never come back.

Step 7

The in-game menu (F1) & Live Editor

Outcome: you can teleport, spawn vehicles, and change the weather without leaving the game.

  1. Connect to your own server (we'll do this in Step 06). For now: imagine you've joined.
  2. In-game, press F1. The txAdmin Menu opens.
  3. Tools available:
  • Teleport, click any waypoint or coords. Instant.
  • Vehicle spawner, type model name, hit enter, drive away.
  • Weather / Time, force "thunder", "clear", noon, midnight.
  • Player actions, spectate, summon, heal, freeze (admin only).
  • Live Editor, place objects, save the layout to a Lua file. The fastest way to build a static map.
Why this exists

The F1 menu is for development, not players. It's locked to admins by default (group.admin ACE). Players never see it. Use it to test scripts: spawn a car, trigger an event, watch the chat for output.

If yes, you passed.

I clicked F1 and nothing happened

Two reasons: (1) you're not in the server yet, (2) you don't have group.admin rights. Check the add_principal line in server.cfg matches the identifier the Players panel shows for you.

txAdmin is unreachable / browser shows "connection refused"

The FXServer process crashed or the txAdmin port (default 40120) is firewalled. Check the OS console window where you launched FXServer.exe; if it's gone, double-click FXServer.exe again. If the port is busy, change tx.adminPort in server-data/.txAdmin/profiles/default/config.json.

Common mistakes & what they look like

I edited a Lua file but my changes don't show in-game

You forgot to Restart the resource in the Resources tab (or use restart my-resource in the Live Console). FXServer caches the last loaded version. No restart = no reload.

F1 menu doesn't open in-game

You don't have admin rights yet. Run Step 04 again: add_ace group.admin command allow + add_principal identifier.fivem:YOUR-ID group.admin. Restart the server. Re-join.

txAdmin keeps logging me out

Your session expires after 24h by default. Settings → Account → bump the session length. Don't share your password, create a second admin account for friends instead.

Recipes panel won't run, errors out mid-deploy

Almost always a network issue: GitHub rate-limited you, or the recipe references a repo that moved. Re-run after a few minutes. If it persists, switch to manual install (download the resource zip yourself, drop into resources/, add an ensure line).

Cheat sheet: panel reference

Live Console
Live server output + command input. Type status, refresh, restart X.
Players
Online list, click a name for identifiers + Warn/Kick/Ban actions.
Resources
Per-resource Start / Stop / Restart. Use Refresh List after dropping new folders.
Recipes
One-shot prefab installer. Don't run twice on the same server.
Settings → CFG Editor
Edit server.cfg in-browser, with syntax highlighting.
Settings → Game Mode
Label shown in server browser (Roleplay, Drift, etc.).
Settings → Privacy
Public = listed. Hidden = direct-connect only. Set Hidden during dev.
In-game F1
Teleport, spawn vehicles, weather/time, Live Editor. Admin-only.

What you learned

  • txAdmin is a web wrapper around FXServer at http://localhost:40120
  • Six panels do the work: Live Console, Players, Resources, Recipes, Settings, F1 in-game
  • Use the Live Console to send commands; status is your "is it alive" check
  • The Players panel exposes identifiers you copy into add_principal
  • Restart a single resource from the Resources tab, no full server restart needed
  • Set Privacy = Hidden while you're learning; flip to Public when you're ready
  • The in-game F1 menu is a developer tool, not a player feature

Five-minute exercise

Without joining the game yet, do this in txAdmin:

  1. Open the Live Console. Type status → Enter. Read the output.
  2. Open the Resources tab. Click Restart on chat. Watch the Live Console echo the restart.
  3. Open Settings → Privacy. Set it to Hidden. Save.

Expected outcome

Live Console shows status reply with player count + uptime. Console also shows Started resource chat after the Restart. Privacy panel saves with a green toast.

Stretch yourself (optional)

  • In Settings → Admin Manager, create a second admin account for a friend. Test that they can kick but you can still revoke their rights from yours.
  • Open the Live Console. Run refresh, then start scoreboard. Watch what each line prints. You're now using the same commands you put in server.cfg, but live.
  • In Settings → Discord Integration, paste a webhook URL. Restarts and admin actions will post to your Discord channel automatically.
