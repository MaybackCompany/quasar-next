The mental model: boxes that change

Last lesson, a variable was a labelled box with text inside. That picture still works, but we have to extend it today in two directions. First: the box doesn't have to hold text. It can hold a number. It can hold other things too (you'll meet those in later lessons). The label on the box doesn't care what's inside; it points to whatever you last put there.

Second: you can change what's inside. Open a box, throw out the old value, drop a new one in. Lua won't complain. The label stays the same, the contents change. That's re-assignment, and you do it all the time.

Line one creates the box called score and puts 0 inside. Line two reads what's in the box (0), adds 10, and puts the result (10) back in the same box. Notice: no local on line two. You only write local once per box, the first time. After that, the box already exists; you're updating it.

FiveM note

Every job script, every inventory, every vehicle spawner you'll ever write or read is built out of this one pattern: create a value, update a value, print or use a value. Master re-assignment now and you've already read half the code on GitHub.

Why no quotes around 64?
  • '64' with quotes is a string: text that happens to look like a number. You can't do math with it.
  • 64 without quotes is a number: an actual numeric value. You can add, subtract, multiply, divide it.
  • Quotes are the flag that tells Lua "this is text". No quotes means "this is a value, figure out what kind".
Naming convention

Lua variable names use camelCase by community convention, start lowercase, capitalise each new word. maxPlayers, not max_players or MaxPlayers. Pick one style, stick with it, your future self will thank you when searching a 400-line file.

Quick tour of the five number operators
  • +: addition. 10 + 3 gives 13.
  • -: subtraction. 10 - 3 gives 7.
  • *: multiplication. 10 * 3 gives 30.
  • /: division. 10 / 3 gives 3.333... (Lua always hands back a decimal for /).
  • %: remainder. 10 % 3 gives 1 (10 divided by 3 leaves 1 left over).

Numbers can be whole (64) or have a decimal part (1.5, 0.25, 3.14). Lua handles both the same way. No special keyword, no type declaration, you type the digits, Lua figures it out.

Step 3

Glue text together with ..

Outcome: one printed line that combines a variable and a piece of text.

Here's the move that trips up every coder arriving from JavaScript, Python, or C#. In Lua, you do not join strings with +. The plus sign is for numbers only. For strings, Lua uses a special operator: two dots, .., called the concatenation operator. Say it out loud, con-cat-en-ation: it means "chain together".

Restart. The console shows Hello, Los Santos! and Slots open: 64. Two things worth noticing.

What happened on each line
  1. greeting .. '!' took the string inside greeting and glued the string '!' onto the end. Result: one longer string.
  2. 'Slots open: ' .. maxPlayers mixed a string and a number. Lua noticed you were concatenating and quietly turned the number 64 into the string '64' so it could join them. This auto-conversion only happens with .., not with math operators.
The classic beginner error

Writing print('Slots: ' + maxPlayers). Lua sees +, assumes you want math, tries to add a string to a number, and crashes with attempt to perform arithmetic on a string value. If your intent is "glue two things into one line of text", the operator is always ..: never +.

Spacing matters

Lua doesn't add spaces for you. 'Slots:' .. maxPlayers prints Slots:64: crammed together. If you want a space, put it inside the string: 'Slots: ' with a trailing space. What you type is exactly what you get.

Step 5

Inspect with type() and add comments

Outcome: you can ask Lua what any variable is, and leave notes for yourself.

Two small tools that pay off forever. First, type(): a built-in function that tells you what kind of value you have. Second, comments, lines Lua ignores, which you use to leave notes for humans (including future-you).

Restart. The three type() calls print string, number, number. That's Lua confirming what's in each box. You'll use type() every time something acts weird, it takes two seconds and saves twenty minutes.

The two comment styles
  • -- a single-line comment: two dashes, then anything to the end of the line. Lua ignores it.
  • --[[ block ]]: opens with --[[, closes with ]]. Everything between is ignored, even across many lines.
  • Use comments to explain why, not what. The code already shows what. Comments explain the reason.
Pro move

When a value misbehaves, drop print(type(thing)) above it. Nine times out of ten you'll find a number where you expected a string, or the other way around. It's the fastest debugging trick in Lua.

One more note for the road: always write local when you first create a variable. If you forget it, you accidentally make a global: a value visible to every other resource on the server. Globals are a common source of hard-to-track bugs. We'll unpack why in Module 02. For now: always local.

Common mistakes & what the console tells you

Error: attempt to perform arithmetic on a string value

You used + to join strings. Change it to ... Two dots, not one plus.

Error: attempt to concatenate a nil value

You're concatenating a variable that doesn't exist, a typo in the name, or you wrote the .. before declaring the variable above it. Lua reads top to bottom; a variable has to exist before you use it.

Output looks like Slots:64 with no space

Not an error, a formatting bug. You wrote 'Slots:' with no trailing space. Put the space inside the string: 'Slots: '.

Error: ')' expected near '..'

You typed a single dot instead of two. 'Hi' . name is wrong. 'Hi' .. name is right. The operator is two dots, always.

Lua gotchas: the traps that catch every newcomer

If you have written JavaScript, Python, or C# before, most of Lua will feel familiar, but a handful of things are just different, and they bite quietly. Lua rarely crashes on these, it does the wrong thing silently and lets your code break three lines later. Read this list once now. Come back to it the first time something behaves in a way you cannot explain.

The trapWhat other languages doWhat Lua does
Counting in a listFirst item is index 0First item is index 1. list[0] is nil.
"Not equal"!=~= (a tilde, then equals). != is a syntax error.
"And" / "or"&& / ||The words and / or. Plain English.
What counts as false0, '', empty things are often falseOnly false and nil are false. 0 and '' are true.
"No value"null / None / undefinednil. One word for all of it.
Add one to a counterx++ or x += 1Write it out: x = x + 1. No shortcuts exist.
The one that surprises everyone: only false and nil are falsy

In JavaScript, if (0) is false and skips the block. In Lua, if 0 then is true and runs the block. The same goes for an empty string: '' is truthy in Lua. The only two values Lua treats as false are false itself and nil (which means "no value here", Lua's version of null). Everything else, including 0 and '', is true. If a check is firing when you swear it should not, this is usually why.

One clean way to print: string.format

You have been gluing pieces with .., and that is perfect for two or three parts. Once a line has many parts, the dots pile up and get hard to read. Lua gives you a tidier tool, format, where you write the sentence once with %s placeholders and pass the values in order. It is the same trick C and Python use. %s means "drop a value here as text", and %d means "drop a whole number here".

You have already met format without realising it, the strict lesson example at the top of this page uses (' ... '):format(...) to build its log line. Now you know what those %s markers are doing.

What you learned

  • Numbers have no quotes and you can do math with them (+, -, *, /, %)
  • The .. operator (two dots) joins strings together -- never use + for that in Lua
  • You can change what is inside a variable (re-assignment) without writing local again
  • type(x) tells you whether a value is a string, number, or something else -- your fastest debug tool
  • Comments (-- and --[[ ]]) let you leave notes that Lua ignores
