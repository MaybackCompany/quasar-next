Prop interaction

Use this when a script needs one world object, one interaction, and one cleanup path.

Purpose
Spawn one prop and remove it on resource stop.
Dependencies
FiveM client runtime. ox_target is optional.
Warning
The client can show interaction. The server must validate rewards.
Expected output
[QUIET][prop_interaction][spawn] then [cleanup] on stop.

Files

  • fxmanifest.lua
  • client.lua

Install

  1. Create resources/[local]/qu_prop_interaction/.
  2. Copy fxmanifest.lua and client.lua.
  3. Add ensure qu_prop_interaction to server.cfg.
  4. Restart the resource and check F8 for [QUIET][prop_interaction][spawn].
  5. Stop the resource and confirm [QUIET][prop_interaction][cleanup].
Security warning

Do not give items, money, or job progress from the prop click. Send a server event and validate there.
