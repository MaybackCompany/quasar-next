Resource skeleton

The minimum FiveM resource: a manifest, a shared logger, and the client / server split. Start every new resource from this shape.

Purpose
Boot a new resource with the right files in the right places.
Dependencies
ox_lib (optional). FiveM server cerulean or newer.
Files
2 starters: fxmanifest.lua, logger.lua.
First log
[QUIET][<resource>:boot][ok] resource started
Lesson
Lesson 7: a real resource
Wall rule

Server-only files (SQL, money, job changes) must never appear in client_scripts. The wall is the one rule that protects everything else.

Files

  • fxmanifest.lua, declares fx_version, game, scripts. Lives in the resource root.
  • logger.lua, shared structured logger. Loads on both sides via shared_scripts.

Install

  1. Create resources/[local]/qu_starter/.
  2. Copy fxmanifest.lua and logger.lua into it.
  3. Add empty config.lua, client.lua, server.lua beside them.
  4. Add ensure qu_starter to server.cfg after ensure ox_lib.
  5. Restart the server and watch for the resource start line.

Next steps

  • Read Lesson 8 for the client / server / shared wall.
  • Add the event gateway next so the client can ask the server safely.
  • Replace config.lua with real values only when a feature needs them.
