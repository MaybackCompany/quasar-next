One log helper, one capture wrapper around pcall. Errors print with traceback. The shape every safe handler reuses.

Lesson
Lesson 20: debugging and error capture
Capture, then stop

After a captured error, return early. Do not continue the path that failed. A swallowed error that keeps running is worse than a crash, it corrupts state silently.

Files

  • server.lua, log, capture, quiet_capture command for a forced traceback.

server.lua

View raw server.lua

Install

  1. Create resources/[local]/qu_debug/.
  2. Copy server.lua and add a manifest with server_script 'server.lua'.
  3. Add ensure qu_debug in server.cfg.
  4. From the server console run quiet_capture.
  5. Match [QUIET][debug_capture][command] followed by [trace] with the stack.
