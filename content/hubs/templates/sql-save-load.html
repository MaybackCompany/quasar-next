SQL save and load

One save, one load. Always with ? placeholders and a params array. Citizenid as the key. The shape every persistence query reuses.

Purpose
Parameterized save and load against MariaDB / MySQL.
Dependencies
oxmysql.
Files
1 server-side script.
First log
[QUIET][sql_save][save] citizenid=…
Lesson
Lesson 22: parameterized SQL
SQL injection

Never join player input into the query string. Always pass values through the params array. 'WHERE citizenid = ?' with { citizenid }, never 'WHERE citizenid = ' .. citizenid.

Files

  • server.lua, saveNote with MySQL.insert, loadNotes with MySQL.query.

Schema

server.lua

View raw server.lua

Install

  1. Run the schema in your database client.
  2. Create resources/[local]/qu_notes/.
  3. Copy server.lua and add a manifest with server_script 'server.lua' and dependency 'oxmysql'.
  4. Add ensure qu_notes after ensure oxmysql in server.cfg.
  5. Wire the gateway from event gateway so the client can request saveNote safely.

Next steps

  • Read Lesson 22 for the five queries: insert, update, select one, select many, delete.
  • Wrap saveNote + loadNotes in MySQL.transaction.await when you write multiple rows.
  • Index any column you filter by. citizenid is already indexed in the schema above.
