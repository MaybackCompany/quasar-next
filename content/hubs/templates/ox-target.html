ox_target

Add a global vehicle target on start, remove it on stop. Target selection is the prompt only. Server still validates the action.

Purpose
Register and clean up an ox_target option.
Dependencies
ox_target. ox_lib optional.
Files
1 client-side script.
First log
[QUIET][ox_target][select] entity=…
Lesson
Lesson 18: ox_target and ox_lib
Selection is not permission

The client can spoof a select. Always send the action through a net event and validate job, distance, and ownership on the server before doing real work.

Files

  • client.lua, addGlobalVehicle on boot, removeGlobalVehicle on stop.

client.lua

View raw client.lua

Install

  1. Create resources/[local]/qu_target/.
  2. Copy client.lua and add a manifest with client_script 'client.lua' and dependency 'ox_target'.
  3. Add ensure qu_target after ensure ox_target in server.cfg.
  4. Aim at any vehicle in game and watch [QUIET][ox_target][select].
  5. Add a server handler on quasar:target:validate from event gateway.

Next steps

  • Read Lesson 18 for ox_target and ox_lib menus.
  • Pair with event gateway so the select triggers a validated server action.
  • Swap addGlobalVehicle for addBoxZone when the target is a static spot.
