Script templates hub

Use these as starter code, not final systems. Each template includes purpose, dependencies, warning, and next steps.

Default
Framework-agnostic first, with QBCore notes where needed.
Rule
Validate on the server before changing money, jobs, SQL, or entities.
Logging
[QUIET][module][action] appears in each template.
Cleanup
Entity and target templates include stop cleanup.
10-minute setup checklist
  1. Create one resource folder.
  2. Copy the matching template files.
  3. Add dependencies before your resource in server.cfg.
  4. Run once and match the expected log or QA result.
  5. Change only one rule at a time.
