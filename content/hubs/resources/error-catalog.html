FiveM and Lua error catalog

Search common console errors. Each entry says what the message means, the cause that hits 90 percent of the time, and the smallest fix to try first. Use the filter chips to narrow by language or framework.

  • attempt to call a nil value (global 'X')

    Lua
    Means
    You called a function that was never defined, or was defined on the wrong side.
    Cause
    Typo in the function name, or the function lives on the server but you called it from the client (or vice versa).
    Fix
    Check spelling. Confirm client_script vs server_script in fxmanifest.lua. If the function is from another resource, use exports.

  • attempt to index a nil value (local 'X')

    Lua
    Means
    You tried to read a field on something that is nil.
    Cause
    A variable was never assigned, a player object was not loaded yet, or a database row returned nothing.
    Fix
    Guard with if X then ... end before access. Wait for the player to load before reading their data. Print the value to confirm what you actually have.

  • attempt to concatenate a nil value

    Lua
    Means
    You used .. on something that is nil.
    Cause
    A variable in the string was never set, or a function returned nothing.
    Fix
    Use tostring(value) or default with (value or "fallback") before concatenation. Print every part separately to find which one is nil.

  • attempt to perform arithmetic on a string value

    Lua
    Means
    You tried to do math on text.
    Cause
    A value came from JSON, a config, or a database column as a string.
    Fix
    Wrap with tonumber(value) before the math. If the conversion fails, validate the input.

  • 'end' expected (to close 'function' / 'if' / 'for' at line N)

    Lua
    Means
    A block was opened but never closed before end of file.
    Cause
    Missing end, or an extra end earlier that closed the wrong block.
    Fix
    Indent every block. Match each function, if, for, while, do to its end. Use a Lua linter or VS Code with the sumneko Lua extension.

  • unfinished string near <eof>

    Lua
    Means
    A string was opened with a quote but never closed.
    Cause
    Missing closing " or ', or an unescaped quote inside a string.
    Fix
    Find the opening quote on the reported line. For strings with quotes inside, use [[long bracket]] form.

  • bad argument #N to 'X' (Y expected, got nil)

    Lua
    Means
    A built-in or framework function was called with the wrong type for argument N.
    Cause
    A variable holding the argument was nil at call time.
    Fix
    Print the argument before the call. Check function docs for the expected type and order.

  • Couldn't load resource X

    FiveM
    Means
    The server tried to start a resource and the manifest failed to parse or files were missing.
    Cause
    Syntax error in fxmanifest.lua, missing fx_version or game line, or referenced files that do not exist.
    Fix
    Open the manifest. Confirm fx_version 'cerulean' and game 'gta5'. List only files that actually exist on disk.

  • script:resource error: file '@resource/file.lua' not found

    FiveM
    Means
    The manifest references a file that is not in the resource folder.
    Cause
    Typo in the path, wrong case (Linux servers are case-sensitive), or the file was renamed.
    Fix
    Match the case exactly. Use ls on Linux to confirm the file name.

  • No such export X in resource Y

    FiveM
    Means
    You called exports.Y:X() but resource Y did not declare that export.
    Cause
    The export line is missing in Y's manifest, or Y started after your script and the call ran too early.
    Fix
    In Y's manifest, add export 'X' (client) or server_export 'X'. In your manifest, declare dependency 'Y' so Y starts first.

  • Trying to network spawn entity from unloaded script

    FiveM
    Means
    You called a server-side spawn while the resource was already shutting down or never finished starting.
    Cause
    An onResourceStop handler tried to spawn, or a thread fired before onResourceStart finished.
    Fix
    Gate spawns behind onResourceStart. Inside onResourceStop, only delete entities, never create them.

  • SetTimeout / Wait is nil on client

    FiveM
    Means
    You used a Node-style API in Lua, or a server function from a client script.
    Cause
    Mixing JS examples into Lua, or copying server code into a client file.
    Fix
    In Lua, use Citizen.Wait(ms) or Wait(ms) inside a Citizen.CreateThread. Remove Node patterns like setTimeout.

  • QBCore is nil / GetCoreObject() returns nil

    QBCore
    Means
    You tried to use QBCore before it was ready, or qb-core never started.
    Cause
    Resource started before qb-core, or the export call ran at file load time before the framework registered.
    Fix
    Add dependency 'qb-core' in your manifest. Wrap the call in a thread or run it inside an event handler, not at file load time.

  • ESX is nil / esx:getSharedObject never fires

    ESX
    Means
    The legacy ESX init pattern timed out without responding.
    Cause
    Newer ESX versions removed the shared-object event. Some old tutorials still teach it.
    Fix
    Use ESX = exports['es_extended']:getSharedObject() directly. Add dependency 'es_extended' to your manifest.

  • attempt to index a nil value (global 'lib')

    ox
    Means
    You called lib.notify or another ox_lib function before ox_lib loaded.
    Cause
    Missing @ox_lib/init.lua import in your manifest.
    Fix
    Add to fxmanifest.lua: shared_script '@ox_lib/init.lua'. Restart the resource.

  • ox_target zone never triggers

    ox
    Means
    The interaction never registers, even when standing in the zone.
    Cause
    Coordinates are wrong, the entity model is not loaded, or the zone was created on the wrong side.
    Fix
    Print the player coords next to the target. Use exports.ox_target:addBoxZone on the client. Confirm distance is at least 2.0.

  • Server crashed / out of memory / 100 percent CPU

    FiveM
    Means
    A thread is running without yielding, or memory keeps growing.
    Cause
    An infinite loop without Wait, or a table being appended forever (event leak).
    Fix
    Add Wait(0) minimum inside every while true do. Search for events that grow a list without ever clearing.

  • SQL error: Table 'X' doesn't exist

    FiveM
    Means
    The script queries a table that the database does not have.
    Cause
    The migration was never run, or the script connects to the wrong database.
    Fix
    Run the resource's .sql file in your database tool. Confirm mysql_connection_string in server.cfg points to the right database.
