Production expansion

Module 03B - Natives

Outcome

You will group natives by entity, ped, vehicle, and HUD use.

Why it matters

Native names look random at first. Families make the docs easier to scan.

Vocabulary

Ped native
A native that reads or changes a character.
Example: PlayerPedId() returns your ped.
Vehicle native
A native that reads or changes a vehicle.
Example: SetVehicleEngineOn changes engine state.
HUD native
A native that affects screen UI.
Example: BeginTextCommandThefeedPost starts a feed notice.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Swap GetEntityCoords for a vehicle native.
  • Add one HUD notification.
  • Write one line explaining the native family.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_native_family' doesn't exist means the migration did not run.
