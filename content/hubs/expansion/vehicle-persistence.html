Module 09A - Vehicles

Outcome

You will save a vehicle plate with validated server data.

Why it matters

Vehicle persistence fails when clients choose final state. The server must validate.

Vocabulary

Plate
The visible vehicle identifier.
Example: QS1234 is a plate.
Mods
Vehicle upgrade data.
Example: engine level and color.
Despawn
Remove a vehicle from the world.
Example: cleanup after storage.

Setup

Resource

server

resources/qu_vehicle_persist/server.lua
Ensure order
ensure qu_vehicle_persist

Minimal example

Expected output: [QUIET][vehicle_persist][store] prints after a valid plate is stored.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Reject plates over 8 chars.
  • Add damage fields.
  • Log despawn success.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_vehicle_persist' doesn't exist means the migration did not run.
