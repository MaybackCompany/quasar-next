Module 08A - Entities

Outcome

You will track spawned entities and clean them on resource stop.

Why it matters

Untracked entities leak. Leaks become server lag and visual clutter.

Vocabulary

Mission entity
An entity your script intends to manage.
Example: SetEntityAsMissionEntity marks it.
Registry
A table that tracks created handles.
Example: spawnedProps[id] = entity.
Cleanup path
Code that deletes entities later.
Example: onResourceStop deletes props.

Setup

Resource
qu_entity_owner
Side
client
File path
resources/qu_entity_owner/client.lua
Ensure order
ensure qu_entity_owner

Minimal example

Expected output: [QUIET][entity_owner][cleanup] prints for each tracked entity.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Track one prop handle.
  • Delete it manually.
  • Restart the resource and confirm cleanup.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_entity_owner' doesn't exist means the migration did not run.
