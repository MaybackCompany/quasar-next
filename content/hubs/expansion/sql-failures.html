Module 07B - SQL

Outcome

You will catch SQL failures and log query context.

Why it matters

Database errors need action names. Slow queries need timing.

Vocabulary

Connection failure
The database is unreachable.
Example: oxmysql cannot connect.
Transaction
A group of writes that should succeed together.
Example: transfer debit and credit.
Slow query
A query that takes too long.
Example: SELECT took 320ms.

Setup

Resource
qu_sql_failures
Side
server
File path
resources/qu_sql_failures/server.lua
Ensure order
ensure qu_sql_failures

Minimal example

Expected output: [QUIET][sql_failures][query] prints query timing.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Log the action name.
  • Add a slow query threshold.
  • Write the exact connection error.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_sql_failures' doesn't exist means the migration did not run.
