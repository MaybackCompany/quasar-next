Module 11 - Debug

Outcome

You will create one logger helper used by every script file.

Why it matters

One logger makes screenshots searchable. Mixed print styles hide the real failure.

Vocabulary

Context
Extra data attached to a log.
Example: source, action, amount.
Stop condition
A rule that halts unsafe work.
Example: missing Player stops payout.
Traceback
The call path for an error.
Example: debug.traceback(err).

Setup

Resource
qu_logger
Side
shared
File path
resources/qu_logger/logger.lua
Ensure order
ensure qu_logger

Minimal example

Expected output: [QUIET][logger][error] and [QUIET][logger][trace] print together.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Add source to the log.
  • Add payload size.
  • Use the helper in another file.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_logger' doesn't exist means the migration did not run.
