Module 07A - SQL

Outcome

You will run one versioned migration and verify the schema row.

Why it matters

Manual table edits drift. Versioned migrations explain what changed.

Vocabulary

Migration
SQL that changes schema.
Example: CREATE TABLE IF NOT EXISTS.
Version table
A table that records applied migrations.
Example: qu_schema_versions.
Index
A lookup helper for frequent queries.
Example: INDEX citizenid speeds reads.

Setup

Resource
qu_migrations
Side
server
File path
resources/qu_migrations/server.lua
Ensure order
ensure qu_migrations

Minimal example

Expected output: [QUIET][migrations][applied] prints after the migration runs.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Add an audit table migration.
  • Add one useful index.
  • Record the migration name.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_migrations' doesn't exist means the migration did not run.
