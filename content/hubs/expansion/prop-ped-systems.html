Module 08B - Entities

Outcome

You will store placed object state without trusting client labels.

Why it matters

Placed props need ids. Player-provided labels need validation.

Vocabulary

Stateful prop
A placed object with saved data.
Example: stash id and coords.
Ped system
Managed NPC spawn and cleanup code.
Example: shop ped registry.
Drop cleanup
Cleanup when a player leaves.
Example: playerDropped removes temp props.

Minimal example

Expected output: [QUIET][prop_ped][place] prints after a validated place request.

Production checklist

  1. Confirm the code is on the correct side.
  2. Validate every payload before acting.
  3. Log with [QUIET][module][action].
  4. Add one cleanup or rollback path.
  5. Copy the expected output into your test note.

Things to try

  • Reject unknown kinds.
  • Add playerDropped cleanup.
  • Store one validated coord.

Common failure modes

  • attempt to call a nil value means a native, export, or helper is unavailable on this side.
  • No such export GetCoreObject in resource qb-core means the dependency is missing or ensured late.
  • attempt to index a nil value means a table, player object, or entity handle is missing.
  • SQL error: Table 'qu_prop_ped' doesn't exist means the migration did not run.
