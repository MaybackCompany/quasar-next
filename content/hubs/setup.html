Beginner rule

If this page takes longer than expected, that does not mean you are behind. Setup always feels slow the first time. The win condition is not speed. The win condition is: when you do decide to practice locally, you know exactly where things are.

Step 1

Install your editor first

Outcome: you can open folders, edit FiveM Lua files, and get auto-complete, snippets, formatting, and syntax help made for FiveM work.

  1. Install VS Code.
  2. Install the core FiveM extensions below, not just one random Lua extension.
  3. Turn on line numbers if they are off.
  4. Create a folder somewhere easy to find, for example C:\fivem-dev\.

That folder will hold your local server and your practice resources when you are ready. Keep it simple. Do not bury it inside Downloads.

Beginner setup rule

For FiveM, use a small stack that understands CFX Lua, natives, and your framework. Skip the clutter. The list below keeps only the tools that are mandatory or genuinely useful.

Install first: the core FiveM stack

  • Cfx Lua IntelliSense is the main FiveM extension to install. It adds CFX runtime globals, native definitions, and FiveM-aware IntelliSense.
  • Lua Language Server (Sumneko Lua) gives you Lua autocomplete, linting, definitions, and cleaner error checking. The Cfx extension depends on this, but it is still worth knowing by name.
  • Prettier - Code formatter keeps HTML, JSON, Markdown, and web-side files readable and consistent.

Install if you use these frameworks

  • ESX Legacy Snippets if you are writing ESX resources and want official ESX snippet help.
  • FiveM QBCore Snippets if you are writing QBCore resources and want common QB patterns generated faster.
Good to use

Turn on VS Code’s built-in Auto Save in settings instead of installing another autosave extension. If you are starting from zero, install the core stack first, then add ESX or QBCore snippets only if that is the framework you are actually studying.

Step 2

Create a local FiveM server folder

Outcome: you have one folder that contains your local server files and the config file that starts everything.

Follow the official FiveM server setup guide and stop when you have a local server that can start. The important thing is not memorising every setup detail. The important thing is understanding the folder layout.

server.cfg is the file the server reads when it starts. resources/ is where scripts live. [local] is a common folder for your own work, separate from downloaded resources.

Step 4

Learn the two consoles

Outcome: you know where server messages appear and where client messages appear.

  • Server console: the txAdmin live console or the FXServer window. This is where print from server scripts shows up.
  • F8 console: inside the FiveM game client. This is where print from client scripts shows up.

This distinction matters for the rest of the course. If a lesson says “check the server console,” it does not mean F8. If it says “check F8,” it does not mean txAdmin.

Step 5

Optional reality check

Outcome: if you choose to run a local server later, you already know what the loop will look like.

  1. Start your local server.
  2. Join it with FiveM.
  3. Open F8.
  4. Confirm the server console is updating while the server is running.

You do not need this step to understand the website lessons. It only matters once you decide to practice on a live local server.

Cheat sheet: absolute-zero vocabulary

server.cfg
The main config file your local server reads at startup.
resources/
The folder where FiveM resources live.
[local]
A common folder name for your own practice scripts.
Server console
Logs from server-side scripts and server startup.
F8 console
Logs from client-side scripts inside the game.
Resource
A folder that FiveM can run as a script package.
