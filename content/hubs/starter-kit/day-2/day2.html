Day 2 · Lua · ~25 min

Variables, conditionals, tables

Yesterday you made code run. Today you make code remember things, choose between options, and group related values together. This is still beginner territory. No fancy math. No hidden tricks. Just three building blocks you will use in almost every FiveM script you ever touch.

Day 2 in plain English

Think of today like this:

A variable is a labelled box. You put one value in it so you can reuse it later.

A conditional is a fork in the road. If something is true, go this way. Otherwise, go that way.

A table is a backpack. It can hold many values under one name.

If you understand those three sentences, you already understand the point of the whole day.

Concepts

Variables: local by default

Always prefix with local. local cash = 500 lives only inside the current file or block. A global like cash = 500 leaks into every other script on the server and is a common source of bugs.

Conditionals: if / elseif / else

Lua reads top to bottom. The first condition that's true runs; the rest skip. Always close with end. Use == to compare, ~= for "not equal" (not !=).

Tables: arrays, dictionaries, and both at once

A table is Lua's one data structure. It can be an array: {'apple', 'bread'}. It can be a map: {cash = 500, bank = 1000}. It can be both. You iterate arrays with ipairs, maps with pairs.

The goal of Lab 1 -> in plain English

Imagine your server has a player counter. You want one tiny function that looks at that number and prints a mood:

  • 0 players means empty
  • 1 to 5 means cozy
  • 6 to 15 means busy
  • Anything above 15 means packed

This is just an if / elseif / else ladder. Read it as plain English: if the number is this, print this. otherwise if the number is that, print that.

Lab 1 · Classify a server's online count

You're writing a snippet for your admin panel: given the number of online players, print a one-word status. empty for 0, cozy for 1 to 5, busy for 6 to 15, packed for anything higher. Wrap it in a function and call it four times with the test values.

What to do:

  1. Write local function classifyOnline(n).
  2. Inside it, start with the most specific check: if n == 0 then.
  3. Add the middle ranges with elseif.
  4. End with else for the biggest values.
  5. Inside each branch, print the exact word the test wants.

If the print-count test fails: one of your branches is printing twice, or you added an extra debug print. The right answer prints exactly four lines, one for each function call below the function.

If a word test fails: compare the exact spelling. packed and busy must match exactly.

The goal of Lab 2 -> in plain English

Now you have a table that works like a tiny inventory:

  • water = 3
  • bread = 7
  • bandage = 2

Your job is to open the backpack, look at each item, and print one readable line for each entry. You do that with a loop. Because this is a key-value table, the loop shape is for name, count in pairs(inventory) do ... end.

Lab 2 · Print an inventory receipt

You've been handed a pre-made inventory table. Loop over it with pairs and print one line per entry in the format item: water x 3. Order doesn't matter, content does.

What to do:

  1. Start a loop with for name, count in pairs(inventory) do.
  2. Inside the loop, glue text together with ...
  3. Print exactly: item: then the item name, then x , then the count.
  4. Close the loop with end.
Pattern. for name, count in pairs(inventory) do print('item: ' .. name .. ' x ' .. count) end.

If the order looks different: that's fine. pairs does not promise order. The tests only care that all three lines exist.

Before you move on

If Day 2 landed properly, you should now be able to say:

  • "A variable stores one value under a name."
  • "An if block decides which branch runs."
  • "A table groups many values together."

Day 3 uses all three ideas immediately. If those sentences still feel fuzzy, run today's labs one more time before moving on.

What you learned today

  • local creates a variable that stays inside the current file or block. Always use it so your value does not leak into other scripts.
  • if / elseif / else / end makes decisions. Lua checks each condition from top to bottom and runs the first branch that is true.
  • Tables store lists and maps under one name. Curly braces {} create them. They are the backbone of almost every FiveM script.
  • pairs() iterates over a table's keys and values so you can look at every entry one at a time.

Common mistakes

Variable works inside the function but is nil outside

You forgot the local keyword, or you put local inside a block and tried to read it outside. A local variable only exists in the block where it was created. If you need it outside, declare it at a higher scope.

Condition never triggers even though the value looks correct

You used a single = instead of ==. In Lua, = assigns a value. == compares two values. Writing if n = 0 then is a syntax error. Write if n == 0 then.

First item in my table is nil or the loop skips index 0

Lua tables start at index 1, not 0. If you wrote inventory[0] expecting the first item, you will get nil. Use inventory[1] for the first entry. This catches almost every beginner who has seen other languages before.
