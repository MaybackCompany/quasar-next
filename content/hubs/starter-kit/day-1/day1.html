Day 1 · Lua + FiveM · ~25 min

Your first command, one line at a time

Never coded before? Perfect. Today you'll type two tiny scripts and watch them work. You don't need a server, an editor, or any installs. Your browser runs everything. By the end, you'll have made a chat command and a tiny ticker.

What is Lua, and why do we type funny words?

A script is a list of instructions you write in a file. The computer reads them from top to bottom and does what they say. Lua is the language FiveM (the GTA 5 multiplayer engine) uses for those instructions. We type Lua; the game obeys.

You'll see two kinds of things all day:

  • A "function" is a built-in action. You call it by writing its name, then parentheses, then what it needs. Example: print('hi') calls the function print with the text 'hi'.
  • A "string" is any text wrapped in quotes. 'hello' is a string. "hello" is also a string. Same thing in Lua.

That's literally the whole vocabulary you need for today.

The goal of Lab 1 -> in plain English

We want this to happen:

  • A player types /hello in chat.
  • The server prints the text Hello Quasar.

To make that happen, we use a built-in function called RegisterCommand. Think of it as saying: "Hey game, when someone types this command, run this code."

Here is the finished answer. Do not worry about understanding every character yet. Just read it from top to bottom:

RegisterCommand('hello', function(source, args, rawCommand)
  print('Hello Quasar')
end, false)

Four things are happening in those three lines:

  1. RegisterCommand( -> this starts the built-in function call. Everything until the matching ) is what we pass into it.
  2. 'hello' -> this is the command name as text. Typing /hello will run it. Do not type the slash in the string. The game adds that part.
  3. function(source, args, rawCommand) ... end -> this is the code that runs when the command is used. Everything between function(...) and end is the handler. The values in the brackets are extra info from the game. You can ignore them for now.
  4. false -> this is just yes or no. It asks if the command is admin-only. false means no, so anyone can use it.

Inside the function, there's one line: print('Hello Quasar'). print writes one line to the console. That's the line the test is checking for.

Exact-match warning. The test checks the text exactly. It must be Hello Quasar -> capital H, capital Q, one space, nothing extra. If you write hello quasar or Hello, Quasar!, the test will fail.

The editor below already has most of the code. The line -- your code here is a comment. Lua ignores any line that starts with --. It is just a placeholder for you to replace.

What to do:

  1. Click in the editor. Find the line that says -- your code here.
  2. Select that whole line and delete it.

  4. Click the green Run & test button (or press Ctrl+Enter, Cmd+Enter on Mac).

The goal of Lab 2 -> in plain English

We want this to happen:

  • A background task starts.
  • It waits one second, then prints tick.
  • It does that three times total, then stops.

Two new built-in functions do this:

  • CreateThread(...) -> starts a background task. In plain English: "run this code while the rest of the script keeps going."
  • Wait(1000) -> pauses for 1000 milliseconds. That means one second. Quick conversion: 1000 = 1 second, 500 = half a second.

And we'll use a loop to do the same thing three times without copy-pasting. Lua's simplest loop looks like this:

for i = 1, 3 do
  -- whatever is in here runs 3 times
end

The i is just a counter that goes 1, 2, 3. You don't need to use it; it just has to be there.

Here's the finished answer for Lab 2:

CreateThread(function()
  for i = 1, 3 do
    Wait(1000)
    print('tick')
  end
end)

Reading it top to bottom:

  1. CreateThread(function() ... end) -> starts a background task. Everything between function() and end is part of that task.
  2. for i = 1, 3 do ... end -> run the code inside 3 times.
  3. Wait(1000) -> pause for one second.
  4. print('tick') -> write tick to the console.

Do not worry about the real wait. In this sandbox, Wait happens instantly -> you will not actually sit there for 3 seconds. The test only checks that you used it.

Lab 2 · Type it in, press the button

What to do:

  1. Click in the editor. Find the line -- your code here inside CreateThread.
  2. Delete that line.
  3. In its place, type (or paste) the three-line loop from above: 
    for i = 1, 3 do
  Wait(1000)
  print('tick')
end
  4. Press Run & test (or Ctrl/Cmd+Enter).

All four tests on the right turn green. Done.

If "CreateThread was called exactly once" fails: you either removed the outer CreateThread(function() ... end) or wrapped it twice. Hit Reset and only edit inside the function body.

If "prints 'tick' three times" shows 0 or 1 print: your loop didn't run 3 times. Use the numeric for loop: for i = 1, 3 do ... end.

If it shows the wrong text: the string is tick exactly (lowercase, no quotes in the output). Compare with the Sandbox output panel on the right.

If "Wait(1000) was used between prints" fails: you printed without waiting. Add Wait(1000) on the line above print('tick').

What you learned today

  • RegisterCommand creates slash commands that players can type in chat. You give it a name, a function to run, and whether it is admin-only.
  • print sends text to the server console. It is lowercase. Capital-P Print does not exist and will break your script.
  • CreateThread starts a background task so code can run without blocking everything else.
  • Wait(ms) pauses the current thread for a number of milliseconds. Without it inside a loop, the game freezes.

Common mistakes

Script error: attempt to call a nil value (RegisterCommand)

You forgot to put quotes around the command name. RegisterCommand(hello, ...) tries to find a variable called hello, which does not exist. Write RegisterCommand('hello', ...) with quotes so Lua treats it as text.

Nothing prints, no error either

You probably wrote Print with a capital P. Lua is case-sensitive. Print is not the same as print. The capital version does not exist, so Lua silently does nothing. Always use lowercase print.

Game freezes completely after running my loop

You forgot Wait inside your loop. Without a pause, the loop runs millions of times per second and the game cannot do anything else. Always put Wait(ms) inside any loop that runs repeatedly. Even Wait(0) is enough to let the game breathe.
