Day 4 · ox_lib · ~30 min

ox_lib: the modern UX toolkit

ox_lib is a ready-made toolbox for player-facing UI. Instead of building every pop-up and menu from scratch, you fill in a small table and let the library do the heavy lifting. Today you start with the two easiest wins: one notification and one tiny menu.

Day 4 in plain English

Yesterday you made code talk across the network. Today you make code feel nicer for the player.

lib.notify is a pop-up message.

lib.registerContext is a menu definition.

lib.showContext opens that menu.

The big idea is simple: you describe the UI, then ox_lib draws it for you. That is why the code feels shorter than the result looks on screen.

Concepts

One import, many tools

On the client, ox_lib gives you a ready-made table named lib. A table is just Lua's box for grouping things together. Inside that box live helpers like lib.notify (show a message), lib.registerContext (define a menu), lib.inputDialog (ask the player for input), and lib.progressBar.

Everything takes one options table

Instead of passing three or four separate arguments, ox_lib usually takes one table with named fields. Example: lib.notify({ title = 'Hi', description = 'All good', type = 'success' }). Read it like a form: title = heading, description = body text, type = color/style.

Lab 1 · Send a welcome notification

Call lib.notify to show a small pop-up message to the player. Use title Welcome, description Day 4 of your starter kit, and type success.

What to do: call lib.notify once and pass one table with exactly three fields: title, description, and type.

Lab 2 · A one-option context menu

Register a context menu with id qu_day4, title Starter Kit Menu, and one option titled Ping the server. Then show the menu. Read the table top to bottom: menu id, menu title, then the list of buttons inside options.

What to do: fill in the missing button title inside options, then call lib.showContext('qu_day4') under the menu definition.

What you learned today

  • lib.notify shows a pop-up message to the player. You pass it one table with title, description, and type.
  • lib.registerContext creates a menu definition. You give it an id, a title, and a list of options, each with its own title.
  • lib.showContext opens a menu you already registered. Pass the menu's id string.
  • ox_lib options are always tables with specific named keys. Get the key names wrong and the UI will be empty or broken.

Common mistakes

Script error: lib is a nil value

ox_lib is not listed in your resource's fxmanifest.lua dependencies. Add shared_script '@ox_lib/init.lua' to your manifest. Without it, the lib table does not exist and every lib. call crashes.

Menu opens but has no buttons or shows blank entries

Your options table is formatted wrong. Each button must be a table inside the options list with at least a title key. Example: options = { { title = 'My Button' } }. Missing the inner curly braces or the title key gives you empty slots.

Menu was registered but nothing appears on screen

You forgot to call lib.showContext('your_menu_id') after registering. Registering only saves the menu definition. You must explicitly show it with showContext for the player to see anything.
