Step 1

Confirm hosting access and write down panel facts

Outcome: you know where the server is managed before uploading files.

  1. Confirm the current Goodleaf benefit, package, and onboarding instructions in Discord.
  2. Open the Goodleaf panel link you were given.
  3. Find the service name, server IP, SFTP host, SFTP username, and game port.
  4. Save them in hosting-notes.md. Do not paste passwords into Discord or public docs.

Expected screenshot

To confirm, the hosted panel's service overview shows your server name and a running status.

Step 2

Open the file manager or SFTP path

Outcome: you can see where server.cfg and resources/ live.

  1. In the panel, open Files or File Manager.
  2. If the panel provides SFTP, open WinSCP.
  3. Use the SFTP host, username, and password or key from the panel.
  4. Navigate to the server root. Common paths are /home/container/, /server-data/, or a provider-specific game folder.
  5. Find or create these locations: server.cfg, resources/, and backups/.

Expected proof

The file tree shows server.cfg and resources/. To confirm, the panel file manager (or WinSCP) lists both server.cfg and the resources folder.

If your host uses a different path

Do not guess. Search the panel docs for "FiveM server.cfg path" or ask support: "Where is the active server.cfg that FXServer reads on boot?"

Step 3

Create the hosted database and prove it connects

Outcome: MariaDB exists and accepts a real login before txAdmin or scripts depend on it.

  1. In the hosting panel, open Databases or MySQL.
  2. Create a database named something clear, such as fivem or the provider's assigned name.
  3. Copy host, port, database name, username, and password into your private password manager.
  4. Open HeidiSQL on your PC.
  5. Create a session named Hosted FiveM.
  6. Use the panel database host, user, password, port, and database name.
  7. Click Open.

Expected proof

HeidiSQL connects and the left tree shows the hosted database.

Secret handling

The database password belongs in private notes and server.cfg only. Blur it in screenshots. Rotate it if posted publicly.

Step 4

Set startup, artifact, and txAdmin proof

Outcome: the hosted service starts and txAdmin opens.

  1. Open the panel's Startup or Settings page.
  2. Confirm the FiveM artifact or build setting. Use a pinned known-good build when the panel allows it.
  3. Confirm the license key variable or sv_licenseKey line is set.
  4. Start or restart the service.
  5. Open txAdmin from the panel button or the txAdmin URL/port provided by the host.
  6. Run status in txAdmin Live Console.

Expected output

txAdmin shows the server online. Live Console prints a status response with players, uptime, and endpoint information.

Step 5

Install one first resource only

Outcome: one script proves the full hosted workflow before you install a pack.

  1. Make a backup folder: backups/before-first-resource/.
  2. Copy server.cfg and the resource's config into that backup folder.
  3. Upload the resource folder to resources/[quasar]/RESOURCE_NAME/ or the category folder your docs require.
  4. Import SQL through HeidiSQL if the resource includes it.
  5. Add dependencies above the resource in server.cfg.
  6. Run refresh, then ensure RESOURCE_NAME in txAdmin.
  7. Join with FiveM direct connect: connect SERVER_IP:30120.
  8. Check txAdmin console and F8.

Expected output

txAdmin prints Started resource RESOURCE_NAME, F8 has no repeating error, and the script's first visible feature works in-game.

Failure recovery

HeidiSQL cannot connect to the hosted database

Check whether the host blocks remote database access. Some panels require using the panel import tool instead of HeidiSQL from your PC. Ask support for the allowed host, port, and remote access rule.

txAdmin opens but the server will not start

Read the first red console line. Common causes are missing license key, wrong database connection string, bad artifact setting, or server.cfg path mismatch.

The first resource fails

Stop the resource, restore the backup copy of its config, check dependencies, verify SQL import, and ask the seller with console proof. Do not install a second resource while the first one is broken.
