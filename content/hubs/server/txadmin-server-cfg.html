Server Owner Path › Lesson 4

txAdmin and server.cfg

Learn the control room and the boot checklist. The goal is simple: back up first, edit the active config file, restart safely, then prove the server came back clean.

Outcome
You can find server.cfg, back it up, edit an ensure line, restart, and read proof.
Analogy
server.cfg is the boot checklist. ensure is the start switch for a resource folder.
Test
Restart one small resource from txAdmin and read the console result.
Duration
30-45 minutes.
Prerequisites
txAdmin access and a dated backup folder.
Pass condition
A backed-up config, one edited ensure line, and a clean restart proof are recorded.
Do this first

Before editing server.cfg, copy it to a dated backup. Beginners should never make config changes without a return path.

Step 1

Open txAdmin and find the active config

Outcome: you know which server.cfg txAdmin is actually reading.

  1. Open txAdmin in your browser. Local default: http://localhost:40120.
  2. Click Settings.
  3. Open CFG Editor or the config editor label your txAdmin version uses.
  4. Confirm it shows the same server.cfg you plan to edit in VS Code or the hosting panel.
  5. If using a hosted panel, also open Files → find server.cfg.

Expected screenshot

To confirm, the txAdmin CFG Editor shows server.cfg open.

Step 2

Back up before editing

Outcome: you can undo a bad config change in under one minute.

  1. Create a backup folder named with today's date, for example backups\2026-06-01-before-server-cfg-edit\.
  2. Copy server.cfg into that folder.
  3. Copy any resource config you are about to change.
  4. Write a one-line reason in change-log.md.

Core commands

Run these in the txAdmin Live Console. The name after the command is the resource folder name, not the full folder path.

Step 3

Read a resource folder name correctly

Outcome: you ensure the folder name FiveM sees.

If the script lives at resources/[quasar]/qs-smartphone-pro, the resource name is qs-smartphone-pro.

Step 4

Edit ensure order intentionally

Outcome: dependencies start before scripts that need them.

Expected proof

When the server restarts, console output shows Started resource oxmysql, then ox_lib, then framework, then the dependent script.

Step 5

Restart and prove the server is clean

Outcome: the edit survived a clean restart.

  1. Save server.cfg.
  2. Open txAdmin → Live Console.
  3. Click Restart Server or use your hosting panel restart button.
  4. Wait until the server shows online.
  5. Run status.
  6. Open FiveM and direct connect.

Common console outputs

Console lineMeaningOwner action
Couldn't find resourceThe folder name or path is wrong.Check spelling, extraction depth, and run refresh.
No such exportA dependency did not start first or is missing.Move dependency above the script and restart both.
Access denied for userDatabase username/password/host is wrong.Fix mysql_connection_string and test in HeidiSQL.
Unknown databaseThe database name in config does not exist.Create it or correct the database name.
Started resource XResource started successfully.Now test the feature in-game.

Rollback if the restart fails

  1. Stop the server or keep it offline.
  2. Copy your backed-up server.cfg over the active server.cfg.
  3. Start the server.
  4. Run status.
  5. Connect once from FiveM.
  6. Write the result in change-log.md.
