Server Owner Path › Lesson 6

Staff and permissions

Separate roleplay trust from admin power. Staff can be trusted socially and still not need database, file, or full txAdmin access.

Outcome
A tested staff matrix, ACE permissions, txAdmin role mapping, and two test accounts.
Why
Unclear staff power creates abuse risk, drama, and production damage faster than most script bugs.
Artifact
Permission table, add_ace/add_principal examples, test evidence, and revoke steps.
Duration
40-60 minutes.
Prerequisites
txAdmin access, Discord identifiers, and two test accounts.
Pass condition
Admin and helper test accounts can do only the actions listed in the staff matrix.
Least privilege

Give every person the smallest access that lets them do their job today. Add more only after a clear need and a test.

Permission models

txAdmin role
Controls dashboard access: live console, players, resources, settings, bans, and admin manager.
ACE permission
FiveM's server permission system. It grants or denies named powers such as command.restart.
Principal
An identity or group. Example: identifier.fivem:1234567 or group.admin.
Framework permission
QBCore, ESX, or admin-menu permissions inside gameplay scripts. These are separate from txAdmin.
Step 1

Create two test accounts

Outcome: you can prove normal players cannot use staff tools.

  1. Ask one trusted friend to join as a normal player. Name the record test-normal.
  2. Ask one staff candidate to join. Name the record test-moderator.
  3. Open txAdmin → Players → click each player → Info.
  4. Copy each fivem: identifier into staff-matrix.md.

Expected proof

The file contains two different identifiers. To confirm, the txAdmin player-info tab shows the fivem: identifier.

Step 2

Add ACE groups in server.cfg

Outcome: groups exist before people are added to them.

Open server-data/server.cfg in VS Code. Add this near the staff permissions section, not inside a random resource config.

Do not grant command allow to everyone

add_ace group.admin command allow is broad. Use it only when you understand every command that becomes available. Beginner default is explicit commands only.

Step 3

Map people to groups with add_principal

Outcome: one test moderator has limited power and one normal player has none.

Replace the sample identifiers with the exact values copied from txAdmin.

Expected output after restart

The server starts without invalid principal or typo errors. The moderator can use only the commands you granted.

txAdmin role mapping

  1. Open txAdmin.
  2. Click Settings → Admin Manager.
  3. Create or edit the staff user.
  4. Use Master Admin only for owners.
  5. For moderators, grant player moderation actions, not settings, resources, or console unless they need it.
  6. For developers, prefer staging dashboard access. Production resources access needs owner approval.

Expected screenshot

To confirm, the txAdmin Admin Manager lists the owner, admin, and moderator roles named clearly.

Step 4

Test permission boundaries

Outcome: you have proof that normal players are blocked.

  1. Restart the server from txAdmin.
  2. Ask test-normal to open F8 and try a staff command, such as kick 1 test.
  3. Ask test-moderator to run an allowed moderator command.
  4. Ask test-moderator to run a blocked owner command, such as restart qb-core.
  5. Copy the console result into staff-permission-test.md.

Revoke access when staff leave

  1. Remove their add_principal line from server.cfg.
  2. Remove or downgrade them in txAdmin → Settings → Admin Manager.
  3. Rotate shared passwords they knew.
  4. Revoke SFTP, database, Discord bot, and payment access if they had any.
  5. Restart and confirm their old account cannot open staff tools.

Failure recovery

Staff powers do not work after editing server.cfg

Check the identifier prefix. identifier.fivem:123, identifier.discord:123, and license:123 are different. Copy from txAdmin instead of typing from memory.

A moderator can restart resources

Search server.cfg for broad rules like command allow. Remove broad grants, restart, and retest with the moderator account.

A former staff member still has power

Check all three layers: server.cfg principals, txAdmin Admin Manager, and framework/admin-menu config. Revoke in every layer, then restart.

Pass condition

You are done when the normal test account is blocked, the moderator test account has limited powers only, the owner account has full approved powers, txAdmin roles match the matrix, and revoke steps are written.
